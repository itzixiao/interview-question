# Java 基础数据类型

## 基本数据类型（8种）

| 类型      | 位数 | 范围             | 默认值      | 包装类       |
|---------|----|----------------|----------|-----------|
| byte    | 8  | -128 ~ 127     | 0        | Byte      |
| short   | 16 | -32768 ~ 32767 | 0        | Short     |
| int     | 32 | -2³¹ ~ 2³¹-1   | 0        | Integer   |
| long    | 64 | -2⁶³ ~ 2⁶³-1   | 0L       | Long      |
| float   | 32 | IEEE 754       | 0.0f     | Float     |
| double  | 64 | IEEE 754       | 0.0d     | Double    |
| char    | 16 | 0 ~ 65535      | '\u0000' | Character |
| boolean | 1  | true/false     | false    | Boolean   |

## 类型转换

### 自动类型转换（小 → 大）

```java
byte → short → int → long → float → double
char → int
```

```java
byte b = 10;
int i = b;  // 自动转换
```

### 强制类型转换（大 → 小）

```java
double d = 3.14;
int i = (int) d;  // 3，截断小数

int big = 130;
byte small = (byte) big;  // -126，溢出
```

### 表达式类型提升

```java
byte a = 10;
byte b = 20;
// byte sum = a + b;  // 编译错误！a + b 结果是 int
int sum = a + b;       // 正确
```

## 包装类

### 对应关系

| 基本类型    | 包装类       |
|---------|-----------|
| byte    | Byte      |
| short   | Short     |
| int     | Integer   |
| long    | Long      |
| float   | Float     |
| double  | Double    |
| char    | Character |
| boolean | Boolean   |

### 自动装箱与拆箱

```java
// 自动装箱
Integer i = 100;  // Integer.valueOf(100)

// 自动拆箱
int n = i;        // i.intValue()
```

### 缓存机制

```java
Integer a = 100;
Integer b = 100;
a == b;  // true（-128 ~ 127 缓存）

Integer c = 200;
Integer d = 200;
c == d;  // false（超出缓存范围）
```

## 引用数据类型

### 类（Class）

```java
String str = new String("Hello");
Object obj = new Object();
```

### 接口（Interface）

```java
List<String> list = new ArrayList<>();
Runnable runnable = () -> {};
```

### 数组（Array）

```java
int[] arr = new int[10];
String[] strs = {"a", "b", "c"};
```

## 基本类型 vs 引用类型

| 特性   | 基本类型         | 引用类型                |
|------|--------------|---------------------|
| 存储内容 | 值本身          | 对象引用（地址）            |
| 存储位置 | 栈            | 堆（对象）+ 栈（引用）        |
| 默认值  | 有（0, false等） | null                |
| 比较   | == 比较值       | == 比较地址，equals 比较内容 |
| 泛型支持 | 不支持          | 支持                  |

## 使用建议

1. **优先使用基本类型**
    - 性能更好（无装箱拆箱）
    - 内存占用小

2. **必须使用包装类的场景**
    - 集合泛型（`List<Integer>`）
    - 需要 null 值
    - 需要使用方法（`Integer.parseInt()`）

3. **注意空指针**
   ```java
   Integer i = null;
   int n = i;  // NullPointerException！
   ```

4. **比较使用 equals**
   ```java
   Integer a = 200;
   Integer b = 200;
   a.equals(b);  // true
   a == b;       // false
   ```

## 💡 高频面试题

**问题 1：Java 有哪些基本数据类型？**

答案：
Java 有 **8 种基本数据类型**，分为 4 类：

**整数类型（4 种）：**

| 类型 | 位数 | 范围 | 默认值 |
|------|------|------|--------|
| byte | 8 | -128 ~ 127 | 0 |
| short | 16 | -32768 ~ 32767 | 0 |
| int | 32 | -2³¹ ~ 2³¹-1 | 0 |
| long | 64 | -2⁶³ ~ 2⁶³-1 | 0L |

**浮点类型（2 种）：**

| 类型 | 位数 | 范围 | 默认值 |
|------|------|------|--------|
| float | 32 | IEEE 754 | 0.0f |
| double | 64 | IEEE 754 | 0.0d |

**字符类型（1 种）：**

- char：16 位，0 ~ 65535，默认值 '\u0000'

**布尔类型（1 种）：**

- boolean：true/false，默认值 false

**注意：**

- Java 没有 unsigned（无符号）类型
- boolean 在数组中用 byte 表示
- char 可以存储中文：`char c = '中';`

**问题 2：自动装箱和拆箱的原理？**

答案：
**自动装箱（Auto-boxing）：** 基本类型 → 包装类
**自动拆箱（Unboxing）：** 包装类 → 基本类型

**示例：**

```java
// 自动装箱
Integer i = 100;  // 编译器转换为：Integer.valueOf(100)

// 自动拆箱
int n = i;  // 编译器转换为：i.intValue()
```

**底层实现：**

```java
// 装箱调用 valueOf()
public static Integer valueOf(int i) {
    if (i >= IntegerCache.low && i <= IntegerCache.high)
        return IntegerCache.cache[i + (-IntegerCache.low)];
    return new Integer(i);
}

// 拆箱调用 intValue()
public int intValue() {
    return value;
}
```

**性能影响：**

```java
// ❌ 频繁装箱拆箱，性能差
Integer sum = 0;
for (int i = 0; i < 1000000; i++) {
    sum += i;  // 每次都要装箱拆箱
}

// ✅ 使用基本类型
int sum = 0;
for (int i = 0; i < 1000000; i++) {
    sum += i;
}
```

**空指针风险：**

```java
Integer i = null;
int n = i;  // NullPointerException！自动拆箱时抛出
```

**问题 3：Integer a = 100 和 Integer b = 100，a == b 的结果？**

答案：
**结果：true**

**原因：** Integer 使用了**缓存池**技术，缓存了 -128 ~ 127 之间的整数。

**源码分析：**

```java
public static Integer valueOf(int i) {
    assert IntegerCache.high >= 127;
    if (i >= IntegerCache.low && i <= IntegerCache.high)
        return IntegerCache.cache[i + (-IntegerCache.low)];
    return new Integer(i);
}
```

**示例对比：**

```java
// 缓存范围内
Integer a = 100;  // valueOf(100) → 返回缓存对象
Integer b = 100;
System.out.println(a == b);  // true

// 超出缓存范围
Integer c = 200;  // valueOf(200) → new Integer(200)
Integer d = 200;
System.out.println(c == d);  // false
System.out.println(c.equals(d));  // true
```

**其他包装类的缓存：**

| 类型 | 缓存范围 | 说明 |
|------|----------|------|
| Byte | -128 ~ 127 | 全部缓存 |
| Short | -128 ~ 127 | 全部缓存 |
| Integer | -128 ~ 127 | 可配置 |
| Long | -128 ~ 127 | 可配置 |
| Character | 0 ~ 127 | 只缓存正数 |
| Boolean | true, false | 全部缓存 |

**配置 Integer 缓存范围：**

```bash
# JVM 参数
-XX:AutoBoxCacheMax=200  # 扩大缓存范围
```

**最佳实践：**

```java
// ✅ 推荐 - 基本类型优先
int i = 100;

// ⚠️ 注意 - 包装类比较用 equals
Integer a = 200, b = 200;
a.equals(b);  // true
a == b;       // false
```

**问题 4：基本类型和包装类的区别？**

答案：

| 特性   | 基本类型         | 包装类                  |
|------|--------------|----------------------|
| 所属   | java.lang 包外 | java.lang 包内         |
| 存储   | 栈上存值         | 堆上存对象                |
| 默认值  | 0、false 等    | null                 |
| 泛型支持 | ❌ 不支持        | ✅ 支持                 |
| 集合使用 | ❌ 不能         | ✅ 能（如 List<Integer>） |
| 方法调用 | ❌ 无方法        | ✅ 有丰富方法              |
| 内存占用 | 小            | 大（对象头 + 对齐）          |

**使用场景对比：**

```java
// ✅ 基本类型 - 计算、性能敏感
int count = 0;
double price = 9.99;

// ✅ 包装类 - 集合、需要 null
List<Integer> list = new ArrayList<>();
Integer age = null;  // 表示未设置

// ✅ 工具方法
Integer.parseInt("123");
Double.parseDouble("3.14");
```

**内存对比：**

```java
int n = 100;           // 4 字节
Integer obj = 100;     // 约 24 字节（对象头 12 + int 4 + 对齐 8）
```

**选择建议：**

- ✅ 优先使用基本类型（性能好）
- ✅ 必须用包装类：集合泛型、需要 null、调用方法
- ⚠️ 注意自动装箱拆箱的性能开销

**问题 5：类型转换的规则？**

答案：

**自动类型转换（隐式）：**

```java
// 小类型 → 大类型（安全）
byte → short → int → long → float → double
char → int

byte b = 10;
int i = b;  // 自动转换

char c = 'A';
int n = c;  // 65，ASCII 码
```

**强制类型转换（显式）：**

```java
// 大类型 → 小类型（可能丢失精度）
double d = 3.14;
int i = (int) d;  // 3，截断小数

long l = 100L;
int n = (int) l;  // 100
```

**溢出示例：**

```java
int big = 130;
byte small = (byte) big;  // -126，溢出

// 原因：byte 范围 -128~127，130 超出范围
// 二进制：10000010 → 补码解释为 -126
```

**表达式类型提升：**

```java
byte a = 10;
byte b = 20;
// byte sum = a + b;  // ❌ 编译错误！
int sum = a + b;      // ✅ 正确，结果是 int

// 原因：byte/short/char 运算时自动提升为 int
```

**特殊转换：**

```java
// String → 基本类型
int i = Integer.parseInt("123");
double d = Double.parseDouble("3.14");
boolean b = Boolean.parseBoolean("true");

// 基本类型 → String
String s1 = String.valueOf(123);
String s2 = 123 + "";  // 不推荐
```

**问题 6：float 和 double 的精度问题？**

答案：

**精度丢失问题：**

```java
float f1 = 0.1f;
float f2 = 0.2f;
System.out.println(f1 + f2);  // 0.30000001，不是 0.3

double d1 = 0.1;
double d2 = 0.2;
System.out.println(d1 + d2);  // 0.30000000000000004
```

**原因：**

- 浮点数采用 IEEE 754 标准
- 0.1 等小数无法用二进制精确表示
- 类似十进制无法精确表示 1/3

**解决方案：**

**方案 1：BigDecimal（推荐）**

```java
BigDecimal bd1 = new BigDecimal("0.1");  // 用 String 构造
BigDecimal bd2 = new BigDecimal("0.2");
BigDecimal result = bd1.add(bd2);
System.out.println(result);  // 0.3
```

**方案 2：转为整数计算**

```java
// 金额用分表示，不用元
int price = 999;  // 9.99 元
int quantity = 3;
int total = price * quantity;  // 2997 分 = 29.97 元
```

**方案 3：四舍五入输出**

```java
double result = 0.1 + 0.2;
System.out.printf("%.2f", result);  // 0.30
```

**金融计算规范：**

```java
// ❌ 禁止使用 double 表示金额
double price = 9.99;

// ✅ 使用 BigDecimal
BigDecimal price = new BigDecimal("9.99");

// ✅ 或使用整数（分）
long priceInCents = 999;
```

**问题 7：char 类型能存储中文吗？**

答案：

**可以！** char 是 16 位 Unicode 字符，可以存储中文。

```java
char c1 = '中';  // ✅ 正确
char c2 = '\u4e2d';  // ✅ Unicode 编码
char c3 = 20013;  // ✅ 十进制 Unicode

System.out.println(c1);  // 中
System.out.println((int) c1);  // 20013
```

**Unicode 范围：**

- 基本汉字：\u4e00 ~ \u9fff（20902 个）
- 扩展 A：\u3400 ~ \u4dbf
- 扩展 B：\u20000 ~ \u2a6df（需两个 char）

**注意事项：**

```java
// 生僻字（超出基本平面）
String s = "𠮷";  // Unicode: \u20bb7
char c = s.charAt(0);  // 只能获取高代理项
System.out.println(c);  // 乱码

// 正确处理
int codePoint = s.codePointAt(0);  // 获取完整 Unicode
System.out.println(codePoint);  // 134007
```

**常见字符：**

```java
char newline = '\n';     // 换行
char tab = '\t';         // 制表符
char backslash = '\\';   // 反斜杠
char quote = '\'';       // 单引号
char chinese = '中';     // 中文
```

**问题 8：boolean 类型的底层实现？**

答案：

**JVM 层面：**

- boolean 在数组中用 **byte** 表示（1 字节）
- 单个 boolean 变量用 **int** 表示（4 字节）

**源码证据：**

```java
// boolean[] 在 JVM 中是 byte[]
boolean[] arr = new boolean[10];  // 占用 10 字节

// 单个 boolean 变量
boolean flag = true;  // 占用 4 字节（int）
```

**为什么这样设计？**

1. **字节对齐**：CPU 按字节访问更高效
2. **历史原因**：JVM 指令集支持
3. **性能考虑**：避免位操作的开销

**内存占用：**

```java
// 理论：1 位
// 实际：1 字节（数组）或 4 字节（单个变量）

boolean b = true;  // 4 字节
boolean[] arr = new boolean[10];  // 10 字节 + 数组头
```

**使用建议：**

```java
// ✅ 正常使用
if (flag) {}

// ⚠️ 大量 boolean 用 BitSet 优化
BitSet bitSet = new BitSet(100);  // 100 个布尔值只用 13 字节
bitSet.set(0, true);
```
