## 一、注解基础概念

### 1.1 什么是注解

**定义：** 注解（Annotation）是 Java 5 引入的一种元数据机制，用于在代码中添加标记信息，可以在编译、类加载、运行时被读取和处理。

**注解的本质：** 注解是一种特殊的接口，继承自 `java.lang.annotation.Annotation`。

```java
// 定义注解
public @interface MyAnnotation {
    String value();
}

// 编译后的字节码
public interface MyAnnotation extends Annotation {
    String value();
}
```

### 1.2 注解的作用

| 作用阶段     | 典型应用       | 示例                                   |
|----------|------------|--------------------------------------|
| **编译时**  | 编译检查、代码生成  | `@Override`、`@Deprecated`、Lombok     |
| **类加载时** | 字节码处理、动态代理 | AspectJ 编译时织入                        |
| **运行时**  | 反射读取、框架处理  | Spring `@Autowired`、`@Transactional` |

### 1.3 注解 vs 注释

```
┌─────────────────────────────────────────────────────────────────────┐
│  对比项        │  注解（Annotation）      │  注释（Comment）        │
├─────────────────────────────────────────────────────────────────────┤
│  本质         │  特殊接口                 │  文本说明               │
│  编译后       │  保留在字节码中           │  被丢弃                 │
│  可被程序读取  │  ✅ 可以                  │  ❌ 不可以              │
│  作用         │  影响程序行为             │  仅供人阅读             │
│  示例         │  @Override               │  // 这是注释            │
└─────────────────────────────────────────────────────────────────────┘
```

## 二、元注解详解

元注解是用于修饰注解的注解，定义注解的行为和生命周期。

### 2.1 @Target - 指定注解作用目标

```java

@Target(ElementType.METHOD)  // 只能用于方法
public @interface MyMethodAnnotation {
}
```

**ElementType 取值：**

```
┌─────────────────────────────────────────────────────────────────────┐
│  ElementType 常量      │  作用目标                                   │
├─────────────────────────────────────────────────────────────────────┤
│  TYPE                  │  类、接口、枚举                              │
│  FIELD                 │  字段（包括枚举常量）                        │
│  METHOD                │  方法                                       │
│  PARAMETER             │  方法参数                                    │
│  CONSTRUCTOR           │  构造方法                                    │
│  LOCAL_VARIABLE        │  局部变量                                    │
│  ANNOTATION_TYPE       │  注解类型                                    │
│  PACKAGE               │  包                                         │
│  TYPE_PARAMETER        │  类型参数（泛型） Java 8+                    │
│  TYPE_USE              │  类型使用处 Java 8+                          │
└─────────────────────────────────────────────────────────────────────┘
```

**多目标示例：**

```java

@Target({ElementType.TYPE, ElementType.METHOD, ElementType.FIELD})
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {
    String value() default "";
}
```

### 2.2 @Retention - 指定注解生命周期

```java

@Retention(RetentionPolicy.RUNTIME)  // 运行时保留
public @interface MyAnnotation {
}
```

**RetentionPolicy 取值：**

```
┌─────────────────────────────────────────────────────────────────────┐
│  策略      │  存在阶段    │  反射可读  │  典型应用                    │
├─────────────────────────────────────────────────────────────────────┤
│  SOURCE    │  源码        │  ❌        │  @Override、@SuppressWarnings│
│  CLASS     │  字节码      │  ❌        │  默认策略，编译时处理        │
│  RUNTIME   │  运行时      │  ✅        │  自定义注解，反射读取        │
└─────────────────────────────────────────────────────────────────────┘
```

**生命周期图示：**

```
源代码 (.java)
    │
    ▼ 编译
字节码 (.class) ──────────► SOURCE 级别注解在此丢失
    │
    ▼ 类加载
运行时 (JVM) ─────────────► CLASS 级别注解在此丢失
    │
    ▼
反射读取 ─────────────────► RUNTIME 级别注解可被读取
```

### 2.3 @Documented - 生成 JavaDoc

```java

@Documented
@Retention(RetentionPolicy.RUNTIME)
public @interface ApiDoc {
    String value();
}
```

**效果：** 使用 `javadoc` 生成文档时，被 `@Documented` 修饰的注解会出现在生成的文档中。

### 2.4 @Inherited - 允许子类继承

```java

@Inherited
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {
    String value() default "";
}

@MyAnnotation("父类注解")
class Parent {
}

class Child extends Parent {
}  // Child 也拥有 @MyAnnotation("父类注解")
```

**注意事项：**

- `@Inherited` 只对**类继承**有效
- 接口继承、方法重写**不生效**

## 三、自定义注解语法

### 3.1 基本语法

```java
[元注解]

public @interface 注解名 {
    // 注解属性（方法形式定义）
    类型 属性名() default 默认值;
}
```

### 3.2 属性定义规则

1. **属性以方法形式定义**：无参数，有返回类型
2. **支持 default 关键字**：指定默认值
3. **value 属性特殊**：单属性时可省略属性名
4. **支持的类型**：基本类型、String、Class、枚举、注解、以上类型的数组

### 3.3 注解类型示例

#### 标记注解（无属性）

```java

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Deprecated {
    // 无属性，仅作为标记使用
}

// 使用
@Deprecated
public void oldMethod() {
}
```

#### 单值注解

```java

@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Column {
    String value();  // 唯一属性，使用时可省略属性名
}

// 使用
@Column("user_name")  // 等价于 @Column(value = "user_name")
private String userName;
```

#### 多属性注解

```java

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface ApiOperation {
    String value();                    // 接口描述

    String notes() default "";         // 备注说明

    String[] tags() default {};        // 标签分组

    boolean hidden() default false;    // 是否隐藏
}

// 使用
@ApiOperation(value = "获取用户", tags = {"用户模块"})
public User getUser(Long id) {
}
```

#### 枚举属性注解

```java
public enum SensitiveType {
    PHONE, ID_CARD, BANK_CARD, EMAIL, ADDRESS
}

@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Sensitive {
    SensitiveType value();
}

// 使用
@Sensitive(SensitiveType.PHONE)
private String phone;
```

#### 数组属性注解

```java

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Cacheable {
    String[] value();           // 缓存名称

    String key() default "";    // 缓存键

    int expire() default 300;   // 过期时间（秒）
}

// 使用
@Cacheable(value = {"user", "userList"}, key = "#id", expire = 600)
public User getUserById(Long id) {
}
```

## 五、实战案例

### 5.1 权限控制注解

**场景：** 实现方法级别的权限控制，类似 Spring Security 的 `@PreAuthorize`。

```java
// 1. 定义权限逻辑枚举
public enum Logical {
    AND,  // 需要拥有所有权限
    OR    // 拥有任意一个权限即可
}

// 2. 定义权限注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RequirePermission {
    String[] value();              // 需要的权限

    Logical logical() default Logical.AND;  // 逻辑关系
}

// 3. 使用注解
class AdminController {

    @RequirePermission("user:read")
    public void getUser(Long id) {
    }

    @RequirePermission(value = {"user:read", "user:write"}, logical = Logical.AND)
    public void updateUser(Long id, String name) {
    }

    @RequirePermission(value = {"user:delete", "admin:all"}, logical = Logical.OR)
    public void deleteUser(Long id) {
    }
}

// 4. 权限校验处理器
public class PermissionChecker {

    public static void checkPermission(Method method, Set<String> userPermissions) {
        if (!method.isAnnotationPresent(RequirePermission.class)) {
            return;  // 无权限注解，放行
        }

        RequirePermission rp = method.getAnnotation(RequirePermission.class);
        String[] requiredPerms = rp.value();

        boolean hasPermission;
        if (rp.logical() == Logical.AND) {
            // AND 逻辑：需要拥有所有权限
            hasPermission = userPermissions.containsAll(Arrays.asList(requiredPerms));
        } else {
            // OR 逻辑：拥有任意一个权限即可
            hasPermission = Arrays.stream(requiredPerms)
                    .anyMatch(userPermissions::contains);
        }

        if (!hasPermission) {
            throw new SecurityException("权限不足");
        }
    }
}
```

### 5.2 参数校验注解

**场景：** 实现类似 JSR-303 Bean Validation 的参数校验功能。

```java
// 1. 定义校验注解
@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
public @interface NotNull {
    String message() default "字段不能为空";
}

@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Length {
    int min() default 0;

    int max() default Integer.MAX_VALUE;

    String message() default "长度不符合要求";
}

@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Range {
    int min() default Integer.MIN_VALUE;

    int max() default Integer.MAX_VALUE;

    String message() default "数值超出范围";
}

@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Pattern {
    String regex();

    String message() default "格式不正确";
}

// 2. 使用注解
class UserDTO {
    @NotNull(message = "用户名不能为空")
    @Length(min = 3, max = 20, message = "用户名长度必须在3-20之间")
    private String username;

    @NotNull(message = "邮箱不能为空")
    @Pattern(regex = "^[A-Za-z0-9+_.-]+@(.+)$", message = "邮箱格式不正确")
    private String email;

    @Range(min = 1, max = 150, message = "年龄必须在1-150之间")
    private Integer age;

    @Pattern(regex = "^1[3-9]\\d{9}$", message = "手机号格式不正确")
    private String phone;
}

// 3. 校验处理器
public class ValidationUtils {

    public static List<String> validate(Object obj) throws Exception {
        List<String> errors = new ArrayList<>();
        Class<?> clazz = obj.getClass();

        for (Field field : clazz.getDeclaredFields()) {
            field.setAccessible(true);
            Object value = field.get(obj);

            // @NotNull 校验
            if (field.isAnnotationPresent(NotNull.class) && value == null) {
                NotNull notNull = field.getAnnotation(NotNull.class);
                errors.add(field.getName() + ": " + notNull.message());
            }

            // @Length 校验
            if (field.isAnnotationPresent(Length.class) && value != null) {
                Length length = field.getAnnotation(Length.class);
                String strValue = value.toString();
                if (strValue.length() < length.min() || strValue.length() > length.max()) {
                    errors.add(field.getName() + ": " + length.message());
                }
            }

            // @Range 校验
            if (field.isAnnotationPresent(Range.class) && value != null) {
                Range range = field.getAnnotation(Range.class);
                int intValue = (Integer) value;
                if (intValue < range.min() || intValue > range.max()) {
                    errors.add(field.getName() + ": " + range.message());
                }
            }

            // @Pattern 校验
            if (field.isAnnotationPresent(Pattern.class) && value != null) {
                Pattern pattern = field.getAnnotation(Pattern.class);
                if (!java.util.regex.Pattern.matches(pattern.regex(), value.toString())) {
                    errors.add(field.getName() + ": " + pattern.message());
                }
            }
        }

        return errors;
    }
}

// 4. 使用示例
UserDTO user = new UserDTO();
user.

setUsername("ab");  // 太短
user.

setEmail("invalid");  // 格式错误
user.

setAge(200);  // 超出范围

List<String> errors = ValidationUtils.validate(user);
if(!errors.

isEmpty()){
        errors.

forEach(System.out::println);
}
```

### 5.3 日志记录注解

**场景：** 通过注解 + AOP 实现方法执行日志记录。

```java
// 1. 定义日志级别
public enum LogLevel {
    DEBUG, INFO, WARN, ERROR
}

// 2. 定义日志注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Log {
    String value() default "";              // 操作描述

    LogLevel level() default LogLevel.INFO; // 日志级别

    boolean recordParams() default true;    // 是否记录参数

    boolean recordResult() default true;    // 是否记录返回值
}

// 3. 使用注解
class OrderService {

    @Log(value = "创建订单", level = LogLevel.INFO)
    public String createOrder(String orderId, String product) {
        // 业务逻辑
        return "订单创建成功: " + orderId;
    }

    @Log(value = "取消订单", level = LogLevel.WARN, recordResult = false)
    public void cancelOrder(String orderId) {
        // 业务逻辑
    }
}

// 4. AOP 切面实现（Spring 环境）
@Aspect
@Component
public class LogAspect {

    @Around("@annotation(log)")
    public Object around(ProceedingJoinPoint point, Log log) throws Throwable {
        // 方法开始
        System.out.println("[" + log.level() + "] 开始执行: " + log.value());

        if (log.recordParams()) {
            System.out.println("  参数: " + Arrays.toString(point.getArgs()));
        }

        long start = System.currentTimeMillis();
        Object result = point.proceed();  // 执行目标方法
        long elapsed = System.currentTimeMillis() - start;

        System.out.println("  耗时: " + elapsed + "ms");

        if (log.recordResult() && result != null) {
            System.out.println("  返回: " + result);
        }

        return result;
    }
}
```

## 六、高频面试题

**问题 1：注解的本质是什么？**

**答：**

注解本质上是一种特殊的接口，继承自 `java.lang.annotation.Annotation`。

```java
// 定义注解
public @interface MyAnnotation {
    String value();
}

// 编译后的字节码
public interface MyAnnotation extends Annotation {
    String value();
}
```

这也是为什么注解属性定义采用方法形式的原因。

**问题 2：@Retention 的三个策略有什么区别？**

**答：**

```
┌─────────────────────────────────────────────────────────────────────┐
│  策略      │  存在阶段    │  反射可读  │  典型应用                    │
├─────────────────────────────────────────────────────────────────────┤
│  SOURCE    │  源码        │  ❌        │  @Override、@SuppressWarnings│
│  CLASS     │  字节码      │  ❌        │  默认策略，编译时处理        │
│  RUNTIME   │  运行时      │  ✅        │  自定义注解，反射读取        │
└─────────────────────────────────────────────────────────────────────┘
```

- **SOURCE**：注解仅存在于源码，编译时被丢弃
- **CLASS**：注解存在于字节码，但运行时不可见（默认策略）
- **RUNTIME**：注解存在于运行时，可通过反射读取（最常用）

---

**问题 3：如何获取注解的属性值？**

**答：**

```java
// 1. 获取类上的注解
MyAnnotation anno = clazz.getAnnotation(MyAnnotation.class);
String value = anno.value();  // 调用属性方法获取值

// 2. 获取方法上的注解
Method method = clazz.getMethod("doSomething");
MyAnnotation anno = method.getAnnotation(MyAnnotation.class);

// 3. 获取字段上的注解
Field field = clazz.getDeclaredField("name");
MyAnnotation anno = field.getAnnotation(MyAnnotation.class);
```

**问题 4：注解可以继承吗？**

**答：**

默认情况下，注解不能继承。但可以通过 `@Inherited` 元注解实现类级别的继承。

```java

@Inherited
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {
}

@MyAnnotation
class Parent {
}

class Child extends Parent {
}  // Child 也拥有 @MyAnnotation
```

**注意事项：**

- `@Inherited` 只对**类继承**有效
- **接口继承**不生效
- **方法重写**不生效

**问题 5：注解和接口的区别？**

**答：**

```
┌─────────────────────────────────────────────────────────────────────┐
│  对比项        │  注解                    │  接口                    │
├─────────────────────────────────────────────────────────────────────┤
│  定义关键字    │  @interface             │  interface               │
│  方法实现      │  不支持                 │  支持（default 方法）    │
│  默认继承      │  Annotation             │  无（可多实现）          │
│  属性定义      │  方法形式，必须有默认值  │  方法定义                │
│  使用方式      │  @注解名                 │  implements              │
│  作用          │  元数据标记              │  定义行为契约            │
└─────────────────────────────────────────────────────────────────────┘
```

**问题 6：如何设计一个通用的注解？**

**答：**

设计通用注解需要考虑以下几点：

1. **合理设置 @Target**：根据实际使用场景选择
2. **使用 RUNTIME 策略**：便于运行时反射读取
3. **提供 value 属性**：单属性时可省略属性名
4. **合理设置默认值**：减少使用时的配置量
5. **添加 @Documented**：生成 JavaDoc 文档
6. **考虑可扩展性**：预留扩展属性

```java

@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface MyAnnotation {
    String value() default "";           // 主属性，可省略属性名

    String description() default "";     // 描述信息

    String[] tags() default {};          // 标签，预留扩展
}
```

**问题 7：Spring 中常用的注解有哪些？它们是如何工作的？**

**答：**

| 注解                | 作用              | 工作原理                 |
|-------------------|-----------------|----------------------|
| `@Component`      | 标记为 Spring Bean | 类路径扫描 + 反射实例化        |
| `@Autowired`      | 依赖注入            | BeanPostProcessor 处理 |
| `@Transactional`  | 声明式事务           | AOP 代理 + 事务管理器       |
| `@RequestMapping` | 请求映射            | HandlerMapping 解析    |
| `@Configuration`  | 配置类             | CGLIB 代理             |

**核心流程：**

## 💡 最佳实践

### ✅ 推荐做法

1. **使用 RUNTIME 策略**：自定义注解通常需要运行时读取
2. **提供合理的默认值**：减少使用时的配置
3. **使用 value 作为主属性**：简化使用语法
4. **添加文档注释**：说明注解的用途和属性含义

### ❌ 避免的做法

1. **过度使用注解**：注解不是万能的，简单场景用代码更清晰
2. **忽略性能影响**：反射读取注解有一定性能开销
3. **设计过于复杂的注解**：保持单一职责原则
