# Java 泛型深入详解

## 一、泛型概述

泛型是 JDK5 引入的特性，允许在定义类、接口、方法时使用类型参数。

### 核心作用

1. **类型安全**：编译期检查类型，避免 ClassCastException
2. **代码复用**：一套代码适用于多种类型
3. **消除强制转换**：编译器自动插入类型转换

---

## 二、类型擦除（Type Erasure）

Java 泛型是通过类型擦除实现的**伪泛型**：

| 阶段  | 泛型信息         |
|-----|--------------|
| 编译期 | 完整保留，进行类型检查  |
| 运行期 | 完全擦除，只保留原始类型 |

### 擦除规则

```
无界类型参数 <T>              → 擦除为 Object
有界类型参数 <T extends Number> → 擦除为边界类型 Number
多边界 <T extends Number & Comparable> → 擦除为第一个边界
```

### 验证类型擦除

```java
List<String> stringList = new ArrayList<>();
List<Integer> intList = new ArrayList<>();

// 输出：true！运行时是同一个类
System.out.println(stringList.getClass() == intList.getClass());
```

---

## 三、查看运行时实际的泛型信息

虽然类型擦除会擦除大部分泛型信息，但以下地方保留了泛型签名（Signature 属性）：

### 1. 通过父类获取泛型信息

```java
class StringList extends ArrayList<String> {}

Type type = StringList.class.getGenericSuperclass();
// 输出：java.util.ArrayList<java.lang.String>
```

### 2. 通过字段获取泛型信息

```java
private Map<String, Integer> map;

Field field = MyClass.class.getDeclaredField("map");
Type type = field.getGenericType();
// 输出：java.util.Map<java.lang.String, java.lang.Integer>
```

### 3. 局部变量无法获取

```java
List<String> localList = new ArrayList<>();
// 无法通过反射获取局部变量的泛型类型 <String>
```

---

## 四、泛型的工作时机

| 阶段    | 泛型信息  | 操作             |
|-------|-------|----------------|
| 源码阶段  | 完整保留  | 类型检查、错误报告      |
| 编译阶段  | 部分擦除  | 插入类型转换、桥接方法    |
| 字节码阶段 | 仅保留签名 | Signature 属性记录 |
| 运行阶段  | 完全擦除  | 只知道原始类型        |

## 五、泛型通配符

## PECS 原则

**Producer Extends, Consumer Super**

| 场景  | 通配符           | 说明   |
|-----|---------------|------|
| 只读取 | `? extends T` | 生产者  |
| 只写入 | `? super T`   | 消费者  |
| 都要  | 精确类型 T        | 精确类型 |

#### 为什么"只读是生产者，只写是消费者"？

关键在于**视角的转换**："生产者/消费者"是从**集合/变量的视角**来看的，而不是从"你"的视角！

```
生产者（? extends T）：集合 → 数据 → 你的代码
                      （集合生产数据给你读）

消费者（? super T）：你的代码 → 数据 → 集合
                      （集合消费你写入的数据）
```

#### 上界通配符 `? extends T` = 生产者（只读）

```java
// 集合是"生产者"——它生产数据给你的代码使用
public double sum(List<? extends Number> producer) {
    // 你只能从list中"读取"数据
    Number n = producer.get(0);  // ✅ 可以读
    // producer.add(1);          // ❌ 不能写
    
    // 集合"生产"了数据给你消费
    double total = 0;
    for (Number n : producer) {
        total += n.doubleValue();
    }
    return total;
}
```

**为什么只能读？** 因为无法保证类型安全：

```java
List<Integer> integers = new ArrayList<>();
List<? extends Number> numbers = integers;

// 假设能写：
// numbers.add(3.14); // Double也是Number的子类
// 但numbers实际指向Integer列表！放Double进去就崩了
```

#### 下界通配符 `? super T` = 消费者（只写）

```java
// 集合是"消费者"——它消费你代码产生的数据
public void addNumbers(List<? super Integer> consumer) {
    // 你只能向list中"写入"数据
    consumer.add(1);              // ✅ 可以写
    // Integer i = consumer.get(0); // ❌ 读出来是Object
    
    // 集合"消费"了你生产的数据
}
```

**为什么只能写？** 因为读取类型不确定：

```java
List<Object> objects = new ArrayList<>();
List<? super Integer> integers = objects;

// 读取时：
// Integer i = integers.get(0); // 实际可能是String、Date...
// 因为底层是Object列表，什么都能放
```

#### 形象类比：餐厅

| 角色          | 行为                        | 对应泛型          |
|-------------|---------------------------|---------------|
| **厨师（生产者）** | 做菜给你吃 → 你只能**读取/消费**他的菜品  | `? extends T` |
| **顾客（消费者）** | 你做菜给他吃 → 你只能**写入/提供**菜品给他 | `? super T`   |

- **`? extends T`**：集合像厨师，生产数据给你读
- **`? super T`**：集合像顾客，消费你写入的数据

#### 记忆口诀

```
PECS = Producer Extends, Consumer Super

从集合的视角看：
- 集合给你数据 = 集合是生产者 = 你只能读
- 集合接受数据 = 集合是消费者 = 你只能写
```

## 六、泛型与数组

### 为什么不能创建泛型数组？

数组是协变的，泛型是不变的，两者结合会产生类型安全问题。

### 解决方案

```java
// 方案1：使用 List 代替数组
List<List<String>> list = new ArrayList<>();

// 方案2：原始类型数组
@SuppressWarnings("unchecked")
List<String>[] array = (List<String>[]) new List[10];
```

## 七、高频面试题

**问题 1：什么是类型擦除？**

答案：
**类型擦除（Type Erasure）**是指泛型信息只在编译期存在，运行时被擦除。

**具体过程：**

1. **编译期**：编译器检查泛型类型安全性
2. **编译后**：泛型类型参数被替换为原始类型（如 `Object`）
3. **运行期**：JVM 看不到泛型信息，只有原始类型

**示例：**

```java
// 编译前
List<String> list = new ArrayList<>();

// 编译后（类型擦除）
List list = new ArrayList(); // String 被擦除为 Object
```

**Java 选择类型擦除的原因：**

- **向后兼容**：保持与 Java 5 之前版本的二进制兼容性
- **平滑迁移**：旧代码无需修改即可与新代码互操作

---

**问题 2：List<String> 和 List<Integer> 运行时有什么区别？**

答案：
**完全没有区别！**运行时都擦除为 `List`。

**原因：**

```java
List<String> stringList = new ArrayList<>();
List<Integer> integerList = new ArrayList<>();

// 运行时类型
System.out.println(stringList.getClass() == integerList.getClass()); 
// true，都是 ArrayList.class
```

**影响：**

- ❌ 不能创建泛型数组
- ❌ 不能用 `instanceof` 检查泛型类型
- ❌ 不能直接 `new T()`

---

**问题 3：运行时还能获取泛型信息吗？**

答案：
**可以！**通过反射的 `Signature` 属性获取。

**可获取的场景：**

1. **父类的泛型声明**
   ```java
   class MyList extends ArrayList<String> {}
   // 可以通过 getGenericSuperclass() 获取 String
   ```

2. **字段的泛型声明**
   ```java
   private List<String> field;
   // 可以通过 field.getGenericType() 获取 String
   ```

3. **方法的返回值和参数**
   ```java
   public List<Integer> method() { }
   // 可以通过 method.getGenericReturnType() 获取 Integer
   ```

**限制：**

- 局部变量的泛型信息在编译后完全擦除
- 数组元素的泛型信息无法获取

---

**问题 4：为什么不能创建泛型数组？**

答案：
**根本原因：**数组协变 + 泛型不变 = 类型安全问题

**数组协变性：**

```java
String[] strArr = new String[10];
Object[] objArr = strArr;  // 合法：数组协变
objArr[0] = new Integer(1);  // 运行时抛 ArrayStoreException
```

**如果允许泛型数组：**

```java
// 假设允许创建泛型数组（实际不允许！）
List<String>[] stringLists = new List<String>[10];
List<Object> objectList = stringLists;  // 危险！
objectList[0] = new ArrayList<Integer>();  // 破坏类型安全！
```

**解决方案：**使用 `ArrayList<T>` 代替数组

---

**问题 5：什么是 PECS 原则？为什么只读是生产者，只写是消费者？**

答案：
**PECS 原则：** Producer Extends, Consumer Super

**核心思想：**从**集合的视角**来看待数据流向

| 通配符           | 集合视角   | 人类视角   | 能做什么     |
|---------------|--------|--------|----------|
| `? extends T` | 集合给你数据 | 你从集合读取 | 只能读（get） |
| `? super T`   | 集合接受数据 | 你向集合写入 | 只能写（add） |

**类比理解：**

- **生产者（extends）**：厨师生产菜品给你吃 → 你只能吃（只读）
- **消费者（super）**：顾客消费你做的菜 → 你只能做菜（只写）

**示例：**

```java
// 生产者 - 只能读
public void process(List<? extends Number> list) {
    Number n = list.get(0);  // ✅ 可以读
    list.add(new Integer(1));  // ❌ 不能写
}

// 消费者 - 只能写
public void fill(List<? super Integer> list) {
    list.add(new Integer(1));  // ✅ 可以写
    Integer i = list.get(0);  // ❌ 不能读（返回 Object）
}
```

**问题 6：什么是桥接方法？**

答案：
**桥接方法（Bridge Method）**是编译器生成的合成方法，用于保证多态的正确性。

**产生原因：**泛型擦除后保持多态性

**示例：**

```java
class Node<T> {
    public T data;
    public void setData(T data) { this.data = data; }
}

class StringNode extends Node<String> {
    @Override
    public void setData(String data) { this.data = data; }
}
```

**编译后生成桥接方法：**

```java
class StringNode extends Node<String> {
    // 原始方法
    public void setData(String data) { ... }
    
    // 桥接方法（编译器生成）
    public synthetic bridge setData(Object data) {
        setData((String)data);  // 强制类型转换
    }
}
```

**作用：**保证调用 `setData(Object)` 时能正确路由到 `setData(String)`

**问题 7：泛型和 C++ 模板有什么区别？**

答案：
| 特性 | Java 泛型 | C++ 模板 |
|------|----------|----------|
| 实现方式 | 类型擦除（伪泛型） | 代码生成（真泛型） |
| 运行时信息 | 无泛型信息 | 保留完整信息 |
| 性能 | 无额外开销 | 可能代码膨胀 |
| 类型检查 | 编译期 | 编译期 |
| 实例化 | 运行时单一类型 | 每个类型生成一份代码 |

**Java 泛型的特点：**

- ✅ 类型安全
- ✅ 向后兼容
- ❌ 运行时无法获取泛型信息

**C++ 模板的特点：**

- ✅ 运行时保留泛型信息
- ✅ 支持非类型参数（如 `template<int N>`）
- ❌ 可能导致代码膨胀

---

**问题 8：泛型的工作时机？**

答案：
**两个阶段：**

1. **编译期**
    - ✅ 类型检查：确保类型安全
    - ✅ 类型推断：自动推断泛型类型
    - ✅ 插入强制转换：自动添加 `(T)` 转换

2. **运行期**
    - ❌ 泛型信息已擦除
    - ✅ 只有原始类型（如 `Object`）
    - ✅ 执行实际的字节码

**示例说明：**

```java
// 编译前
List<String> list = new ArrayList<>();
list.add("hello");  // ✅ 编译通过
list.add(123);      // ❌ 编译错误

// 编译后（反编译结果）
List list = new ArrayList();
list.add("hello");  // 自动转换
list.add(Integer.valueOf(123));  // 编译错误，不会生成这行
```

**关键结论：**泛型是编译器的特性，JVM 层面不存在泛型。
