## 一、响应式编程概述

### 1.1 什么是响应式编程

**定义：**
响应式编程（Reactive Programming）是一种面向数据流和变化传播的编程范式。它允许开发者以声明式的方式处理异步数据流，而无需关心底层的线程调度和同步问题。

**核心特点：**
- **异步非阻塞**：不阻塞线程，提高资源利用率
- **事件驱动**：基于事件和回调的组合
- **声明式风格**：代码简洁，易于理解和维护
- **背压支持**：下游控制上游的生产速度

### 1.2 为什么需要响应式编程

#### 传统阻塞 I/O 的问题

```
┌─────────────────────────────────────┐
│   阻塞 I/O 的问题                    │
├─────────────────────────────────────┤
│ ❌ 线程浪费                          │
│    - 每个请求占用一个线程            │
│    - I/O 等待时线程空闲               │
│    - 高并发需要大量线程              │
│                                     │
│ ❌ 资源限制                          │
│    - 线程池大小有限                  │
│    - 上下文切换开销                  │
│    - 内存消耗大                      │
│                                     │
│ ❌ 扩展性差                          │
│    - 难以应对突发流量                │
│    - 资源利用率低                    │
└─────────────────────────────────────┘
```

#### 响应式编程解决方案

```
┌─────────────────────────────────────┐
│   响应式编程的优势                   │
├─────────────────────────────────────┤
│ ✅ 高效资源利用                      │
│    - 少量线程处理大量请求            │
│    - I/O 等待时不阻塞                 │
│    - 减少线程切换开销                │
│                                     │
│ ✅ 弹性伸缩                          │
│    - 轻松应对高并发                  │
│    - 背压保护系统                    │
│    - 故障隔离                        │
│                                     │
│ ✅ 组合性强                          │
│    - 声明式 API                      │
│    - 操作符链式调用                  │
│    - 错误处理统一                    │
└─────────────────────────────────────┘
```

### 1.3 Reactive Streams 规范

**Reactive Streams** 是一个 JVM 响应式流的标准化规范，定义了四个核心接口：

```java
// 发布者（Publisher）
public interface Publisher<T> {
    void subscribe(Subscriber<? super T> s);
}

// 订阅者（Subscriber）
public interface Subscriber<T> {
    void onSubscribe(Subscription s);
    void onNext(T t);
    void onError(Throwable t);
    void onComplete();
}

// 订阅（Subscription）
public interface Subscription {
    void request(long n);  // 请求 n 个元素
    void cancel();         // 取消订阅
}

// 处理器（Processor）
public interface Processor<T, R> extends Subscriber<T>, Publisher<R> {
}
```

## 二、Reactor 模式深入

### 2.1 Reactor 模式原理

**Reactor 模式**是一种事件驱动的设计模式，用于处理多个并发的 I/O 操作。

#### 核心组件

```
┌─────────────────────────────────────────┐
│         Reactor 模式架构                 │
│                                         │
│  ┌─────────┐     ┌──────────────────┐  │
│  │ Acceptor│────▶│  Connection      │  │
│  └─────────┘     │  Handler Pool    │  │
│                  └──────────────────┘  │
│                           │             │
│                  ┌────────┴────────┐   │
│                  │  Event Handler  │   │
│                  │  (业务逻辑)      │   │
│                  └─────────────────┘   │
└─────────────────────────────────────────┘
```

**工作流程：**
1. **Acceptor**：监听并接受新连接
2. **Reactor**：分发事件到对应的 Handler
3. **Handler**：处理具体的 I/O 事件
4. **Business Logic**：执行实际的业务逻辑

### 2.2 Reactor 模式实现对比

| 框架 | 实现 | 特点 |
|------|------|------|
| **Netty** | NIO + Reactor | 高性能网络框架 |
| **Vert.x** | Event Loop | 多语言支持 |
| **Node.js** | libuv | JavaScript 运行时 |
| **Project Reactor** | Reactor Streams | Spring WebFlux 基础 |

## 三、Project Reactor 核心 API

### 3.1 Mono 和 Flux

#### Mono：发布 0-1 个元素

```java
// 创建 Mono
Mono<String> just = Mono.just("Hello");
Mono<String> empty = Mono.empty();
Mono<String> error = Mono.error(new RuntimeException("Error"));

// 订阅消费
just.subscribe(
    data -> log.info("收到：{}", data),
    error -> log.error("错误：{}", error.getMessage()),
    () -> log.info("完成")
);
```

#### Flux：发布 0-N 个元素

```java
// 创建 Flux
Flux<Integer> range = Flux.range(1, 10);
Flux<String> array = Flux.fromArray(new String[]{"A", "B", "C"});
Flux<String> list = Flux.fromIterable(List.of("X", "Y", "Z"));

// 链式操作
Flux.range(1, 10)
    .filter(x -> x % 2 == 0)
    .map(x -> x * 2)
    .subscribe(System.out::println);
```

### 3.2 常用操作符

#### 转换操作符

```java
// map：一对一转换
Flux.range(1, 5)
    .map(x -> x * 2)
    .subscribe(System.out::println);

// flatMap：一对多转换（扁平化）
Flux.just("hello", "world")
    .flatMap(word -> 
        Flux.fromArray(word.split(""))
    )
    .subscribe(System.out::println);

// concatMap：有序扁平化
Flux.just("hello", "world")
    .concatMap(word -> 
        Flux.fromArray(word.split(""))
    )
    .subscribe(System.out::println);
```

#### 过滤操作符

```java
// filter：条件过滤
Flux.range(1, 10)
    .filter(x -> x > 5)
    .subscribe(System.out::println);

// take：取前 N 个
Flux.range(1, 100)
    .take(5)
    .subscribe(System.out::println);

// distinct：去重
Flux.just(1, 2, 2, 3, 3, 3)
    .distinct()
    .subscribe(System.out::println);
```

#### 组合操作符

```java
// merge：合并多个流（无序）
Flux.merge(
    Flux.interval(Duration.ofMillis(100)).take(3),
    Flux.interval(Duration.ofMillis(150)).take(3)
).subscribe(System.out::println);

// zip：成对组合
Flux.zip(
    Flux.just("A", "B", "C"),
    Flux.just(1, 2, 3)
).subscribe(tuple -> 
    System.out.println(tuple.getT1() + "-" + tuple.getT2())
);

// concat：顺序拼接
Flux.concat(
    Flux.just("A", "B"),
    Flux.just("C", "D")
).subscribe(System.out::println);
```

### 3.3 错误处理

```java
// onErrorReturn：返回默认值
Flux.just(1, 2, 0)
    .map(x -> 10 / x)
    .onErrorReturn(0)
    .subscribe(System.out::println);

// onErrorResume：使用备用流
Flux.just(1, 2, 0)
    .map(x -> 10 / x)
    .onErrorResume(error -> Flux.just(-1))
    .subscribe(System.out::println);

// doOnError：副作用处理
Flux.just(1, 2, 0)
    .map(x -> 10 / x)
    .doOnError(error -> log.error("发生错误", error))
    .subscribe(System.out::println);
```

### 4.2 响应式控制器示例

#### RESTful API

```java
@RestController
@RequestMapping("/api/users")
@RequiredArgsConstructor
public class UserController {
    
    private final UserService userService;
    
    @GetMapping("/{id}")
    public Mono<User> getUser(@PathVariable Long id) {
        return userService.getUserById(id);
    }
    
    @GetMapping
    public Flux<User> getAllUsers() {
        return userService.getAllUsers();
    }
    
    @PostMapping
    public Mono<User> createUser(@RequestBody User user) {
        return userService.createUser(user);
    }
    
    @PutMapping("/{id}")
    public Mono<User> updateUser(@PathVariable Long id, 
                                  @RequestBody User user) {
        return userService.updateUser(id, user);
    }
    
    @DeleteMapping("/{id}")
    public Mono<Void> deleteUser(@PathVariable Long id) {
        return userService.deleteUser(id);
    }
}
```

#### Server-Sent Events (SSE)

```java
@GetMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public Flux<User> streamUsers() {
    return userService.getAllUsers()
            .delayElements(Duration.ofSeconds(1));
}
```

### 4.3 响应式服务层

```java
@Service
@RequiredArgsConstructor
public class UserService {
    
    private final UserRepository repository;
    
    // 查询单个用户
    public Mono<User> getUserById(Long id) {
        return repository.findById(id);
    }
    
    // 查询所有用户
    public Flux<User> getAllUsers() {
        return repository.findAll();
    }
    
    // 创建用户
    public Mono<User> createUser(User user) {
        return repository.save(user);
    }
    
    // 更新用户
    public Mono<User> updateUser(Long id, User updatedUser) {
        return repository.findById(id)
                .flatMap(existingUser -> {
                    existingUser.setName(updatedUser.getName());
                    existingUser.setEmail(updatedUser.getEmail());
                    return repository.save(existingUser);
                });
    }
    
    // 删除用户
    public Mono<Void> deleteUser(Long id) {
        return repository.deleteById(id);
    }
    
    // 复杂业务：搜索并聚合
    public Flux<Order> getUserOrdersWithDetails(Long userId) {
        return repository.findOrdersByUserId(userId)
                .flatMap(order -> 
                    repository.findOrderDetails(order.getId())
                            .collectList()
                            .map(details -> {
                                order.setDetails(details);
                                return order;
                            })
                );
    }
}
```

## 五、RSocket 协议

### 5.1 什么是 RSocket

**RSocket** 是一个应用层协议，专为响应式编程设计的二进制协议。

**特点：**
- **多交互模式**：支持 4 种通信模式
- **背压支持**：内置背压机制
- **二进制协议**：高效传输
- **连接复用**：单连接多路复用

### 5.2 RSocket 四种交互模式

#### 1. Request-Response（请求 - 响应）

```java
@MessageMapping("user.get")
public Mono<User> getUser(Mono<Long> request) {
    return request.map(id -> new User(id, "User-" + id));
}
```

#### 2. Request-Stream（请求 - 流）

```java
@MessageMapping("users.stream")
public Flux<User> getUsersStream(Mono<Long> count) {
    return count.flatMapMany(n -> 
        Flux.range(1, n.intValue())
            .map(i -> new User((long) i, "User-" + i))
            .delayElements(Duration.ofMillis(500))
    );
}
```

#### 3. Fire-and-Forget（发后即忘）

```java
@MessageMapping("user.log")
public Mono<Void> logUser(User user) {
    log.info("记录用户：{}", user);
    return Mono.empty();  // 不返回任何内容
}
```

#### 4. Channel（双向流）

```java
@MessageMapping("chat")
public Flux<String> chat(Flux<String> clientMessages) {
    return clientMessages
        .doOnNext(msg -> log.info("收到：{}", msg))
        .map(msg -> "Echo: " + msg);
}
```

### 5.3 RSocket 配置

#### 服务端配置

```yaml
spring:
  rsocket:
    server:
      port: 7000
      mapping-path: /rsocket
```

#### 客户端配置

```java
@Configuration
public class RSocketConfig {
    
    @Bean
    public RSocketRequester rSocketRequester() {
        return RSocketRequester.builder()
            .connectTcp("localhost", 7000)
            .block();
    }
}
```

#### 客户端使用

```java
@Service
@RequiredArgsConstructor
public class RSocketClient {
    
    private final RSocketRequester requester;
    
    // Request-Response
    public Mono<User> getUser(Long id) {
        return requester.route("user.get")
                .data(id)
                .retrieveMono(User.class);
    }
    
    // Request-Stream
    public Flux<User> getUsersStream(int count) {
        return requester.route("users.stream")
                .data(count)
                .retrieveFlux(User.class);
    }
    
    // Fire-and-Forget
    public void logUser(User user) {
        requester.route("user.log")
                .data(user)
                .send()
                .subscribe();
    }
}
```

## 六、背压（Backpressure）机制

### 6.1 什么是背压

**背压（Backpressure）** 是响应式编程中的核心概念，指下游（消费者）向上游（生产者）发送信号，控制上游的数据生产速度。

**为什么需要背压？**
- 防止快速生产者压垮慢速消费者
- 避免内存溢出（OOM）
- 保护系统稳定性

### 6.2 背压策略

#### 1. onError（抛出异常）

```java
Flux<Long> fastProducer = Flux.interval(Duration.ofMillis(10));

fastProducer
    .onBackpressureError()  // 默认行为
    .subscribe(x -> {
        try {
            Thread.sleep(100);  // 慢速消费
        } catch (InterruptedException e) {}
        System.out.println("消费：" + x);
    });
// 结果：抛出 BackpressureException
```

#### 2. onBackpressureBuffer（缓冲）

```java
Flux<Long> fastProducer = Flux.interval(Duration.ofMillis(10));

fastProducer
    .onBackpressureBuffer(100)  // 最多缓冲 100 个
    .subscribe(x -> {
        try {
            Thread.sleep(50);
        } catch (InterruptedException e) {}
        System.out.println("消费：" + x);
    });
// 结果：缓冲区满后丢弃新数据
```

#### 3. onBackpressureDrop（丢弃）

```java
Flux<Long> fastProducer = Flux.interval(Duration.ofMillis(10));

fastProducer
    .onBackpressureDrop()  // 丢弃无法处理的数据
    .subscribe(x -> {
        try {
            Thread.sleep(50);
        } catch (InterruptedException e) {}
        System.out.println("消费：" + x);
    });
// 结果：直接丢弃新数据
```

#### 4. onBackpressureLatest（保留最新）

```java
Flux<Long> fastProducer = Flux.interval(Duration.ofMillis(10));

fastProducer
    .onBackpressureLatest()  // 只保留最新的数据
    .subscribe(x -> {
        try {
            Thread.sleep(50);
        } catch (InterruptedException e) {}
        System.out.println("消费：" + x);
    });
// 结果：总是消费最新的数据
```

### 6.3 背压在 WebFlux 中的应用

#### HTTP 层面的背压

```java
@PostMapping("/upload")
public Mono<Void> handleUpload(ServerHttpRequest request) {
    return request.getBody()
            .publishOn(Schedulers.boundedElastic())
            .doOnNext(dataBuffer -> {
                // 处理数据块
                // 如果处理慢，会自动触发背压
            })
            .then();
}
```

#### 数据库背压

```java
@GetMapping("/stream")
public Flux<User> streamUsers() {
    return userRepository.findAll()
            .limitRate(10);  // 每批请求 10 个
}
```

### 响应式编程口诀

```
异步非阻塞用 Reactor，
Mono Flux 要分清。
操作符链式来组合，
背压保护不能忘。
WebFlux 建服务，
RSocket 更高效。
阻塞代码要避免，
响应式里全异步。
```

### 常见陷阱

```
1. ⚠️ 在响应式链中使用阻塞调用（.block()）
2. ⚠️ 忘记处理错误情况
3. ⚠️ 不当的线程调度导致性能下降
4. ⚠️ 没有背压保护导致 OOM
5. ⚠️ 过度使用 flatMap 导致无序
```

### 最佳实践

```
✅ 使用 checkstyle 检查 .block() 调用
✅ 统一的错误处理策略
✅ 合理设置背压缓冲区
✅ 使用 StepVerifier 进行测试
✅ 监控响应式流的性能指标
```
