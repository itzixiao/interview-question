# 等级保护三级详解

## 概述

等级保护三级（等保三级）是指信息系统受到破坏后，会对社会秩序和公共利益造成损害，或者对国家安全造成损害。属于"监督保护级"
，必须进行公安备案，并每年进行一次等级测评。

## 一、三级等保定级标准

### 1.1 定级依据

根据 GB/T 22240-2020《信息安全技术 网络安全等级保护定级指南》：

**三级系统特征：**

| 特征维度 | 具体内容                  |
|:----:|:----------------------|
| 侵害客体 | 社会秩序、公共利益，或国家安全       |
| 侵害程度 | 造成严重损害或特别严重损害         |
| 典型系统 | 金融系统、医疗系统、政务系统、大型电商平台 |

### 1.2 典型三级系统

|  系统类型   | 定级原因            |
|:-------:|:----------------|
| 银行核心系统  | 涉及资金安全、大量用户敏感信息 |
| 医院HIS系统 | 涉及个人健康信息、医疗服务   |
| 政务服务系统  | 涉及公民信息、政务服务     |
| 大型电商平台  | 涉及交易数据、用户隐私     |
| 电力调度系统  | 涉及能源安全、社会运行     |
| 交通管理系统  | 涉及公共安全、社会秩序     |

### 1.3 与二级对比

| 对比维度 |   二级   |     三级     |
|:----:|:------:|:----------:|
| 侵害客体 | 组织合法权益 | 社会秩序/公共利益  |
| 备案要求 |  公安备案  |    公安备案    |
| 测评周期 | 建议2年一次 | **强制每年一次** |
| 测评结论 |  建议整改  |  **必须整改**  |
| 监管力度 |  指导保护  |  **监督保护**  |

## 二、三级等保安全技术要求

### 2.1 安全物理环境（增强）

### 2.4 安全计算环境（重点）

#### 2.4.1 身份鉴别（三级强制双因子）

|   要求项   | 三级要求内容                  |
|:-------:|:------------------------|
| 身份标识唯一  | 用户具有唯一标识，不可伪造           |
| 鉴别信息复杂度 | 密码长度≥8位，包含大小写字母、数字、特殊字符 |
|  双因子认证  | **必须采用两种或以上组合的鉴别技术**    |
| 登录失败处理  | 配置登录失败处理功能，限制非法登录尝试次数   |
| 会话超时管理  | 会话空闲超时自动断开              |
| 鉴别信息保护  | 鉴别信息加密存储、传输             |

**双因子认证实现：**

```java
/**
 * 双因子认证服务（三级等保要求）
 */
@Service
public class TwoFactorAuthService {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    // 双因子认证方式
    public enum TwoFactorType {
        TOTP,           // 基于时间的一次性密码（Google Authenticator）
        SMS,            // 短信验证码
        EMAIL,          // 邮箱验证码
        HARDWARE_TOKEN  // 硬件令牌
    }

    // 生成TOTP密钥
    public String generateTOTPSecret(String username) {
        String secret = GoogleAuthenticator.generateSecret();
        // 加密存储密钥
        String encryptedSecret = AESUtil.encrypt(secret, getEncryptionKey());
        redisTemplate.opsForValue().set("totp:secret:" + username,
                encryptedSecret, 365, TimeUnit.DAYS);
        return secret;
    }

    // 验证TOTP
    public boolean verifyTOTP(String username, int code) {
        String encryptedSecret = redisTemplate.opsForValue()
                .get("totp:secret:" + username);
        if (encryptedSecret == null) {
            return false;
        }
        String secret = AESUtil.decrypt(encryptedSecret, getEncryptionKey());
        return GoogleAuthenticator.verify(secret, code);
    }

    // 发送短信验证码
    public void sendSMSCode(String phone) {
        String code = RandomStringUtils.randomNumeric(6);
        redisTemplate.opsForValue().set("sms:code:" + phone, code,
                5, TimeUnit.MINUTES);
        // 调用短信服务发送
        smsService.send(phone, "您的验证码是：" + code + "，5分钟内有效");
    }

    // 双因子认证验证
    public boolean verifyTwoFactor(String username, String password,
                                   TwoFactorType type, String code) {
        // 第一步：验证密码
        if (!passwordService.verify(username, password)) {
            return false;
        }

        // 第二步：验证第二因子
        switch (type) {
            case TOTP:
                return verifyTOTP(username, Integer.parseInt(code));
            case SMS:
                String storedCode = redisTemplate.opsForValue()
                        .get("sms:code:" + getUserPhone(username));
                return code.equals(storedCode);
            case EMAIL:
                return verifyEmailCode(username, code);
            case HARDWARE_TOKEN:
                return verifyHardwareToken(username, code);
            default:
                return false;
        }
    }
}
```

**登录流程设计：**

```java
/**
 * 三级等保登录控制器
 */
@RestController
public class LoginController {

    @Autowired
    private TwoFactorAuthService twoFactorAuthService;

    // 第一步：密码认证
    @PostMapping("/auth/password")
    public Result<AuthStepResponse> passwordAuth(@RequestBody PasswordAuthRequest request) {
        // 验证密码
        if (!userService.verifyPassword(request.getUsername(), request.getPassword())) {
            // 记录登录失败
            loginFailureHandler.handle(request.getUsername());
            return Result.fail("用户名或密码错误");
        }

        // 生成临时token，用于第二步认证
        String tempToken = UUID.randomUUID().toString();
        redisTemplate.opsForValue().set("auth:temp:" + tempToken,
                request.getUsername(),
                5, TimeUnit.MINUTES);

        // 返回可用的双因子认证方式
        List<TwoFactorType> availableTypes = getUserTwoFactorTypes(request.getUsername());
        return Result.success(new AuthStepResponse(tempToken, availableTypes));
    }

    // 第二步：双因子认证
    @PostMapping("/auth/twofactor")
    public Result<LoginResponse> twoFactorAuth(@RequestBody TwoFactorAuthRequest request) {
        // 验证临时token
        String username = redisTemplate.opsForValue()
                .get("auth:temp:" + request.getTempToken());
        if (username == null) {
            return Result.fail("认证会话已过期，请重新登录");
        }

        // 验证双因子
        if (!twoFactorAuthService.verifyTwoFactor(username, null,
                request.getType(),
                request.getCode())) {
            return Result.fail("验证码错误");
        }

        // 删除临时token
        redisTemplate.delete("auth:temp:" + request.getTempToken());

        // 生成正式token
        String accessToken = jwtService.generateToken(username);

        // 记录登录成功审计日志
        auditService.log(username, AuditEventType.LOGIN_SUCCESS,
                "双因子认证登录成功");

        return Result.success(new LoginResponse(accessToken));
    }
}
```

#### 2.4.2 访问控制（三级增强）

|  要求项   | 三级要求内容                    |

```java
public class RoleSeparationModel {

            "security:policy",      // 安全策略配置
            "security:role",        // 角色管理
            "security:permission",  // 权限分配
            "security:password",    // 密码策略
            "user:password:reset"   // 密码重置
    );

    // 审计管理员职责
    public static final Set<String> AUDIT_ADMIN_PERMISSIONS = Set.of(
            "audit:view",           // 审计查看
            "audit:export",         // 审计导出
            "audit:analyze",        // 审计分析
            "audit:report"          // 审计报告
    );

    // 权限互斥检查
    public static void checkRoleConflict(Set<String> currentRoles, String newRole) {
        // 三权分立：同一用户不能同时拥有两个管理角色
        Set<String> adminRoles = Set.of("SYSTEM_ADMIN", "SECURITY_ADMIN", "AUDIT_ADMIN");

        if (currentRoles.stream().anyMatch(adminRoles::contains)
                && adminRoles.contains(newRole)) {
            throw new RoleConflictException(
                    "三权分立原则：同一用户不能兼任多个管理角色");
        }
    }

    // 敏感操作需要审批
    public enum SensitiveOperation {
        USER_DELETE,           // 用户删除
        PERMISSION_GRANT,      // 权限授予
        SECURITY_POLICY_CHANGE, // 安全策略变更
        AUDIT_LOG_DELETE       // 审计日志删除
    }
}
```

**行级访问控制实现：**

```java
/**
 * 行级数据访问控制（三级等保要求）
 */
@Aspect
@Component
public class RowLevelAccessControlAspect {

    @Autowired
    private UserService userService;

    @Before("@annotation(rowLevelControl)")
    public void checkRowAccess(JoinPoint joinPoint, RowLevelControl rowLevelControl) {
        // 获取当前用户
        String username = SecurityContextHolder.getCurrentUsername();
        User currentUser = userService.getByUsername(username);

        // 获取目标数据
        Object[] args = joinPoint.getArgs();
        Long dataId = extractDataId(args);

        // 检查数据归属
        DataEntity data = dataRepository.findById(dataId);
        if (data == null) {
            throw new AccessDeniedException("数据不存在");
        }

        // 行级权限检查
        if (!canAccess(currentUser, data, rowLevelControl.operation())) {
            // 记录越权访问审计日志
            auditService.log(username, AuditEventType.PERMISSION_DENIED,
                    "越权访问数据: " + dataId);
            throw new AccessDeniedException("无权访问该数据");
        }
    }

    private boolean canAccess(User user, DataEntity data, String operation) {
        // 数据归属检查
        if (data.getOwnerId().equals(user.getId())) {
            return true;
        }

        // 部门数据共享检查
        if (data.getDepartmentId().equals(user.getDepartmentId())
                && data.isDepartmentShared()) {
            return true;
        }

        // 特殊权限检查
        return user.hasPermission("data:" + operation + ":all");
    }
}

    }

    public void updateOrder(Long orderId, OrderDTO orderDTO) {
        // 更新订单
    }
}
```

#### 2.4.3 安全审计（三级增强）

|  要求项   | 三级要求内容                  |
|:------:|:------------------------|
|  审计范围  | **覆盖所有用户行为和重要安全事件**     |
|  审计内容  | 事件日期、时间、用户、事件类型、结果、详细信息 |
| 审计日志保护 | 防止审计日志被篡改、删除，**异地备份**   |
| 审计日志保存 | **保存期限≥12个月**           |
|  审计分析  | 定期分析审计日志，发现异常行为         |
|  审计报警  | 对重要安全事件实时报警             |

**完整审计系统实现：**

```java
/**
 * 三级等保安全审计系统
 */
@Service
public class Level3AuditService {

    @Autowired

    @Autowired
    private AuditLogBackupService backupService;

    // 审计事件类型（三级要求覆盖所有用户行为）
    public enum AuditEventType {
        // 认证相关
        LOGIN_SUCCESS, LOGIN_FAILURE, LOGOUT, SESSION_TIMEOUT,
        TWO_FACTOR_SUCCESS, TWO_FACTOR_FAILURE, PASSWORD_CHANGE,

        // 访问控制相关
        PERMISSION_GRANTED, PERMISSION_REVOKED, ROLE_ASSIGNED,
        ACCESS_DENIED, PRIVILEGE_ESCALATION,

        // 数据操作相关
        DATA_CREATE, DATA_READ, DATA_UPDATE, DATA_DELETE,
        DATA_EXPORT, DATA_IMPORT, SENSITIVE_DATA_ACCESS,

        // 系统管理相关
        USER_CREATE, USER_MODIFY, USER_DELETE, USER_ENABLE, USER_DISABLE,
        SYSTEM_CONFIG_CHANGE, SECURITY_POLICY_CHANGE,

        // 审计相关
        AUDIT_LOG_ACCESS, AUDIT_LOG_EXPORT, AUDIT_REPORT_GENERATE,

        // 安全事件
        INTRUSION_DETECTED, MALWARE_DETECTED, ANOMALY_DETECTED,

        AuditLog auditLog = AuditLog.builder()
                .eventTime(LocalDateTime.now())
                .eventId(UUID.randomUUID().toString())
                .username(event.getUsername())
                .userId(event.getUserId())
                .eventType(event.getEventType().name())
                .eventLevel(event.getEventLevel().name())
                .resource(event.getResource())
                .resourceId(event.getResourceId())
                .action(event.getAction())
                .result(event.getResult())
                .detail(event.getDetail())
                .clientIp(event.getClientIp())
                .userAgent(event.getUserAgent())
                .sessionId(event.getSessionId())
                .requestId(event.getRequestId())
                .build();

        // 保存审计日志
        auditLogRepository.save(auditLog);

        // 计算日志完整性校验码
        String checksum = calculateChecksum(auditLog);
        auditLog.setChecksum(checksum);
        auditLogRepository.save(auditLog);

        // 异地备份（三级要求）
        backupService.backupToRemote(auditLog);

        // 实时报警检查
        if (isAlertEvent(event.getEventType())) {
            alertService.sendAlert(event);
        }
    }

    // 审计日志完整性校验
    private String calculateChecksum(AuditLog log) {
        String data = String.join("|",
                log.getEventTime().toString(),
                log.getEventId(),
                log.getUsername(),
                log.getEventType(),
                log.getResource(),
                log.getResult()
        );
        return DigestUtils.sha256Hex(data + getSecretKey());
    }

    // 验证日志完整性
    public boolean verifyIntegrity(AuditLog log) {
        String expectedChecksum = calculateChecksum(log);
        return expectedChecksum.equals(log.getChecksum());
    }
```

```java
/**
 * 三级等保数据加密服务

@Service
public class DataEncryptionService {

    // 数据分类分级
    public enum DataSensitivity {
        PUBLIC,         // 公开数据
        INTERNAL,       // 内部数据
        CONFIDENTIAL,   // 机密数据（需加密存储）
        SECRET          // 绝密数据（需强加密存储）
    }

    // 敏感字段加密
    @Autowired
    private EncryptionKeyManager keyManager;

    // 加密敏感数据
    public String encryptSensitiveData(String data, DataSensitivity sensitivity) {
        if (sensitivity == DataSensitivity.PUBLIC ||
                sensitivity == DataSensitivity.INTERNAL) {
            return data; // 非敏感数据不加密
        }

        // 获取加密密钥
        String keyId = keyManager.getCurrentKeyId(sensitivity);
        SecretKey key = keyManager.getKey(keyId);

        // 加密
        byte[] iv = generateIV();
        byte[] encrypted = AES_GCM_Encrypt(data.getBytes(StandardCharsets.UTF_8),
                key, iv);

        // 返回格式：keyId|iv|encryptedData
        return keyId + "|" + Base64.encode(iv) + "|" + Base64.encode(encrypted);
    }

    // 解密敏感数据
    public String decryptSensitiveData(String encryptedData) {
        String[] parts = encryptedData.split("\\|");
        if (parts.length != 3) {
            return encryptedData; // 非加密数据
        }

        String keyId = parts[0];
        byte[] iv = Base64.decode(parts[1]);
        byte[] data = Base64.decode(parts[2]);

        SecretKey key = keyManager.getKey(keyId);
        byte[] decrypted = AES_GCM_Decrypt(data, key, iv);

        return new String(decrypted, StandardCharsets.UTF_8);
    }

    // 数据库字段加密（MyBatis拦截器）
    @Intercepts({
            @Signature(type = Executor.class, method = "update",
                    args = {MappedStatement.class, Object.class})
    })
    public class FieldEncryptionInterceptor implements Interceptor {

        @Override
        public Object intercept(Invocation invocation) throws Throwable {
            MappedStatement ms = (MappedStatement) invocation.getArgs()[0];
            Object parameter = invocation.getArgs()[1];

            // 加密敏感字段
            if (parameter != null) {
                encryptSensitiveFields(parameter);
            }

            return invocation.proceed();
        }

        private void encryptSensitiveFields(Object obj) {
            Field[] fields = obj.getClass().getDeclaredFields();
            for (Field field : fields) {
                SensitiveData annotation = field.getAnnotation(SensitiveData.class);
                if (annotation != null) {
                    field.setAccessible(true);
                    Object value = field.get(obj);
                    if (value instanceof String) {
                        String encrypted = encryptSensitiveData(
                                (String) value, annotation.sensitivity());
                        field.set(obj, encrypted);
                    }
                }
            }
        }
    }
}
```

三级等保身份鉴别的具体要求：

1. **身份标识唯一性** - 每个用户具有唯一标识，不可伪造

2. **鉴别信息复杂度** - 密码长度≥8位，包含大小写字母、数字、特殊字符

3. **双因子认证（核心要求）** - 必须采用两种或以上组合的鉴别技术：
    - 密码 + TOTP（推荐，如Google Authenticator）
    - 密码 + 短信验证码
    - 密码 + 硬件令牌
    - 密码 + 生物特征

4. **登录失败处理** - 配置登录失败锁定功能

5. **会话管理** - 会话空闲超时自动断开

6. **鉴别信息保护** - 加密存储和传输

**问题 3：什么是"三权分立"？三级等保为什么要求三权分立？**

**答：**

**三权分立定义：**

将系统管理权限分为三个独立角色，实现相互制约：

- **系统管理员**：负责系统配置、备份恢复、日常运维
- **安全管理员**：负责安全策略、权限分配、用户管理
- **审计管理员**：负责审计日志查看、安全分析、合规检查

**三权分立目的：**

1. **防止权限滥用** - 单一管理员无法完成所有敏感操作
2. **相互制约** - 任何操作都有其他角色监督
3. **审计追溯** - 所有操作可追溯，责任可定位
4. **降低风险** - 减少内部人员安全威胁

**实现要点：**

```java
// 同一用户不能同时拥有多个管理角色
if(currentRoles.contains("SYSTEM_ADMIN") &&newRole.

equals("AUDIT_ADMIN")){
        throw new

RoleConflictException("三权分立原则：不能兼任多个管理角色");
}
```

**问题 4：三级等保对安全审计有哪些具体要求？**

**答：**

三级等保安全审计的具体要求：

1. **审计范围**
    - 覆盖所有用户行为
    - 覆盖所有重要安全事件

2. **审计内容**
    - 事件日期、时间
    - 用户标识、客户端IP
    - 事件类型、操作对象
    - 操作结果、详细信息

3. **审计日志保护**
    - 防止篡改（完整性校验）
    - 防止删除（权限控制）
    - 异地备份

4. **日志保存期限**
    - **至少保存12个月**

5. **审计分析**
    - 定期分析审计日志
    - 发现异常行为及时报警

**问题 5：三级等保对数据安全有哪些要求？**

**答：**

三级等保数据安全要求：

1. **数据保密性**
    - 重要数据加密存储
    - 敏感字段单独加密
    - 密钥安全管理

2. **数据完整性**
    - 采用校验技术
    - 数字签名验证
    - 防篡改机制

3. **数据备份**
    - 本地备份
    - **异地备份（必须）**
    - 定期恢复演练

4. **数据恢复**
    - 具备数据恢复能力
    - 恢复流程文档化

5. **剩余信息保护**
    - 鉴别信息存储空间释放前完全清除
    - 敏感数据删除后不可恢复

**问题 6：如何实现双因子认证？**

**答：**

双因子认证实现方案：

**方案一：密码 + TOTP（推荐）**

```java
// 1. 生成TOTP密钥
String secret = GoogleAuthenticator.generateSecret();
// 生成二维码供用户扫描
String qrCodeUrl = "otpauth://totp/MyApp:" + username + "?secret=" + secret;

// 2. 验证TOTP
public boolean verifyTOTP(String secret, int code) {
    return GoogleAuthenticator.verify(secret, code);
}
```

**方案二：密码 + 短信验证码**

```java

equals(redis.get("sms:"+phone))){
        redis.

del("sms:"+phone);  // 一次性使用

            }
```

**登录流程：**

1. 用户输入用户名、密码
2. 验证密码正确后，要求输入第二因子
3. 验证第二因子（TOTP/短信）
4. 两个因子都验证通过，登录成功

- 收集系统资料
- 进行系统调研
- 确定测评范围

**第二阶段：方案编制（1周）**

- 编制测评方案
- 确定测评对象
- 制定测评计划

**问题 9：什么是异地备份？三级等保为什么要求异地备份？**

**答：**

**异地备份定义：**

将数据备份存储在地理上分离的地点，与主系统不在同一城市或区域。

**三级等保要求原因：**

1. **灾难恢复** - 防止自然灾害（火灾、地震、洪水）导致数据完全丢失
2. **业务连续性** - 确保极端情况下能快速恢复业务
3. **数据安全** - 降低单点故障风险

**实现方式：**

```
主数据中心（北京）
       │
       │ 实时/定时同步
       ▼
异地备份中心（上海）
```

**技术方案：**

- 数据库主从复制
- 存储阵列远程复制
- 云存储异地冗余
- 定期磁带异地运输

**问题 10：三级等保常见问题及整改方案？**

**答：**

|    问题    | 整改方案                      |
|:--------:|:--------------------------|

| 数据未加密存储  | 实施数据加密，使用字段级加密或透明加密       |
|  异地备份缺失  | 建立异地备份中心，配置定时同步           |
|  入侵检测缺失  | 部署IDS/IPS设备，配置告警策略        |
| 应急预案不完善  | 制定详细应急预案，定期开展演练           |
