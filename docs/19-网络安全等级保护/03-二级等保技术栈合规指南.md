## 一、Linux服务器安全加固

### 1.1 安全要求对照

|  等保要求  | 具体措施         | 配置文件                           |
|:------:|:-------------|:-------------------------------|
|  身份鉴别  | 账户唯一标识、密码复杂度 | `/etc/security/pwquality.conf` |
| 登录失败处理 | 账户锁定机制       | `/etc/pam.d/password-auth`     |
|  访问控制  | 最小权限原则       | 用户组管理                          |
|  安全审计  | 系统日志记录       | `/etc/audit/auditd.conf`       |

### 1.2 SSH安全配置

```bash
# /etc/ssh/sshd_config

# 基本安全配置
Port 22                          # 建议修改为非标准端口
PermitRootLogin no               # 禁止root远程登录
PasswordAuthentication no        # 禁用密码登录
PubkeyAuthentication yes         # 启用密钥登录

# 认证安全配置
MaxAuthTries 3                   # 最大认证尝试次数
MaxSessions 5                    # 最大会话数
LoginGraceTime 60                # 登录超时时间（秒）

# 会话安全配置
ClientAliveInterval 300          # 会话超时检测间隔（秒）
ClientAliveCountMax 2            # 超时检测次数
TCPKeepAlive yes                 # TCP保活

# 其他安全配置
X11Forwarding no                 # 禁用X11转发
AllowTcpForwarding no            # 禁用TCP转发
GatewayPorts no                  # 禁用网关端口
AllowUsers appuser@192.168.1.0/24 # 限制允许登录的用户和IP

# 重启SSH服务
systemctl restart sshd
```

### 1.3 密码策略配置

```bash
# /etc/security/pwquality.conf

# 密码复杂度要求
minlen = 8                       # 最小长度8位
minclass = 4                     # 至少包含4类字符
dcredit = -1                     # 至少1个数字
ucredit = -1                     # 至少1个大写字母
lcredit = -1                     # 至少1个小写字母
ocredit = -1                     # 至少1个特殊字符
maxrepeat = 2                    # 最多连续重复2次
maxclassrepeat = 4               # 同类字符最多连续4次
usercheck = 1                    # 检查是否包含用户名
enforcing = 1                    # 强制执行

# /etc/login.defs 密码有效期配置
PASS_MAX_DAYS 90                 # 密码最大有效期90天
PASS_MIN_DAYS 0                  # 密码最小修改间隔
PASS_MIN_LEN 8                   # 密码最小长度
PASS_WARN_AGE 7                  # 密码过期前警告天数
```

### 1.4 登录失败锁定

```bash
# /etc/pam.d/password-auth 或 /etc/pam.d/sshd

# 在 auth 部分添加（第一行）
auth required pam_faillock.so preauth deny=5 unlock_time=1800 even_deny_root root_unlock_time=1800

# 在 auth 部分添加（password 部分之前）
auth [default=die] pam_faillock.so authfail deny=5 unlock_time=1800 even_deny_root root_unlock_time=1800

# 在 account 部分添加
account required pam_faillock.so

# 参数说明：
# deny=5           连续失败5次后锁定
# unlock_time=1800 锁定时间1800秒（30分钟）
# even_deny_root   对root用户同样生效
```

### 1.5 系统审计配置

```bash

-w /etc/shadow -p wa -k identity
-w /etc/sudoers -p wa -k sudoers
-w /etc/ssh/sshd_config -p wa -k ssh-config

-w /var/log/lastlog -p wa -k logins

# 监控系统调用

-a always,exit -F arch=b64 -S creat,open,openat,truncate -F auid>=1000 -F auid!=4294967295 -k file_create

# 启动审计服务
systemctl enable auditd
systemctl start auditd
```

### 2.2 账户安全配置

```sql
-- 1. 删除空账户和测试数据库
DELETE
FROM mysql.user
WHERE User = '';
DELETE
FROM mysql.user
WHERE User = 'test'
   OR User = 'test_%';
DROP DATABASE IF EXISTS test;

-- 2. 修改root账户（限制登录IP）
UPDATE mysql.user
SET Host='127.0.0.1'
WHERE User = 'root'
  AND Host = '%';

-- 3. 创建应用账户（最小权限原则）
CREATE USER 'app_user'@'192.168.1.%'
    IDENTIFIED BY 'Complex@Pass123';
GRANT SELECT, INSERT, UPDATE, DELETE ON app_db.*
    TO 'app_user'@'192.168.1.%';

-- 4. 创建只读账户
CREATE USER 'read_user'@'192.168.1.%'
    IDENTIFIED BY 'Complex@Pass123';
GRANT SELECT ON app_db.*
    TO 'read_user'@'192.168.1.%';

-- 5. 创建备份账户
CREATE USER 'backup_user'@'localhost'
    IDENTIFIED BY 'Complex@Pass123';
GRANT SELECT, LOCK TABLES, SHOW VIEW, RELOAD ON *.*
    TO 'backup_user'@'localhost';

-- 6. 刷新权限
FLUSH PRIVILEGES;
```

### 2.5 备份策略

```bash
#!/bin/bash
# MySQL备份脚本 /opt/scripts/mysql_backup.sh

BACKUP_DIR="/backup/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
MYSQL_USER="backup_user"
MYSQL_PASS="Complex@Pass123"
DATABASES="app_db"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 全量备份
mysqldump -u$MYSQL_USER -p$MYSQL_PASS \
    --single-transaction \
    --routines \
    --triggers \
    --events \
    --flush-logs \
    --master-data=2 \
    --databases $DATABASES \
    | gzip > $BACKUP_DIR/full_backup_$DATE.sql.gz

# 保留最近7天的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

# 异地备份（可选）
# scp $BACKUP_DIR/full_backup_$DATE.sql.gz backup-server:/backup/mysql/
```

```bash
# 定时任务 crontab -e
# 每天凌晨2点执行备份
0 2 * * * /opt/scripts/mysql_backup.sh >> /var/log/mysql_backup.log 2>&1
```

### 3.2 账户安全配置

```sql
-- 1. 修改postgres默认密码
ALTER USER postgres WITH PASSWORD 'Complex@Pass123';

-- 2. 创建应用账户
CREATE USER app_user WITH PASSWORD 'Complex@Pass123';
GRANT CONNECT ON DATABASE app_db TO app_user;
GRANT USAGE ON SCHEMA public TO app_user;
GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO app_user;
ALTER
DEFAULT PRIVILEGES IN SCHEMA public
GRANT SELECT, INSERT, UPDATE, DELETE ON TABLES TO app_user;

-- 3. 创建只读账户
CREATE USER read_user WITH PASSWORD 'Complex@Pass123';
GRANT CONNECT ON DATABASE app_db TO read_user;
GRANT USAGE ON SCHEMA public TO read_user;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO read_user;

-- 4. 创建复制账户
CREATE USER replicator WITH REPLICATION PASSWORD 'Complex@Pass123';
```

### 3.3 配置文件安全

```ini
# postgresql.conf

# 连接安全
listen_addresses = '192.168.1.100'    # 绑定内网IP
port = 5432
max_connections = 200

# 密码加密方式

ssl_cert_file = '/etc/postgresql/server.crt'
ssl_key_file = '/etc/postgresql/server.key'
ssl_ca_file = '/etc/postgresql/ca.crt'
ssl_min_protocol_version = 'TLSv1.2'

# 日志配置
logging_collector = on
log_directory = 'pg_log'
log_filename = 'postgresql-%Y-%m-%d_%H%M%S.log'
log_rotation_age = 1d
log_rotation_size = 100MB

# 审计日志
log_connections = on                  # 记录连接
log_disconnections = on               # 记录断开
log_statement = 'ddl'                 # 记录DDL语句
log_checkpoints = on
log_lock_waits = on

# pgAudit扩展（需安装）
shared_preload_libraries = 'pgaudit'
pgaudit.log = 'write, ddl'            # 记录写入和DDL操作
pgaudit.log_catalog = on
pgaudit.log_client = on
```

## 四、MongoDB数据库安全

### 4.1 安全要求对照

| 等保要求 | 具体措施    | 配置方式                   |
|:----:|:--------|:-----------------------|

| 传输加密 | TLS/SSL | net.tls配置              |
| 安全审计 | 审计日志    | auditLog配置             |

### 4.2 配置文件安全

```yaml
# mongod.conf

# 存储配置
storage:
  dbPath: /var/lib/mongodb
  journal:
    enabled: true

# 网络配置
net:
  port: 27017
  bindIp: 192.168.1.100,127.0.0.1    # 绑定内网IP
  tls:
    mode: requireTLS                  # 强制TLS
    certificateKeyFile: /etc/mongodb/server.pem
    CAFile: /etc/mongodb/ca.pem
    allowConnectionsWithoutCertificates: false

# 安全配置
security:
  authorization: enabled              # 启用身份认证
  keyFile: /etc/mongodb/keyfile       # 集群间认证（副本集使用）

# 审计日志
auditLog:
  destination: file
  format: JSON
  path: /var/log/mongodb/audit.log
  filter: '{ atype: { $in: [ "authenticate", "createUser", "dropUser", "grantRolesToUser" ] } }'

# 日志配置
systemLog:
  destination: file
  logAppend: true
  path: /var/log/mongodb/mongod.log
  verbosity: 1
```

## 五、Redis缓存安全

### 5.1 安全要求对照

|  等保要求  | 具体措施      | 配置方式           |
|:------:|:----------|:---------------|
|  身份鉴别  | 访问密码      | requirepass    |
|  访问控制  | 绑定内网IP    | bind           |
| 危险命令控制 | 禁用/重命名    | rename-command |
|  传输加密  | TLS（6.0+） | tls-port       |

### 5.2 配置文件安全

```conf
# redis.conf

# 网络安全
bind 127.0.0.1 192.168.1.100         # 绑定内网IP
protected-mode yes                    # 保护模式
port 0                                # 禁用非TLS端口

# TLS配置（Redis 6.0+）
tls-port 6379
tls-cert-file /etc/redis/redis.crt
tls-key-file /etc/redis/redis.key
tls-ca-cert-file /etc/redis/ca.crt
tls-auth-clients optional
tls-protocols "TLSv1.2 TLSv1.3"

# 访问控制
requirepass "Complex@Redis123"        # 访问密码

# 禁用危险命令
rename-command KEYS ""
rename-command FLUSHALL ""
rename-command FLUSHDB ""
rename-command CONFIG ""
rename-command DEBUG ""
rename-command SHUTDOWN ""
rename-command BGREWRITEAOF ""
rename-command BGSAVE ""

# 慢查询日志（审计）
slowlog-log-slower-than 10000         # 慢查询阈值（微秒）
slowlog-max-len 128                   # 慢查询日志长度

# 持久化配置
save 900 1
save 300 10
save 60 10000
appendonly yes
appendfsync everysec

# 日志配置
loglevel notice
logfile /var/log/redis/redis.log
```

### 6.2 账户权限配置

```bash
# 删除guest默认账户
rabbitmqctl delete_user guest

# 创建管理员账户
rabbitmqctl add_user admin Complex@Pass123
rabbitmqctl set_user_tags admin administrator
rabbitmqctl set_permissions -p / admin ".*" ".*" ".*"

# 创建应用专用vhost
rabbitmqctl add_vhost /app_vhost

# 创建应用账户
```

```java
public class PasswordValidator {

    // 密码正则：至少8位，包含大小写字母、数字、特殊字符
    private static final String PASSWORD_PATTERN =
            "^(?=.*[a-z])(?=.*[A-Z])(?=.*\\d)(?=.*[@$!%*?&])[A-Za-z\\d@$!%*?&]{8,}$";

    private static final Pattern pattern = Pattern.compile(PASSWORD_PATTERN);

    /**
     * 校验密码复杂度
     */
    public static boolean validate(String password) {
        if (password == null || password.isEmpty()) {
            return false;
        }
        return pattern.matcher(password).matches();
    }

    /**
     * 获取密码强度描述
     */
    public static String getStrengthDescription(String password) {
        List<String> missing = new ArrayList<>();

        if (password.length() < 8) {
            missing.add("长度不足8位");
        }
        if (!password.matches(".*[a-z].*")) {
            missing.add("缺少小写字母");
        }
        if (!password.matches(".*[A-Z].*")) {
            missing.add("缺少大写字母");
        }
        if (!password.matches(".*\\d.*")) {
            missing.add("缺少数字");
        }
        if (!password.matches(".*[@$!%*?&].*")) {
            missing.add("缺少特殊字符");
        }

        return missing.isEmpty() ? "密码强度符合要求" : "缺少：" + String.join("、", missing);
    }
```

### 9.2 登录失败锁定

```java
/**
 * 登录失败处理服务
 */
@Service
@Slf4j
public class LoginFailureHandler {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    @Autowired
    private UserService userService;

    // 配置参数
    private static final int MAX_ATTEMPTS = 5;          // 最大尝试次数
    private static final long LOCK_DURATION = 30;       // 锁定时间（分钟）
    private static final long ATTEMPT_WINDOW = 30;      // 尝试窗口（分钟）

    /**
     * 处理登录失败
     */
    public void handleFailure(String username, String clientIp) {
        String attemptKey = "login:attempt:" + username;
        String ipAttemptKey = "login:ip:" + clientIp;

        // 增加失败计数
        Long attempts = redisTemplate.opsForValue().increment(attemptKey);
        Long ipAttempts = redisTemplate.opsForValue().increment(ipAttemptKey);

        // 设置过期时间
        if (attempts == 1) {
            redisTemplate.expire(attemptKey, ATTEMPT_WINDOW, TimeUnit.MINUTES);
        }
        if (ipAttempts == 1) {
            redisTemplate.expire(ipAttemptKey, ATTEMPT_WINDOW, TimeUnit.MINUTES);
        }

        // 记录审计日志
        log.warn("登录失败 - 用户: {}, IP: {}, 尝试次数: {}", username, clientIp, attempts);

        // 达到阈值，锁定账户
        if (attempts >= MAX_ATTEMPTS) {
            lockAccount(username, clientIp);
            throw new AccountLockedException("账户已锁定，请" + LOCK_DURATION + "分钟后重试");
        }

        // IP封禁（防止暴力破解）
        if (ipAttempts >= MAX_ATTEMPTS * 3) {
            blockIp(clientIp);
            throw new IpBlockedException("IP已被临时封禁");
        }
    }

    /**
     * 锁定账户
     */
    private void lockAccount(String username, String ip) {
        String lockKey = "login:locked:" + username;
        redisTemplate.opsForValue().set(lockKey, ip, LOCK_DURATION, TimeUnit.MINUTES);

        // 更新数据库状态
        userService.lockAccount(username);

        // 发送告警
        alertService.sendAlert("账户锁定告警",
                String.format("用户 %s 从 IP %s 连续登录失败，账户已锁定", username, ip));
    }

    /**
     * 检查账户是否锁定
     */
    public boolean isAccountLocked(String username) {
        String lockKey = "login:locked:" + username;
        return Boolean.TRUE.equals(redisTemplate.hasKey(lockKey));
    }

    /**
     * 登录成功，清除失败记录
     */
    public void handleSuccess(String username) {
        String attemptKey = "login:attempt:" + username;
        redisTemplate.delete(attemptKey);
    }
}
```

```java
/**
 * 审计日志注解
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface AuditLog {
    String operation();                              // 操作名称

    AuditEventType eventType() default AuditEventType.OPERATION;  // 事件类型

    AuditLevel level() default AuditLevel.INFO;     // 审计级别

    String resource() default "";                    // 资源名称
}

/**
 * 审计事件类型
 */
public enum AuditEventType {
    LOGIN_SUCCESS,       // 登录成功
    LOGIN_FAILURE,       // 登录失败
    LOGOUT,              // 登出
    OPERATION,           // 业务操作
    DATA_ACCESS,         // 数据访问
    DATA_MODIFY,         // 数据修改
    CONFIG_CHANGE,       // 配置变更
    PERMISSION_DENIED,   // 权限拒绝
    SECURITY_EVENT       // 安全事件
}

/**
 * 审计级别
 */
public enum AuditLevel {
    INFO,
    WARN,
    ERROR,
    CRITICAL
}

/**
 * 审计日志切面
 */
@Aspect
@Component
@Slf4j
public class AuditLogAspect {

    @Autowired
    private AuditLogService auditLogService;

    @Autowired
    private HttpServletRequest request;

    @Around("@annotation(auditLog)")
    public Object around(ProceedingJoinPoint pjp, AuditLog auditLog) throws Throwable {
        String username = getCurrentUsername();
        String clientIp = getClientIp();
        String sessionId = request.getSession().getId();
        String requestId = MDC.get("requestId");

        long startTime = System.currentTimeMillis();
        Object result = null;
        Throwable exception = null;

        try {
            result = pjp.proceed();
            return result;
        } catch (Throwable e) {
            exception = e;
            throw e;
        } finally {
            long duration = System.currentTimeMillis() - startTime;

            // 构建审计日志
            AuditEvent event = AuditEvent.builder()
                    .eventTime(LocalDateTime.now())
                    .username(username)
                    .eventType(auditLog.eventType())
                    .level(exception != null ? AuditLevel.ERROR : auditLog.level())
                    .operation(auditLog.operation())
                    .resource(auditLog.resource())
                    .result(exception != null ? "FAILURE" : "SUCCESS")
                    .detail(exception != null ? exception.getMessage() : null)
                    .clientIp(clientIp)
                    .sessionId(sessionId)
                    .requestId(requestId)
                    .duration(duration)
                    .requestUrl(request.getRequestURI())
                    .requestMethod(request.getMethod())
                    .build();

            // 异步保存审计日志
            auditLogService.saveAsync(event);
        }
    }

    private String getCurrentUsername() {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        return auth != null ? auth.getName() : "anonymous";
    }

    private String getClientIp() {
        String ip = request.getHeader("X-Forwarded-For");
        if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
            ip = request.getHeader("X-Real-IP");
        }
        if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
            ip = request.getRemoteAddr();
        }
        // 多级代理取第一个IP
        if (ip != null && ip.contains(",")) {
            ip = ip.split(",")[0].trim();
        }
        return ip;
    }
}

/**
 * 审计日志服务
 */
@Service
public class AuditLogService {

    @Autowired
    private AuditLogRepository repository;

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    @Async
    public void saveAsync(AuditEvent event) {
        // 保存到数据库
        AuditLogEntity entity = convertToEntity(event);
        repository.save(entity);

        // 发送到审计队列（可选）
        redisTemplate.convertAndSend("audit:log", event);

        // 实时告警检查
        if (event.getLevel() == AuditLevel.CRITICAL) {
            alertService.sendAlert("安全事件告警", event.toString());
        }
    }
}
```

```java
                // 添加JWT过滤器
                .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class)

                // 安全头配置
                .headers()
                .frameOptions().sameOrigin()
                .xssProtection().block(true)
                .contentSecurityPolicy("default-src 'self'")
                .and()

                // CORS配置
                .cors()
                .configurationSource(corsConfigurationSource());

        return http.build();
    }

    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration configuration = new CorsConfiguration();
        configuration.setAllowedOrigins(Arrays.asList("https://example.com"));
        configuration.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE"));
        configuration.setAllowedHeaders(Arrays.asList("*"));
        configuration.setAllowCredentials(true);
        configuration.setMaxAge(3600L);

        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/api/**", configuration);
        return source;
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(12);
    }
}
```

```javascript
// utils/sanitize.js

import DOMPurify from 'dompurify'

/**
 * HTML内容净化，防止XSS攻击
 */
export function sanitizeHtml(html) {
    return DOMPurify.sanitize(html, {
        ALLOWED_TAGS: ['p', 'br', 'strong', 'em', 'u', 'a', 'img'],
        ALLOWED_ATTR: ['href', 'src', 'alt', 'title'],
        ALLOW_DATA_ATTR: false
    })
}

/**
 * 转义HTML特殊字符
 */
export function escapeHtml(str) {
    const escapeMap = {
        '&': '&amp;',
        '<': '&lt;',
        '>': '&gt;',
        '"': '&quot;',
        "'": '&#x27;',
        '/': '&#x2F;'
    }
    return String(str).replace(/[&<>"'/]/g, char => escapeMap[char])
}

/**
 * URL净化，防止javascript:协议攻击
 */
export function sanitizeUrl(url) {
    const safeProtocols = ['http://', 'https://', 'mailto:', 'tel:']
    const trimmedUrl = url ? url.trim().toLowerCase() : ''

    if (safeProtocols.some(protocol => trimmedUrl.startsWith(protocol))) {
        return url
    }

    if (trimmedUrl.startsWith('/') || trimmedUrl.startsWith('#')) {
        return url
    }

    return ''
}
```

```javascript
/**
 * 安全存储工具
 * 注意：敏感信息不应存储在前端

// 使用sessionStorage存储临时数据
export const secureSession = {
    set(key, value) {
        sessionStorage.setItem(key, JSON.stringify(value))
    },
    get(key) {
        const value = sessionStorage.getItem(key)
        return value ? JSON.parse(value) : null
    },
    remove(key) {
        sessionStorage.removeItem(key)
    },
    clear() {
        sessionStorage.clear()
    }
}

// 加密存储（使用AES）
import CryptoJS from 'crypto-js'

const SECRET_KEY = process.env.VUE_APP_STORAGE_KEY

export const encryptedStorage = {
    encrypt(data) {
        return CryptoJS.AES.encrypt(JSON.stringify(data), SECRET_KEY).toString()
    },
    decrypt(encrypted) {
        const bytes = CryptoJS.AES.decrypt(encrypted, SECRET_KEY)
        return JSON.parse(bytes.toString(CryptoJS.enc.Utf8))
    },
    set(key, value) {
        localStorage.setItem(key, this.encrypt(value))
    },
    get(key) {
        const encrypted = localStorage.getItem(key)
        return encrypted ? this.decrypt(encrypted) : null
    }
}

// 敏感数据脱敏
export function maskSensitiveData(data, type) {
    switch (type) {
        case 'phone':
            return data.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2')
        case 'idCard':
            return data.replace(/(\d{4})\d{10}(\d{4})/, '$1**********$2')
        case 'bankCard':
            return data.replace(/(\d{4})\d+(\d{4})/, '$1****$2')
        case 'email':
            return data.replace(/(.{2}).*@/, '$1***@')
        default:
            return data
    }
}
```

### 10.4 HTTP请求安全

```javascript
// utils/request.js

import axios from 'axios'
import {Message} from 'element-ui'
import router from '@/router'

// 创建axios实例
const service = axios.create({
    baseURL: process.env.VUE_APP_API_URL,
    timeout: 30000,
    withCredentials: true,
    headers: {
        'Content-Type': 'application/json',
        'X-Requested-With': 'XMLHttpRequest'
    }
})

// 请求拦截器
service.interceptors.request.use(
    config => {
        // 添加CSRF Token
        const csrfToken = document.querySelector('meta[name="csrf-token"]')
        if (csrfToken) {
            config.headers['X-CSRF-Token'] = csrfToken.getAttribute('content')
        }

        // 添加请求ID（审计追踪）
        config.headers['X-Request-Id'] = generateRequestId()

        return config
    },
    error => {
        return Promise.reject(error)
    }
)

// 响应拦截器
service.interceptors.response.use(
    response => {
        const res = response.data

        // 业务错误处理
        if (res.code !== 0) {
            Message.error(res.message || '请求失败')
            return Promise.reject(new Error(res.message || '请求失败'))
        }

        return res
    },
    error => {
        // HTTP错误处理
        if (error.response) {
            switch (error.response.status) {
                case 401:
                    // 未授权，跳转登录
                    router.push('/login')
                    break
                case 403:
                    Message.error('没有权限访问')
                    break
                case 404:
                    Message.error('请求资源不存在')
                    break
                case 500:
                    Message.error('服务器内部错误')
                    break
                default:
                    Message.error(error.message)
            }
        } else {
            Message.error('网络连接失败')
        }

        return Promise.reject(error)
    }
)

// 生成请求ID
function generateRequestId() {
    return 'req_' + Date.now() + '_' + Math.random().toString(36).substr(2, 9)
}

export default service
```

## 十二、高频面试题

**问题 1：二级等保对Linux服务器有哪些安全要求？**

**答：**

二级等保对Linux服务器的主要安全要求：

1. **身份鉴别**
    - 用户具有唯一标识
    - 密码长度≥8位，包含大小写字母、数字、特殊字符
    - 密码定期更换（建议90天）
    - 登录失败锁定（如5次失败锁定30分钟）

2. **访问控制**
    - 禁用root远程登录
    - 实施最小权限原则
    - 配置sudo权限分离

3. **安全审计**
    - 开启系统审计日志（auditd）
    - 记录用户行为和系统事件
    - 日志保存≥6个月

4. **服务最小化**
    - 关闭不必要的服务和端口
    - 配置防火墙规则

---

**问题 2：如何配置MySQL满足二级等保要求？**

**答：**

MySQL二级等保配置要点：

1. **账户安全**
    - 删除空账户和测试数据库
    - 修改root默认密码和登录IP限制
    - 创建应用专用账户，授予最小权限

2. **密码策略**
    - 安装validate_password插件
    - 配置密码复杂度要求

3. **传输加密**
    - 配置SSL证书
    - 强制SSL连接

4. **安全审计**
    - 开启慢查询日志
    - 部署审计插件（企业版或MariaDB审计插件）

5. **数据备份**
    - 定期全量备份
    - 异地备份存储

**问题 3：SpringBoot应用如何实现安全审计？**

**答：**

SpringBoot安全审计实现方案：

1. **审计日志记录**
    - 使用AOP切面自动记录操作日志
    - 记录内容：时间、用户、操作类型、结果、IP地址

2. **审计事件类型**
    - 登录成功/失败
    - 数据访问/修改
    - 权限变更
    - 配置变更

3. **日志存储**
    - 数据库持久化
    - 日志文件归档
    - 保存期限≥6个月

4. **日志保护**
    - 访问权限控制
    - 完整性校验
    - 防篡改机制

**问题 4：Redis如何配置满足二级等保要求？**

**答：**

Redis二级等保配置要点：

1. **访问控制**
    - 绑定内网IP（bind 127.0.0.1 内网IP）
    - 设置访问密码（requirepass）
    - 开启保护模式

2. **危险命令控制**
    - 禁用KEYS、FLUSHALL、FLUSHDB等危险命令
    - 使用rename-command重命名或禁用

3. **传输加密**

    - 禁用非TLS端口

4. **审计日志**
    - 开启慢查询日志
