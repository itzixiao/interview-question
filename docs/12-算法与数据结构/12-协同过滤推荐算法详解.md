## 一、算法概述

### 1.1 什么是协同过滤

协同过滤（Collaborative Filtering）是**推荐系统中最经典、应用最广泛的算法**。

**核心思想**：利用用户群体的行为数据，发现用户或物品之间的相似性，进行推荐。

```
协同过滤的核心假设：
┌─────────────────────────────────────────┐
│ 如果用户A和用户B在过去有相似的喜好       │
│ 那么他们在未来也很可能有相似的喜好       │
└─────────────────────────────────────────┘

示例：
- 用户A喜欢：{电影1, 电影2, 电影3}
- 用户B喜欢：{电影1, 电影2, 电影4}
- A和B相似度高
- 推荐：向A推荐电影4，向B推荐电影3
```

### 1.2 主要类型

| 类型 | 简称 | 核心思想 | 适用场景 |
|------|------|----------|----------|
| **基于用户的协同过滤** | UserCF | 找到相似用户，推荐他们喜欢的物品 | 用户少、物品多（如新闻推荐） |
| **基于物品的协同过滤** | ItemCF | 推荐与用户喜欢物品相似的物品 | 物品少、用户多（如电商推荐） |
| **基于模型的协同过滤** | ModelCF | 使用矩阵分解等机器学习方法 | 大规模数据 |

**UserCF vs ItemCF对比**：
```
UserCF：
- 计算用户相似度
- 找到相似用户群体
- 推荐相似用户喜欢的物品
- 适合：用户相对稳定，物品更新快

ItemCF：
- 计算物品相似度
- 找到与用户喜欢物品相似的其他物品
- 推荐相似物品
- 适合：物品相对稳定，用户量大
```

## 二、相似度计算

### 2.1 余弦相似度

```java
public double cosineSimilarity(Map<String, Double> ratingsA, 
                                Map<String, Double> ratingsB) {
    double dotProduct = 0.0;
    double normA = 0.0;
    double normB = 0.0;
    
    Set<String> commonItems = new HashSet<>(ratingsA.keySet());
    commonItems.retainAll(ratingsB.keySet());
    
    for (String item : commonItems) {
        dotProduct += ratingsA.get(item) * ratingsB.get(item);
    }
    
    for (double rating : ratingsA.values()) normA += rating * rating;
    for (double rating : ratingsB.values()) normB += rating * rating;
    
    return dotProduct / (Math.sqrt(normA) * Math.sqrt(normB));
}
```

**公式**：cos(A,B) = (A·B) / (||A|| × ||B||)

**范围**：[-1, 1]，1表示完全相似

### 2.2 皮尔逊相关系数

```java
public double pearsonCorrelation(Map<String, Double> ratingsA,
                                  Map<String, Double> ratingsB) {
    Set<String> commonItems = new HashSet<>(ratingsA.keySet());
    commonItems.retainAll(ratingsB.keySet());
    
    int n = commonItems.size();
    double sumA = 0.0, sumB = 0.0;
    for (String item : commonItems) {
        sumA += ratingsA.get(item);
        sumB += ratingsB.get(item);
    }
    
    double avgA = sumA / n;
    double avgB = sumB / n;
    
    double numerator = 0.0;
    double sumSqA = 0.0;
    double sumSqB = 0.0;
    
    for (String item : commonItems) {
        double diffA = ratingsA.get(item) - avgA;
        double diffB = ratingsB.get(item) - avgB;
        numerator += diffA * diffB;
        sumSqA += diffA * diffA;
        sumSqB += diffB * diffB;
    }
    
    return numerator / Math.sqrt(sumSqA * sumSqB);
}
```

**特点**：考虑了评分的平均值，减少用户评分偏置的影响

### 2.3 杰卡德相似度

```java
public double jaccardSimilarity(Set<String> itemsA, Set<String> itemsB) {
    Set<String> intersection = new HashSet<>(itemsA);
    intersection.retainAll(itemsB);
    
    Set<String> union = new HashSet<>(itemsA);
    union.addAll(itemsB);
    
    return (double) intersection.size() / union.size();
}
```

**公式**：J(A,B) = |A∩B| / |A∪B|

**适用**：仅考虑是否交互（0/1），不考虑具体评分

## 三、基于用户的协同过滤（UserCF）

### 3.1 算法步骤

```
1. 计算目标用户与其他用户的相似度
2. 选择最相似的K个邻居
3. 预测目标用户对未评分物品的评分
4. 返回评分最高的N个物品
```

### 3.2 代码实现

```java
public List<Map.Entry<String, Double>> userBasedRecommend(
        String targetUser, int k, int n) {
    
    Map<String, Double> targetRatings = userItemRatings.get(targetUser);
    
    // 1. 计算相似度
    Map<String, Double> similarities = new HashMap<>();
    for (String otherUser : userItemRatings.keySet()) {
        if (!otherUser.equals(targetUser)) {
            double sim = pearsonCorrelation(targetRatings, 
                userItemRatings.get(otherUser));
            if (sim > 0) similarities.put(otherUser, sim);
        }
    }
    
    // 2. 选择K个邻居
    List<Map.Entry<String, Double>> neighbors = similarities.entrySet().stream()
        .sorted(Map.Entry.<String, Double>comparingByValue().reversed())
        .limit(k)
        .collect(Collectors.toList());
    
    // 3. 预测评分
    Map<String, Double> predictions = new HashMap<>();
    Map<String, Double> simSums = new HashMap<>();
    
    for (Map.Entry<String, Double> neighbor : neighbors) {
        String neighborId = neighbor.getKey();
        double similarity = neighbor.getValue();
        
        for (Map.Entry<String, Double> itemRating : 
                userItemRatings.get(neighborId).entrySet()) {
            String item = itemRating.getKey();
            if (!targetRatings.containsKey(item)) {
                predictions.merge(item, similarity * itemRating.getValue(), 
                    Double::sum);
                simSums.merge(item, similarity, Double::sum);
            }
        }
    }
    
    // 4. 归一化并排序
    Map<String, Double> finalPredictions = new HashMap<>();
    for (String item : predictions.keySet()) {
        finalPredictions.put(item, predictions.get(item) / simSums.get(item));
    }
    
    return finalPredictions.entrySet().stream()
        .sorted(Map.Entry.<String, Double>comparingByValue().reversed())
        .limit(n)
        .collect(Collectors.toList());
}
```

## 四、基于物品的协同过滤（ItemCF）

### 4.1 算法步骤

```
1. 构建物品-用户倒排表
2. 计算物品相似度矩阵
3. 根据用户历史评分和物品相似度预测评分
4. 返回评分最高的N个物品
```

### 4.2 代码实现

```java
public List<Map.Entry<String, Double>> itemBasedRecommend(String targetUser, int n) {
    Map<String, Double> targetRatings = userItemRatings.get(targetUser);
    
    // 1. 构建物品-用户倒排表
    Map<String, Map<String, Double>> itemUserRatings = new HashMap<>();
    for (Map.Entry<String, Map<String, Double>> entry : userItemRatings.entrySet()) {
        String user = entry.getKey();
        for (Map.Entry<String, Double> rating : entry.getValue().entrySet()) {
            itemUserRatings.computeIfAbsent(rating.getKey(), k -> new HashMap<>())
                           .put(user, rating.getValue());
        }
    }
    
    // 2. 计算物品相似度
    Map<String, Map<String, Double>> itemSimilarities = new HashMap<>();
    for (String item1 : itemUserRatings.keySet()) {
        for (String item2 : itemUserRatings.keySet()) {
            if (!item1.equals(item2)) {
                double sim = cosineSimilarity(
                    itemUserRatings.get(item1),
                    itemUserRatings.get(item2)
                );
                itemSimilarities.computeIfAbsent(item1, k -> new HashMap<>())
                               .put(item2, sim);
            }
        }
    }
    
    // 3. 预测评分
    Map<String, Double> predictions = new HashMap<>();
    Map<String, Double> simSums = new HashMap<>();
    
    for (String ratedItem : targetRatings.keySet()) {
        double rating = targetRatings.get(ratedItem);
        Map<String, Double> similarItems = itemSimilarities.get(ratedItem);
        
        if (similarItems != null) {
            for (Map.Entry<String, Double> entry : similarItems.entrySet()) {
                String candidateItem = entry.getKey();
                if (!targetRatings.containsKey(candidateItem)) {
                    double sim = entry.getValue();
                    predictions.merge(candidateItem, sim * rating, Double::sum);
                    simSums.merge(candidateItem, sim, Double::sum);
                }
            }
        }
    }
    
    // 4. 归一化并排序
    Map<String, Double> finalPredictions = new HashMap<>();
    for (String item : predictions.keySet()) {
        finalPredictions.put(item, predictions.get(item) / simSums.get(item));
    }
    
    return finalPredictions.entrySet().stream()
        .sorted(Map.Entry.<String, Double>comparingByValue().reversed())
        .limit(n)
        .collect(Collectors.toList());
}
```

## 五、UserCF vs ItemCF

| 对比维度 | UserCF | ItemCF |
|----------|--------|--------|
| 适用场景 | 用户少、物品多 | 用户多、物品少 |
| 推荐解释 | 和你相似的人喜欢 | 和你喜欢的物品相似 |
| 冷启动 | 新用户难处理 | 新物品难处理 |
| 计算量 | 用户相似度计算量大 | 物品相似度可离线计算 |
| 稳定性 | 用户兴趣易变 | 物品相似度更稳定 |
| 典型应用 | 新闻推荐 | 电商推荐 |

## 六、面试高频题详解

### 题1：协同过滤的优缺点？⭐⭐⭐⭐⭐

**优点**：
- ✅ **无需物品内容信息**：只需要用户行为数据（评分、点击、购买）
- ✅ **发现潜在兴趣**：可以推荐用户没接触过但可能感兴趣的物品
- ✅ **个性化程度高**：基于用户个人行为历史
- ✅ **自动化程度高**：不需要人工标注物品特征

**缺点**：
- ❌ **冷启动问题**：新用户/新物品没有历史数据，无法计算相似度
- ❌ **数据稀疏性**：用户-物品矩阵通常非常稀疏（99%以上为空）
- ❌ **可解释性差**：难以解释为什么推荐某个物品
- ❌ **热门物品偏见**：热门物品容易被过度推荐

### 题2：如何解决冷启动问题？⭐⭐⭐⭐⭐

| 场景 | 解决方案 | 具体实现 |
|------|----------|----------|
| **新用户** | 热门推荐 | 推荐全站最热门的物品 |
| **新用户** | 引导选择 | 让用户选择感兴趣的标签/分类 |
| **新用户** | 人口统计 | 根据年龄、性别、地域推荐 |
| **新物品** | 内容推荐 | 基于物品内容特征推荐 |
| **新物品** | 探索推荐 | 随机推荐给部分用户，收集反馈 |
| **系统冷启动** | 专家标注 | 人工标注初始物品特征 |

**混合策略**：
```
新用户前N次访问：
- 使用基于内容的推荐（如果有用户画像）
- 或热门推荐 + 探索推荐

收集足够数据后：
- 切换到协同过滤
- 或混合推荐（协同过滤 + 内容推荐）
```

### 题3：协同过滤 vs 基于内容的推荐？⭐⭐⭐⭐⭐

| 维度 | 协同过滤 | 基于内容 |
|------|----------|----------|
| **数据来源** | 用户行为数据 | 物品内容特征 |
| **核心思想** | 相似用户/物品有相似喜好 | 相似特征的物品是相似的 |
| **冷启动** | 有问题 | 新用户无问题 |
| **多样性** | 可以发现跨领域兴趣 | 容易局限在已有兴趣 |
| **可解释性** | 差（黑盒） | 好（特征匹配） |
| **新物品** | 难处理 | 容易处理 |
| **典型应用** | 电影、音乐推荐 | 新闻、文章推荐 |

### 题4：矩阵分解在推荐系统中的作用？⭐⭐⭐⭐

**核心思想**：
```
用户-物品评分矩阵R（m×n，m用户，n物品）通常非常稀疏

矩阵分解：R ≈ P × Q^T
- P：用户潜在因子矩阵（m×k）
- Q：物品潜在因子矩阵（n×k）
- k：潜在因子维度（通常k << min(m,n)）

每个用户/物品用k维向量表示，捕捉潜在特征
```

**作用**：
1. **降维**：从高维稀疏矩阵到低维稠密向量
2. **缓解稀疏性**：可以预测缺失值
3. **发现潜在因子**：自动学习用户和物品的隐式特征
4. **提高推荐精度**：比传统协同过滤更准确

**常用算法**：
- SVD（奇异值分解）
- ALS（交替最小二乘）
- NMF（非负矩阵分解）

### 题5：推荐系统的评价指标？⭐⭐⭐⭐

| 指标 | 说明 | 适用场景 |
|------|------|----------|
| **准确率/召回率** | 推荐物品中用户喜欢的比例 | 有明确评分数据 |
| **RMSE/MAE** | 预测评分与真实评分的误差 | 评分预测任务 |
| **覆盖率** | 推荐物品占总物品的比例 | 多样性评估 |
| **多样性** | 推荐列表中物品的差异性 | 避免过于集中 |
| **新颖性** | 推荐物品的流行度 | 探索用户潜在兴趣 |
| **点击率(CTR)** | 推荐被点击的比例 | 在线评估 |
| **转化率(CVR)** | 推荐被购买的比例 | 电商场景 |
