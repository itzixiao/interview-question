## 一、算法概述

### 1.1 什么是霍夫曼编码

霍夫曼编码（Huffman Coding）由David A. Huffman于1952年提出，是一种**经典的数据压缩算法**，也是贪心算法的典型应用。

**核心思想**：根据字符出现频率构建最优前缀编码树，高频字符用短编码，低频字符用长编码。

```
编码对比示例：
文本："AAABBC"

定长编码（ASCII）：
A: 01000001 (8位)
B: 01000010 (8位)
C: 01000011 (8位)
总长度：6 × 8 = 48位

霍夫曼编码：
A: 0   (1位)  ← 频率最高，编码最短
B: 10  (2位)
C: 11  (2位)
总长度：3×1 + 2×2 + 1×2 = 9位

压缩率：(48-9)/48 = 81.25%
```

### 1.2 前缀编码

**前缀编码（Prefix Code）**：任何字符的编码都不是其他字符编码的前缀，确保解码时不会产生歧义。

```
正确的前缀编码：
A: 0
B: 10
C: 11

解码 "010"：
0  → A
10 → B
结果："AB" ✓

错误（非前缀编码）：
A: 0
B: 01  (0是01的前缀，会产生歧义)

解码 "01"：
可能是 "AB" (0 + 1? 1没有定义)
也可能是 "B"  (01)
产生歧义！✗
```

**前缀编码的优势**：
- ✅ 无需分隔符，解码无歧义
- ✅ 可以用二叉树表示
- ✅ 贪心算法可构造最优解

## 二、算法步骤

### 2.1 构建流程

```
1. 统计字符频率
   例: "this is an example of a huffman tree"
   
   空格:7  a:4  e:4  f:3  h:2  i:2  m:2  n:2  s:2  t:2  
   l:1  o:1  p:1  r:1  u:1  x:1

2. 构建霍夫曼树
   - 每次选择频率最小的两个节点合并
   - 新节点频率为两者之和
   - 重复直到只剩一个节点

3. 生成编码
   - 左分支标记为0
   - 右分支标记为1
   - 从根到叶子的路径即为编码

4. 编码数据
   - 用生成的编码替换原始字符
```

### 2.2 构建过程图解

```
初始节点: [a:4] [e:4] [空格:7] [f:3] [h:2] [i:2] [m:2] [n:2] [s:2] [t:2] ...

第1步: 合并 [l:1] [o:1] → [*:2]
第2步: 合并 [p:1] [x:1] → [*:2]
第3步: 合并 [r:1] [u:1] → [*:2]
...
最终形成一棵完整的霍夫曼树
```

## 三、Java实现

### 3.1 霍夫曼树节点

```java
static class HuffmanNode implements Comparable<HuffmanNode> {
    char ch;           // 字符
    int freq;          // 频率
    HuffmanNode left;  // 左子节点
    HuffmanNode right; // 右子节点
    
    // 叶子节点
    HuffmanNode(char ch, int freq) {
        this.ch = ch;
        this.freq = freq;
    }
    
    // 内部节点
    HuffmanNode(int freq, HuffmanNode left, HuffmanNode right) {
        this.freq = freq;
        this.left = left;
        this.right = right;
    }
    
    boolean isLeaf() {
        return left == null && right == null;
    }
}
```

### 3.2 构建霍夫曼树

```java
public static HuffmanNode buildHuffmanTree(Map<Character, Integer> freqMap) {
    PriorityQueue<HuffmanNode> pq = new PriorityQueue<>();
    
    // 创建叶子节点
    for (Map.Entry<Character, Integer> entry : freqMap.entrySet()) {
        pq.offer(new HuffmanNode(entry.getKey(), entry.getValue()));
    }
    
    // 构建霍夫曼树
    while (pq.size() > 1) {
        HuffmanNode left = pq.poll();
        HuffmanNode right = pq.poll();
        HuffmanNode parent = new HuffmanNode(left.freq + right.freq, left, right);
        pq.offer(parent);
    }
    
    return pq.poll();
}
```

### 3.3 生成编码表

```java
public static Map<Character, String> buildEncodingMap(HuffmanNode root) {
    Map<Character, String> encodingMap = new HashMap<>();
    buildEncodingMapHelper(root, "", encodingMap);
    return encodingMap;
}

private static void buildEncodingMapHelper(HuffmanNode node, String code,
                                           Map<Character, String> encodingMap) {
    if (node == null) return;
    
    if (node.isLeaf()) {
        encodingMap.put(node.ch, code);
        return;
    }
    
    buildEncodingMapHelper(node.left, code + "0", encodingMap);
    buildEncodingMapHelper(node.right, code + "1", encodingMap);
}
```

### 3.4 编码与解码

```java
// 编码
public static String encode(String text, Map<Character, String> encodingMap) {
    StringBuilder encoded = new StringBuilder();
    for (char ch : text.toCharArray()) {
        encoded.append(encodingMap.get(ch));
    }
    return encoded.toString();
}

// 解码
public static String decode(String encodedData, HuffmanNode root) {
    StringBuilder decoded = new StringBuilder();
    HuffmanNode current = root;
    
    for (char bit : encodedData.toCharArray()) {
        current = (bit == '0') ? current.left : current.right;
        
        if (current.isLeaf()) {
            decoded.append(current.ch);
            current = root;  // 回到根节点
        }
    }
    
    return decoded.toString();
}
```

## 五、霍夫曼编码特点

### 5.1 优点

- **最优前缀编码**：在已知频率的情况下，达到最优压缩
- **无损压缩**：可完全恢复原始数据
- **自适应**：可根据数据动态调整编码

### 5.2 缺点

- **需要存储编码表**：小数据时可能得不偿失
- **需要两次扫描**：一次统计频率，一次编码
- **对随机数据效果差**：无法压缩已压缩数据

## 六、面试高频题详解

### 题1：霍夫曼编码为什么是最优前缀编码？⭐⭐⭐⭐

**最优性证明思路**：
```
1. 贪心选择性质：
   存在一棵最优前缀编码树，频率最低的两个字符是兄弟节点
   
2. 最优子结构：
   合并两个最低频率字符后，新问题仍是原问题的子问题

3. 数学归纳法：
   - 基础：1个字符，显然最优
   - 归纳：假设k-1个字符时最优，证明k个字符时也最优
```

**直观理解**：
```
高频字符（如空格，出现100次）：
  - 编码长度1位，总贡献：100 × 1 = 100位

低频字符（如z，出现1次）：
  - 编码长度5位，总贡献：1 × 5 = 5位

贪心策略确保高频字符路径最短，总编码长度最小
```

### 题2：霍夫曼编码的时间复杂度？⭐⭐⭐⭐

| 步骤 | 时间复杂度 | 说明 |
|------|------------|------|
| **统计频率** | O(n) | n为文本长度 |
| **构建优先队列** | O(k) | k为字符种类数 |
| **构建霍夫曼树** | O(k log k) | 每次合并操作O(log k)，共k-1次 |
| **生成编码表** | O(k) | 遍历树一次 |
| **编码数据** | O(n) | 每个字符替换为编码 |

**总时间复杂度**：O(n + k log k)

**空间复杂度**：O(k)（存储编码表和树）

### 题3：霍夫曼编码和ASCII的区别？⭐⭐⭐⭐

| 特性 | ASCII | 霍夫曼编码 |
|------|-------|------------|
| **编码长度** | 固定8位 | 变长（1-?位） |
| **压缩效果** | 无压缩 | 根据频率优化 |
| **需要额外信息** | 不需要 | 需要编码表 |
| **适用场景** | 通用文本 | 特定数据分布 |

**实际应用中的权衡**：
```
小数据（<100字节）：
  - 霍夫曼编码表可能比数据还大
  - 使用ASCII或其他固定编码

大数据（>1KB）：
  - 编码表开销相对较小
  - 霍夫曼编码优势明显
```

### 题4：实际应用场景？⭐⭐⭐⭐

| 应用 | 使用方式 | 说明 |
|------|----------|------|
| **ZIP压缩** | 与LZ77结合 | DEFLATE算法 |
| **GZIP** | 与LZ77结合 | HTTP压缩 |
| **JPEG** | 对DCT系数编码 | 图像压缩 |
| **MP3** | 对音频采样编码 | 音频压缩 |
| **PNG** | 对过滤后的数据编码 | 无损图像压缩 |

**DEFLATE算法（ZIP/GZIP核心）**：
```
1. LZ77：将重复字符串替换为(距离, 长度)引用
2. 霍夫曼编码：对LZ77输出进行熵编码

结合两种算法的优点：
- LZ77：消除重复模式
- 霍夫曼：根据频率优化编码
```
