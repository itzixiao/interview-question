## 一、哈希算法概述

### 1.1 什么是哈希算法

哈希算法（Hash Algorithm）是一种将**任意长度的数据**映射为**固定长度数据**的算法。核心特性：

| 特性 | 说明 | 重要性 |
|------|------|--------|
| **确定性** | 相同输入始终产生相同输出 | ⭐⭐⭐⭐⭐ |
| **高效性** | 计算速度快，O(1)时间复杂度 | ⭐⭐⭐⭐⭐ |
| **雪崩效应** | 微小输入变化导致输出巨大差异 | ⭐⭐⭐⭐⭐ |
| **抗碰撞性** | 难以找到两个不同输入产生相同输出 | ⭐⭐⭐⭐⭐ |

### 1.2 算法分类体系

```
哈希算法
├── 非加密哈希
│   ├── 简单哈希：取模哈希
│   ├── 字符串哈希：DJB2、BKDR、Java hashCode
│   └── 校验和：CRC32、Adler32
├── 加密哈希（单向）
│   ├── MD系列：MD5（已不安全）
│   ├── SHA系列：SHA-256、SHA-3
│   └── 密码哈希：bcrypt、scrypt、Argon2
└── 高级应用
    ├── 双重哈希：解决冲突
    ├── 布隆过滤器：多哈希组合
    └── 一致性哈希：分布式系统
```

### 1.3 常见应用场景

| 场景 | 说明 | 常用算法 | 性能要求 |
|------|------|----------|----------|
| **数据完整性校验** | 文件校验、下载验证 | CRC32、MD5 | 极快 |
| **密码存储** | 用户密码哈希存储 | bcrypt、SHA-256+盐 | 慢（安全优先） |
| **数据分片** | 分布式缓存数据分布 | 一致性哈希 | 快 |
| **数字签名** | 区块链、证书验证 | SHA-256 | 中等 |
| **哈希表索引** | HashMap、数据库索引 | 取模、DJB2 | 极快 |
| **去重检测** | 大数据去重 | 布隆过滤器 | 极快 |

## 二、哈希算法分类与实现

### 2.1 简单取模哈希

```java
public static int simpleModHash(int key, int tableSize) {
    return Math.abs(key % tableSize);
}
```

**原理**：`hash(key) = key % tableSize`

**适用场景**：简单的哈希表实现

### 2.2 字符串哈希算法

#### DJB2算法

```java
public static long djb2Hash(String str) {
    long hash = 5381;
    for (int i = 0; i < str.length(); i++) {
        hash = ((hash << 5) + hash) + str.charAt(i); // hash * 33 + c
    }
    return hash;
}
```

**特点**：计算快速，分布均匀

#### BKDR算法

```java
public static long bkdrHash(String str) {
    long hash = 0;
    long seed = 131; // 31, 131, 1313, 13131...
    for (int i = 0; i < str.length(); i++) {
        hash = hash * seed + str.charAt(i);
    }
    return hash & 0x7FFFFFFFFFFFFFFFL;
}
```

**特点**：经典字符串哈希，Java String的hashCode()基于此

#### Java String hashCode原理

```java
public static int javaStringHashCode(String str) {
    int hash = 0;
    for (int i = 0; i < str.length(); i++) {
        hash = 31 * hash + str.charAt(i);
    }
    return hash;
}
```

**选择31的原因**：
1. 是质数，减少哈希冲突
2. `31 * i = (i << 5) - i`，可被优化为位运算

### 2.3 校验和算法

#### CRC32

```java
public static long crc32Hash(String data) {
    CRC32 crc32 = new CRC32();
    crc32.update(data.getBytes(StandardCharsets.UTF_8));
    return crc32.getValue();
}
```

**原理**：基于多项式除法的校验算法
**特点**：计算快速，用于数据完整性校验

### 2.4 密码学哈希算法

#### MD5（已不推荐用于安全场景）

```java
public static String md5Hash(String input) {
    MessageDigest md = MessageDigest.getInstance("MD5");
    byte[] digest = md.digest(input.getBytes(StandardCharsets.UTF_8));
    // 转换为十六进制字符串
    StringBuilder sb = new StringBuilder();
    for (byte b : digest) {
        sb.append(String.format("%02x", b));
    }
    return sb.toString(); // 32位十六进制
}
```

**注意**：MD5已不再安全，存在碰撞攻击，仅用于教学或非安全场景

#### SHA-256（推荐）

```java
public static String sha256Hash(String input) {
    MessageDigest md = MessageDigest.getInstance("SHA-256");
    byte[] digest = md.digest(input.getBytes(StandardCharsets.UTF_8));
    StringBuilder sb = new StringBuilder();
    for (byte b : digest) {
        sb.append(String.format("%02x", b));
    }
    return sb.toString(); // 64位十六进制
}
```

**特点**：目前安全的哈希算法，广泛用于区块链、数字签名

### 2.5 高级哈希技术

#### 双重哈希（解决哈希冲突）

```java
public static int doubleHash(String key, int tableSize, int probe) {
    int hash1 = Math.abs(key.hashCode() % tableSize);
    int hash2 = 1 + Math.abs(key.hashCode() % (tableSize - 1));
    return (hash1 + probe * hash2) % tableSize;
}
```

**原理**：`hash1(key) + i * hash2(key)`

#### 布隆过滤器哈希

```java
public static int[] bloomFilterHashes(String key, int bitSize, int hashNum) {
    int[] positions = new int[hashNum];
    long hash1 = djb2Hash(key);
    long hash2 = bkdrHash(key);
    
    for (int i = 0; i < hashNum; i++) {
        positions[i] = (int) Math.abs((hash1 + i * hash2) % bitSize);
    }
    return positions;
}
```

**特点**：
- 空间效率高
- 存在误判可能（可能误报，不会漏报）
- 适合大数据集的存在性检测

#### 加盐哈希（密码存储）

```java
public static String saltedHash(String password, String salt) {
    MessageDigest md = MessageDigest.getInstance("SHA-256");
    String salted = password + salt;
    byte[] digest = md.digest(salted.getBytes(StandardCharsets.UTF_8));
    
    StringBuilder sb = new StringBuilder();
    for (byte b : digest) {
        sb.append(String.format("%02x", b));
    }
    return sb.toString();
}
```

**目的**：防止彩虹表攻击，相同密码产生不同哈希值

## 三、哈希算法对比

| 算法 | 输出长度 | 安全性 | 速度 | 适用场景 |
|------|----------|--------|------|----------|
| 简单取模 | 可变 | 低 | 极快 | 哈希表索引 |
| DJB2/BKDR | 64位 | 低 | 快 | 字符串哈希 |
| CRC32 | 32位 | 低 | 极快 | 数据校验 |
| MD5 | 128位 | 低（已破解） | 快 | 非安全场景 |
| SHA-256 | 256位 | 高 | 中等 | 安全场景 |
| bcrypt/scrypt | 可变 | 很高 | 慢 | 密码存储 |

## 五、面试高频题详解

### 题1：为什么HashMap的容量要是2的幂次？⭐⭐⭐⭐⭐

**答案要点**：
1. **位运算优化**：`hash % n` 可以替换为 `hash & (n-1)`，位运算比取模快10倍以上
2. **均匀分布**：2的幂次保证哈希值能均匀分布在数组的各个位置
3. **扩容便利**：2倍扩容时，元素要么在原来位置，要么在`原位置+旧容量`的位置

**代码示例**：
```java
// 取模运算
int index = hash % 16;  // 需要除法，较慢

// 位运算（当容量为2的幂次时）
int index = hash & (16 - 1);  // hash & 15，极快
```

### 题2：如何解决哈希冲突？⭐⭐⭐⭐⭐

| 方法 | 原理 | 优点 | 缺点 | 应用场景 |
|------|------|------|------|----------|
| **链地址法** | 冲突元素放入链表 | 实现简单、删除方便 | 链表过长时退化 | HashMap、数据库 |
| **开放寻址法** | 寻找下一个空槽 | 无指针开销、缓存友好 | 删除复杂、聚集问题 | ThreadLocalMap |
| **再哈希法** | 使用多个哈希函数 | 分布均匀 | 计算量大 | 高负载场景 |
| **建立公共溢出区** | 冲突元素放单独区域 | 主表查询快 | 需要额外空间 | 特殊场景 |

### 题3：MD5为什么不安全了？⭐⭐⭐⭐

**安全问题**：
1. **碰撞攻击**：2004年，王小云教授证明可以在1小时内找到MD5碰撞
2. **彩虹表攻击**：预计算的哈希表可快速破解弱密码
3. **长度扩展攻击**：可在不知道原始数据的情况下扩展哈希值

**替代方案**：
- 密码存储：bcrypt、scrypt、Argon2（专门设计，计算慢）
- 数据完整性：SHA-256、SHA-3
- 文件校验：CRC32（非安全场景）

### 题4：布隆过滤器的优缺点？⭐⭐⭐⭐

**优点**：
- ✅ 空间效率极高：存储1亿个元素仅需约100MB
- ✅ 查询速度极快：O(k)，k为哈希函数个数
- ✅ 不存在漏报：元素存在则一定返回可能存在

**缺点**：
- ❌ 存在误判率：可能将不存在的元素判断为存在
- ❌ 不支持删除：删除一个元素可能影响其他元素
- ❌ 无法获取元素：只能判断存在性

**误判率公式**：
```
f ≈ (1 - e^(-kn/m))^k

其中：
n = 元素数量
m = 位数组大小
k = 哈希函数个数
```

**实际应用**：
- Google Chrome恶意网址检测
- 数据库查询优化（避免查询不存在的key）
- 缓存穿透防护
