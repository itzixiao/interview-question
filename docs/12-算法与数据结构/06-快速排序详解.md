## 一、快速排序概述

### 1.1 算法简介

快速排序（Quick Sort）由东尼·霍尔（C. A. R. Hoare）于1962年提出，是**实际应用中最快的排序算法之一**，被广泛应用于标准库实现（Java Arrays.sort()、C++ std::sort()）。

**核心思想**：分治法（Divide and Conquer）
```
快速排序流程：
┌─────────────────────────────────────────┐
│ 1. 选择基准值（Pivot）                   │
│    └── 可选：首元素、尾元素、随机、中位数  │
├─────────────────────────────────────────┤
│ 2. 分区（Partition）                     │
│    └── 将数组分为 < pivot 和 > pivot 两部分│
├─────────────────────────────────────────┤
│ 3. 递归排序子数组                        │
│    └── 对左右两部分递归执行快排           │
└─────────────────────────────────────────┘
```

### 1.2 时间复杂度分析

| 情况 | 时间复杂度 | 发生条件 | 概率 |
|------|------------|----------|------|
| **最优** | O(n log n) | 每次分区平衡（中位数作为pivot） | 理论最优 |
| **平均** | O(n log n) | 随机数据 | 绝大多数情况 |
| **最坏** | O(n²) | 已排序/逆序数组，每次选最值作为pivot | 可优化避免 |

**数学推导**：
```
T(n) = 2T(n/2) + O(n)  // 每次分区O(n)，递归深度log n
     = O(n log n)
```

**空间复杂度**：O(log n) - 递归栈空间

**稳定性**：❌ 不稳定（相等元素可能交换位置）

### 1.3 算法演进路线

```
基础快排
    │
    ├── 优化1：随机化快排 ──→ 避免最坏情况
    │
    ├── 优化2：三数取中 ──→ 提升分区质量
    │
    ├── 优化3：双路快排 ──→ 处理大量重复元素
    │
    ├── 优化4：三路快排 ──→ 荷兰国旗问题
    │
    └── 优化5：非递归快排 ──→ 避免栈溢出

扩展应用
    │
    └── 快速选择算法 ──→ 求第K小元素（O(n)）
```

## 二、基础实现

### 2.1 标准快速排序

```java
public static void quickSort(int[] arr, int left, int right) {
    if (left < right) {
        int pivotIndex = partition(arr, left, right);
        quickSort(arr, left, pivotIndex - 1);
        quickSort(arr, pivotIndex + 1, right);
    }
}

// Lomuto分区方案
private static int partition(int[] arr, int left, int right) {
    int pivot = arr[right]; // 选择最右元素作为基准值
    int i = left - 1;       // i指向小于pivot的最后一个元素
    
    for (int j = left; j < right; j++) {
        if (arr[j] <= pivot) {
            i++;
            swap(arr, i, j);
        }
    }
    
    swap(arr, i + 1, right);
    return i + 1;
}
```

## 三、优化版本

### 3.1 随机化快速排序

```java
public static void randomizedQuickSort(int[] arr, int left, int right) {
    if (left < right) {
        int pivotIndex = randomizedPartition(arr, left, right);
        randomizedQuickSort(arr, left, pivotIndex - 1);
        randomizedQuickSort(arr, pivotIndex + 1, right);
    }
}

private static int randomizedPartition(int[] arr, int left, int right) {
    // 随机选择一个位置与最右元素交换
    int randomIndex = left + random.nextInt(right - left + 1);
    swap(arr, randomIndex, right);
    return partition(arr, left, right);
}
```

**优化点**：随机选择基准值，避免最坏情况

### 3.2 三数取中快速排序

```java
private static int medianOfThreePartition(int[] arr, int left, int right) {
    int mid = left + (right - left) / 2;
    
    // 将中位数放到最右位置
    if (arr[left] > arr[mid]) swap(arr, left, mid);
    if (arr[left] > arr[right]) swap(arr, left, right);
    if (arr[mid] > arr[right]) swap(arr, mid, right);
    
    swap(arr, mid, right - 1);
    return partition(arr, left, right);
}
```

**优化点**：选择左、中、右三个元素的中位数作为基准值，对于已排序或逆序数组表现更好

### 3.3 三路快速排序（处理重复元素）

```java
public static void threeWayQuickSort(int[] arr, int left, int right) {
    if (left >= right) return;
    
    // 随机选择基准值
    int randomIndex = left + random.nextInt(right - left + 1);
    swap(arr, left, randomIndex);
    
    int pivot = arr[left];
    int lt = left;      // arr[left..lt] < pivot
    int gt = right;     // arr[gt..right] > pivot
    int i = left + 1;   // arr[lt+1..i-1] = pivot
    
    while (i <= gt) {
        if (arr[i] < pivot) {
            swap(arr, i, lt + 1);
            lt++;
            i++;
        } else if (arr[i] > pivot) {
            swap(arr, i, gt);
            gt--;
        } else {
            i++;
        }
    }
    swap(arr, left, lt);
    
    threeWayQuickSort(arr, left, lt - 1);
    threeWayQuickSort(arr, gt + 1, right);
}
```

**优化点**：专门处理包含大量重复元素的情况，将数组分为三部分：< pivot, = pivot, > pivot

### 3.4 非递归快速排序

```java
public static void iterativeQuickSort(int[] arr) {
    if (arr == null || arr.length <= 1) return;
    
    Stack<int[]> stack = new Stack<>();
    stack.push(new int[]{0, arr.length - 1});
    
    while (!stack.isEmpty()) {
        int[] range = stack.pop();
        int left = range[0];
        int right = range[1];
        
        if (left < right) {
            int pivotIndex = partition(arr, left, right);
            
            // 先压入较大的子数组，后压入较小的
            if (pivotIndex - left > right - pivotIndex) {
                stack.push(new int[]{left, pivotIndex - 1});
                stack.push(new int[]{pivotIndex + 1, right});
            } else {
                stack.push(new int[]{pivotIndex + 1, right});
                stack.push(new int[]{left, pivotIndex - 1});
            }
        }
    }
}
```

**优化点**：避免递归栈溢出，适合大规模数据

## 四、经典应用

### 4.1 快速选择算法（求第K小元素）

```java
public static int quickSelect(int[] arr, int left, int right, int k) {
    if (left == right) return arr[left];
    
    int pivotIndex = randomizedPartition(arr, left, right);
    int rank = pivotIndex - left + 1;
    
    if (k == rank) {
        return arr[pivotIndex];
    } else if (k < rank) {
        return quickSelect(arr, left, pivotIndex - 1, k);
    } else {
        return quickSelect(arr, pivotIndex + 1, right, k - rank);
    }
}
```

**时间复杂度**：O(n) 平均情况

**应用**：
- 求中位数
- 求Top K元素
- 数组中的第K大元素

## 五、快速排序 vs 其他排序

| 排序算法 | 平均时间 | 最坏时间 | 空间 | 稳定性 | 适用场景 |
|----------|----------|----------|------|--------|----------|
| 快速排序 | O(n log n) | O(n²) | O(log n) | 不稳定 | 通用排序 |
| 归并排序 | O(n log n) | O(n log n) | O(n) | 稳定 | 链表排序、外部排序 |
| 堆排序 | O(n log n) | O(n log n) | O(1) | 不稳定 | 内存受限 |
| 插入排序 | O(n²) | O(n²) | O(1) | 稳定 | 小规模数据 |

## 六、面试高频题详解

### 题1：快速排序的时间复杂度为什么是O(n log n)？⭐⭐⭐⭐⭐

**推导过程**：
```
递归树分析：
                    [n个元素]                    第0层：处理n个元素
                   /           \
              [n/2]           [n/2]              第1层：处理n个元素
             /     \         /     \
          [n/4]   [n/4]   [n/4]   [n/4]          第2层：处理n个元素
          ...     ...     ...     ...
          
树高：log₂n
每层工作量：O(n)
总时间：O(n) × O(log n) = O(n log n)
```

### 题2：快速排序为什么是不稳定的？⭐⭐⭐⭐⭐

**反例演示**：
```
原始数组：[5a, 3, 5b, 2]  （5a和5b是值相同的不同元素）

以最后一个元素2为pivot进行分区：
1. 扫描发现3 < 2? 否
2. 扫描发现5b < 2? 否
3. 扫描发现2 < 2? 否
4. 将2与首元素5a交换

结果：[2, 3, 5b, 5a]  ← 5a和5b的相对顺序改变了！
```

**对比稳定排序**：归并排序在合并时保持相等元素的相对顺序。

### 题3：如何优化快速排序？⭐⭐⭐⭐⭐

| 优化策略 | 解决的问题 | 实现方式 | 效果 |
|----------|------------|----------|------|
| **随机化** | 最坏情况O(n²) | 随机选择pivot | 期望O(n log n) |
| **三数取中** | 已排序数组性能差 | 取左中右的中位数 | 避免极端分区 |
| **三路快排** | 大量重复元素 | 分为<、=、>三部分 | 处理重复元素最优 |
| **小数组优化** | 递归开销大 | 长度<10用插入排序 | 减少递归深度 |
| **尾递归优化** | 栈溢出风险 | 先处理小的一半 | 栈深度O(log n) |

### 题4：快速选择算法的时间复杂度？⭐⭐⭐⭐

**算法原理**：
```
求第k小元素：
1. 随机选择pivot进行分区
2. pivot的位置就是它的排名rank
3. 如果k == rank，返回pivot
4. 如果k < rank，在左半部分递归
5. 如果k > rank，在右半部分递归，k = k - rank
```

**复杂度分析**：
```
期望比较次数：
T(n) = n + n/2 + n/4 + n/8 + ... 
     = n × (1 + 1/2 + 1/4 + ...)
     = n × 2 = 2n = O(n)
```

**实际应用**：
- LeetCode 215. 数组中的第K个最大元素
- 求中位数（k = n/2）
- 求Top K问题

### 题5：归并排序 vs 快速排序如何选择？⭐⭐⭐⭐

| 对比维度 | 快速排序 | 归并排序 | 选择建议 |
|----------|----------|----------|----------|
| **时间复杂度** | O(n log n) | O(n log n) | 相同 |
| **空间复杂度** | O(log n) | O(n) | 快排更省空间 |
| **稳定性** | 不稳定 | 稳定 | 需要稳定选归并 |
| **最坏情况** | O(n²) | O(n log n) | 数据有序选归并 |
| **适用场景** | 数组、随机访问 | 链表、外部排序 | 数组快排、链表归并 |

**Java中的选择**：
- `Arrays.sort()`（基本类型）：Dual-Pivot快排
- `Arrays.sort()`（对象类型）：归并排序（稳定）
- `Collections.sort()`：归并排序（稳定）
