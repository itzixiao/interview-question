# 线程池详解

## 概述

线程池是管理和复用线程的机制，减少线程创建销毁的开销，提高系统性能。

## 核心参数

```java
public ThreadPoolExecutor(
    int corePoolSize,      // 核心线程数
    int maximumPoolSize,   // 最大线程数
    long keepAliveTime,    // 空闲线程存活时间
    TimeUnit unit,         // 时间单位
    BlockingQueue<Runnable> workQueue,  // 任务队列
    ThreadFactory threadFactory,         // 线程工厂
    RejectedExecutionHandler handler     // 拒绝策略
)
```

### 参数说明

| 参数 | 说明 | 注意事项 |
|------|------|----------|
| corePoolSize | 常驻线程数，即使空闲也保留 | 除非设置 allowCoreThreadTimeOut |
| maximumPoolSize | 最大线程数 | 包含核心线程 |
| keepAliveTime | 非核心线程空闲存活时间 | 核心线程默认永久存活 |
| workQueue | 任务等待队列 | 类型影响性能和策略 |
| handler | 拒绝策略 | 队列满且线程数达最大时触发 |

## 任务执行流程

```
提交任务
    │
    ├─ 核心线程数 < corePoolSize ─→ 创建新线程执行任务
    │
    ├─ 核心线程数 ≥ corePoolSize ─→ 任务进入队列
    │
    ├─ 队列满 ─→ 创建非核心线程（当前线程数 < maximumPoolSize）
    │
    └─ 线程数 ≥ maximumPoolSize ─→ 执行拒绝策略
```

## 任务队列类型

| 队列类型 | 特点 | 适用场景 |
|----------|------|----------|
| ArrayBlockingQueue | 有界数组队列 | 需要限制队列大小 |
| LinkedBlockingQueue | 可选有界链表队列 | 默认无界，可能OOM |
| SynchronousQueue | 不存储元素 | 直接传递，适合高并发 |
| PriorityBlockingQueue | 优先队列 | 需要优先级排序 |
| DelayQueue | 延时队列 | 延时执行任务 |

## 拒绝策略

| 策略 | 行为 |
|------|------|
| AbortPolicy（默认）| 抛出 RejectedExecutionException |
| CallerRunsPolicy | 由调用线程执行任务 |
| DiscardPolicy | 静默丢弃任务 |
| DiscardOldestPolicy | 丢弃队列最老任务，重试提交 |

## 预定义线程池（Executors）

| 方法 | 特点 | 风险 |
|------|------|------|
| newFixedThreadPool | 固定线程数，无界队列 | 队列可能无限增长，OOM |
| newCachedThreadPool | 线程数无限，60秒回收 | 线程数可能无限增长 |
| newSingleThreadExecutor | 单线程，无界队列 | 同 FixedThreadPool |
| newScheduledThreadPool | 定时任务线程池 | - |

**阿里巴巴开发手册建议**：禁止使用 Executors 创建线程池，应直接使用 ThreadPoolExecutor。

## Worker 内部类

Worker 是线程池的核心内部类：
- 继承 AQS，实现不可重入锁
- 封装线程和任务
- 维护线程运行状态

## 线程池监控

```java
// 获取线程池状态
int poolSize = executor.getPoolSize();           // 当前线程数
int activeCount = executor.getActiveCount();     // 活跃线程数
long completed = executor.getCompletedTaskCount(); // 已完成任务数
long taskCount = executor.getTaskCount();         // 总任务数
int queueSize = executor.getQueue().size();       // 队列大小
```

## 最佳实践

1. **合理设置队列大小**：避免OOM和无限等待
2. **自定义线程工厂**：设置有意义的线程名
3. **自定义拒绝策略**：记录日志或持久化任务
4. **优雅关闭**：先 shutdown，再 awaitTermination
5. **监控告警**：监控队列积压和活跃线程数
