# 代码生成器设计

## 一、代码生成器架构

### 1.1 核心组件

1. **数据库元数据解析器**
   - 读取表结构
   - 解析字段信息
   - 识别主键、索引

2. **模板引擎**
   - Velocity / FreeMarker
   - 模板语法解析
   - 变量替换

3. **代码生成器**
   - Entity 生成
   - Mapper生成
   - Service 生成
   - Controller生成

4. **文件写入器**
   - 文件路径计算
   - 文件创建/覆盖
   - 格式化输出

## 二、代码生成流程

```
┌─────────────┐
│ 选择数据库表 │
└──────┬──────┘
       ↓
┌─────────────┐
│ 解析表结构   │ → 表名、字段、类型、注释
└──────┬──────┘
       ↓
┌─────────────┐
│ 加载模板     │ → Entity/Mapper/Service/Controller
└──────┬──────┘
       ↓
┌─────────────┐
│ 渲染模板     │ → Velocity 变量替换
└──────┬──────┘
       ↓
┌─────────────┐
│ 生成文件     │ → 写入磁盘
└──────┬──────┘
       ↓
┌─────────────┐
│ 格式化代码   │ → IDE 格式化
└─────────────┘
```

## 三、Velocity 模板示例

### 3.1 Entity 模板

```velocity
package $packageName.entity;

import com.baomidou.mybatisplus.annotation.TableName;
import lombok.Data;
import java.io.Serializable;

/**
 * $tableName 实体类
 */
@Data
@TableName("$tableName")
public class ${className} implements Serializable {
#foreach($field in $fields)
    /**
     * $field.comment
     */
#if($field.primaryKey)
    @TableId(type = IdType.AUTO)
#end
    private $field.type $field.name;
#end
}
```

### 3.2 Mapper模板

```velocity

import org.apache.ibatis.annotations.Mapper;

/**
 * $tableName Mapper 接口
 */
@Mapper
public interface ${className}Mapper extends BaseMapper<${className}> {
    
}
```

### 3.3 Service 模板

```velocity

import $packageName.entity.${className};
import com.baomidou.mybatisplus.extension.service.IService;

/**
 * $tableName Service 接口

```

### 3.4 ServiceImpl 模板

```velocity
package $packageName.service.impl;

import $packageName.entity.${className};
import $packageName.mapper.${className}Mapper;
import $packageName.service.${className}Service;
import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;
import org.springframework.stereotype.Service;

/**
 * $tableName Service 实现
 */
@Service
public class ${className}ServiceImpl extends ServiceImpl<${className}Mapper, ${className}> 
        implements ${className}Service {
    
}
```

### 3.5 Controller模板

```velocity
package $packageName.controller;

import $packageName.entity.${className};
import $packageName.service.${className}Service;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

/**
 * $tableName Controller
 */
@RestController
@RequestMapping("/api/${className?uncap_first}")
public class ${className}Controller {
    
    @Autowired
    private ${className}Service ${className?uncap_first}Service;
    
    /**
     * 分页查询
     */
    @GetMapping("/page")
    public Map<String, Object> page(
            @RequestParam(defaultValue = "1") Integer pageNum,
            @RequestParam(defaultValue = "10") Integer pageSize) {
        IPage<${className}> page = new Page<>(pageNum, pageSize);
        IPage<${className}> result = ${className?uncap_first}Service.page(page);
        
        Map<String, Object> map = new HashMap<>();
        map.put("records", result.getRecords());
        map.put("total", result.getTotal());
        return map;
    }
    
    /**
     * 根据 ID 查询
     */
    @GetMapping("/{id}")
    public ${className} getById(@PathVariable Long id) {
        return ${className?uncap_first}Service.getById(id);
    }
    
    /**
     * 新增
     */
    @PostMapping
    public boolean save(@RequestBody ${className} entity) {
        return ${className?uncap_first}Service.save(entity);
    }
    
    /**
     * 修改
     */
    @PutMapping
    public boolean update(@RequestBody ${className} entity) {
        return ${className?uncap_first}Service.updateById(entity);
    }
    
    /**
     * 删除
     */
    @DeleteMapping("/{id}")
    public boolean delete(@PathVariable Long id) {
        return ${className?uncap_first}Service.removeById(id);
    }
}
```

## 五、代码生成器实现要点

### 5.1 命名规范

- **表名转驼峰**：`sys_user` → `SysUser`
- **下划线转驼峰**：`user_name` → `userName`
- **首字母小写**：`SysUser` → `sysUser`

### 5.2 包路径规划

```
src/main/java/
├── entity/        # 实体类
├── mapper/        # DAO 层
├── service/       # 服务层
│   └── impl/     # 服务实现
└── controller/    # 控制器层

src/main/resources/
└── mapper/        # MyBatis XML
```

### 5.3 特殊处理

- **保留字处理**：避免使用 Java 关键字作为字段名
- **重复字段**：添加后缀区分（如 status1, status2）
- **超长字段名**：适当截断或缩写

## 八、代码生成器面试题

### 1. 代码生成器的核心原理？

**参考答案：**

代码生成器的核心原理是**模板 + 数据 = 代码**。

**工作流程：**
1. 读取数据库元数据（表结构、字段信息）
2. 加载预定义的模板文件（Velocity/Freemarker）
3. 将元数据填充到模板变量中
4. 渲染生成最终代码文件
5. 写入到指定目录

### 2. 如何设计可扩展的代码生成器？

**参考答案：**

**设计要点：**
1. **模板可配置**：支持自定义模板
2. **插件化架构**：SPI 机制加载扩展
3. **策略模式**：不同类型采用不同生成策略
4. **工厂模式**：统一创建生成器实例

**扩展点：**
- 模板引擎切换
- 字段类型映射自定义
- 命名规则自定义
- 输出格式扩展

### 3. Velocity 和 FreeMarker 的区别？

**参考答案：**

| 特性 | Velocity | FreeMarker |
|------|---------|-----------|
| 语法 | 简单直接 | 更丰富 |
| 学习曲线 | 低 | 中等 |
| 性能 | 高 | 高 |
| Spring Boot集成 | 支持 | 支持更好 |
| 社区活跃度 | 一般 | 活跃 |

**选择建议：**
- 简单场景：Velocity
- 复杂场景：FreeMarker

### 4. 如何处理数据库类型到 Java 类型的映射？

**参考答案：**

**实现方式：**
1. **配置映射表**：维护数据库类型与 Java 类型的映射关系
2. **自定义转换**：支持用户自定义转换规则
3. **类型推断**：根据字段长度、精度自动推断

**示例代码：**
```java
Map<String, String> typeMapping = new HashMap<>();
typeMapping.put("INT", "Integer");
typeMapping.put("BIGINT", "Long");
typeMapping.put("VARCHAR", "String");
typeMapping.put("DATETIME", "LocalDateTime");
typeMapping.put("DECIMAL", "BigDecimal");

// 根据数据库类型获取 Java 类型
String javaType = typeMapping.get(dbType.toUpperCase());
```

### 5. 如何实现批量代码生成？

**参考答案：**

**实现步骤：**
1. 选择多个数据库表
2. 循环遍历每个表
3. 为每个表生成全套代码（Entity/Mapper/Service/Controller）
4. 批量写入文件

**注意事项：**
- 文件命名冲突检测
- 生成进度展示
- 错误处理和回滚机制
- 生成结果汇总报告
