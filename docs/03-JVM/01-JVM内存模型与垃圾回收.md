# JVM 内存模型与垃圾回收

## JVM 内存结构

### 运行时数据区

```
┌─────────────────────────────────────────────────────────────┐
│                      堆内存（Heap）                          │
│  ┌─────────────┬─────────────┬─────────────┬─────────────┐  │
│  │   新生代     │   Eden区    │  Survivor0  │  Survivor1  │  │
│  │  (Young)    │   (8/10)    │   (1/10)    │   (1/10)    │  │
│  └─────────────┴─────────────┴─────────────┴─────────────┘  │
│                         ↓ 晋升                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │                   老年代（Old）                      │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────┐
│                    元空间（Metaspace）                       │
│  - 类元数据                                                  │
│  - 常量池                                                    │
│  - 方法信息                                                  │
│  （JDK8 之前：永久代 PermGen）                               │
└─────────────────────────────────────────────────────────────┘
┌─────────────┐  ┌─────────────┐  ┌─────────────────────────┐
│   虚拟机栈   │  │   本地方法栈 │  │       程序计数器         │
│  - 栈帧     │  │  - Native  │  │  - 当前执行字节码位置    │
│  - 局部变量  │  │             │  │  - 线程私有              │
└─────────────┘  └─────────────┘  └─────────────────────────┘
```

### 堆内存详解

#### 新生代（Young Generation）

| 区域 | 比例 | 说明 |
|------|------|------|
| Eden | 8/10 | 新对象分配区域 |
| Survivor0 | 1/10 | 存活对象复制区 |
| Survivor1 | 1/10 | 存活对象复制区 |

**对象晋升过程**：
1. 新对象在 Eden 分配
2. Minor GC 后存活对象进入 Survivor
3. 对象在 Survivor 之间复制，年龄 +1
4. 年龄达到阈值（默认15）晋升老年代

#### 老年代（Old Generation）

- 存放长期存活对象
- 大对象直接进入（-XX:PretenureSizeThreshold）
- Full GC 时回收

### 栈帧结构

```
┌─────────────────┐
│   局部变量表     │  基本数据类型、对象引用
├─────────────────┤
│    操作数栈      │  方法执行的工作区
├─────────────────┤
│  动态链接        │  指向运行时常量池的方法引用
├─────────────────┤
│  方法返回地址    │  方法执行完毕后的返回位置
├─────────────────┤
│  附加信息        │  调试信息等
└─────────────────┘
```

## 垃圾回收

### 判断对象存活

#### 1. 引用计数法

- 每个对象维护引用计数
- 计数为 0 时回收
- **缺点**：无法解决循环引用

#### 2. 可达性分析（JVM 使用）

```
GC Roots：
  - 虚拟机栈中引用的对象
  - 方法区中类静态属性引用的对象
  - 方法区中常量引用的对象
  - 本地方法栈中 JNI 引用的对象
  - 所有被同步锁持有的对象
```

### 垃圾回收算法

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 标记-清除 | 标记存活，清除未标记 | 简单 | 产生碎片 |
| 复制 | 存活对象复制到另一半 | 无碎片 | 内存利用率低 |
| 标记-整理 | 存活对象向一端移动 | 无碎片 | 移动开销 |
| 分代收集 | 新生代复制，老年代标记-整理 | 高效 | 复杂 |

### 垃圾回收器

| 收集器 | 算法 | 适用区域 | 特点 |
|--------|------|----------|------|
| Serial | 复制/标记-整理 | 新生代/老年代 | 单线程，简单高效 |
| ParNew | 复制 | 新生代 | Serial 的多线程版 |
| Parallel Scavenge | 复制 | 新生代 | 吞吐量优先 |
| Parallel Old | 标记-整理 | 老年代 | Parallel Scavenge 的老年代版 |
| CMS | 标记-清除 | 老年代 | 低停顿 |
| G1 | 标记-整理 + 复制 | 整堆 | 区域化分代 |
| ZGC | 染色指针 | 整堆 | 超低延迟（<10ms）|
| Shenandoah | 转发指针 | 整堆 | 超低延迟 |

### CMS 收集器

**执行阶段**：
1. 初始标记（STW）：标记 GC Roots 直接关联
2. 并发标记：遍历对象图
3. 重新标记（STW）：修正并发期间变动
4. 并发清除：清除垃圾

**缺点**：
- 对 CPU 敏感
- 无法处理浮动垃圾
- 产生内存碎片

### G1 收集器

**特点**：
- 将堆划分为多个 Region
- 优先回收价值最大的 Region
- 可预测的停顿时间模型

**执行阶段**：
1. 初始标记（STW）
2. 并发标记
3. 最终标记（STW）
4. 筛选回收（STW）

### ZGC/Shenandoah

**目标**：
- 停顿时间 < 10ms
- 堆大小不影响停顿时间

**技术**：
- 染色指针（ZGC）
- 转发指针（Shenandoah）
- 读屏障
- 并发整理

## 垃圾回收调优

### 常用参数

```bash
# 堆内存设置
-Xms4g -Xmx4g           # 初始/最大堆内存
-Xmn2g                  # 新生代大小
-XX:MetaspaceSize=256m  # 元空间初始大小
-XX:MaxMetaspaceSize=256m

# 垃圾回收器选择
-XX:+UseG1GC            # 使用 G1
-XX:+UseZGC             # 使用 ZGC
-XX:+UseConcMarkSweepGC # 使用 CMS

# G1 调优
-XX:MaxGCPauseMillis=200    # 最大停顿时间目标
-XX:G1HeapRegionSize=16m    # Region 大小
-XX:InitiatingHeapOccupancyPercent=45  # 触发并发标记的堆占用率
```

### 调优目标选择

| 场景 | 推荐收集器 | 关注点 |
|------|-----------|--------|
| 低延迟 | ZGC/Shenandoah/G1 | 停顿时间 |
| 高吞吐 | Parallel/G1 | 吞吐量 |
| 小内存 | Serial/Parallel | 简单高效 |
| 大内存 | G1/ZGC | 可预测停顿 |

### 对象优先在 Eden 分配

```java
// 大部分对象在 Eden 分配，Minor GC 后存活进入 Survivor
```

### 大对象直接进入老年代

```bash
-XX:PretenureSizeThreshold=1m  # 大于 1MB 的对象直接进入老年代
```

### 长期存活对象进入老年代

```bash
-XX:MaxTenuringThreshold=15  # 对象年龄阈值，默认 15
```

### 动态对象年龄判定

Survivor 中相同年龄所有对象大小总和大于 Survivor 空间一半，年龄大于等于该年龄的对象直接进入老年代。

## 内存泄漏排查

### 常见内存泄漏场景

1. **静态集合**：静态集合持有对象引用
2. **未关闭资源**：数据库连接、文件流等
3. **监听器未移除**：事件监听器持有引用
4. **ThreadLocal**：线程池场景下未 remove
5. **缓存**：无限增长的缓存

### 排查工具

| 工具 | 用途 |
|------|------|
| jmap | 生成堆转储文件 |
| jhat | 分析堆转储文件 |
| VisualVM | 可视化监控分析 |
| MAT | 内存分析工具 |
| Arthas | 实时诊断 |

### 排查步骤

1. **确认泄漏**：堆内存持续增长，GC 无法回收
2. **生成 dump**：`jmap -dump:format=b,file=heap.hprof <pid>`
3. **分析 dump**：使用 MAT 等工具分析
4. **定位原因**：找到占用内存最多的对象及引用链
5. **修复代码**：解除不必要的引用

### GC 日志解读

正常情况 vs 内存泄漏：

```
正常 GC:
[GC 94206K->5000K(98304K)]  ← GC 后内存显著下降

内存泄漏 GC:
[Full GC 94206K->94200K(98304K)]  ← GC 后几乎不变！
```

**典型内存泄漏 GC 日志**：
```
[Full GC (Ergonomics) 
  [PSYoungGen: 25600K->25599K(29696K)]   # 新生代几乎满
  [ParOldGen: 68606K->68600K(68608K)]    # 老年代几乎满
  94206K->94200K(98304K)]                # GC 前后几乎不变！
```

### 内存泄漏现象总结

| 指标 | 正常情况 | 内存泄漏 |
|------|----------|----------|
| GC 后内存 | 明显下降 | 几乎不变 |
| Full GC 频率 | 偶尔 | 持续频繁 |
| 老年代使用 | 波动 | 持续高位 |
| 最终结果 | 稳定运行 | OOM 崩溃 |

### 常见面试题

**问题 1：如何判断是否存在内存泄漏？**

答：观察 GC 日志，如果 Full GC 后内存使用量几乎不变，说明存在内存泄漏。

**问题 2：内存泄漏和内存溢出有什么区别？**

| 内存泄漏 | 内存溢出 |
|----------|----------|
| 对象无法被回收 | 内存不足 |
| 内存逐渐增长 | 一次性失败 |
| 最终导致 OOM | 可能因泄漏或配置不足 |

**问题 3：如何避免静态集合导致的内存泄漏？**

```java
// 错误：静态集合持有对象引用
private static final List<Object> CACHE = new ArrayList<>();

// 正确方案1：使用 WeakHashMap
private static final Map<Object, Object> CACHE = new WeakHashMap<>();

// 正确方案2：使用软引用
private static final List<SoftReference<Object>> CACHE = new ArrayList<>();

// 正确方案3：定期清理
CACHE.clear();  // 或使用有界队列
```
