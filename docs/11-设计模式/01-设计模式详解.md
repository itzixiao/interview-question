# 23种设计模式详解

## 一、设计模式概述

设计模式是软件开发中经过验证的解决方案，是前辈们总结的最佳实践。GoF（Gang of Four）总结了23种经典设计模式，分为三大类：

| 类型 | 数量 | 关注点 |
|------|------|--------|
| 创建型模式 | 5种 | 对象的创建过程 |
| 结构型模式 | 7种 | 类和对象的组合 |
| 行为型模式 | 11种 | 对象之间的通信 |

---

## 二、创建型模式（5种）

关注对象的创建过程，将对象的创建和使用分离。

### 1. 单例模式（Singleton）

**定义**：确保一个类只有一个实例，并提供一个全局访问点。

**实现方式**：
- 饿汉式：类加载时就创建实例
- 懒汉式：首次使用时创建实例
- 双重检查锁：懒加载 + 线程安全 + 高性能
- 静态内部类：利用类加载机制保证线程安全（推荐）
- 枚举：天然防止反射攻击和序列化破坏（最佳实践）

```java
// 双重检查锁单例
public class Singleton {
    private static volatile Singleton instance;
    
    private Singleton() {}
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

### 2. 工厂方法模式（Factory Method）

**定义**：定义一个创建对象的接口，让子类决定实例化哪一个类。

**核心组件**：
- Product：产品接口
- ConcreteProduct：具体产品
- Factory：工厂接口
- ConcreteFactory：具体工厂

```java
interface Product { void use(); }

class ProductA implements Product {
    public void use() { System.out.println("使用产品A"); }
}

interface Factory { Product create(); }

class FactoryA implements Factory {
    public Product create() { return new ProductA(); }
}
```

**示例代码**：[FactoryMethodDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/creational/factorymethod/FactoryMethodDemo.java)

---

### 3. 抽象工厂模式（Abstract Factory）

**定义**：提供一个创建一系列相关或相互依赖对象的接口。

**与工厂方法的区别**：
- 工厂方法：一种产品
- 抽象工厂：产品族（多种产品）

**示例代码**：[AbstractFactoryDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/creational/abstractfactory/AbstractFactoryDemo.java)

---

### 4. 建造者模式（Builder）

**定义**：将一个复杂对象的构建与它的表示分离。

**两种实现方式**：
1. 传统方式：包含 Director（指挥者）
2. 简化方式：链式调用（省略 Director）

```java
// 链式调用
Computer computer = new Computer.Builder()
    .cpu("Intel i7")
    .ram("16GB")
    .storage("512GB SSD")
    .build();
```

**典型应用**：StringBuilder、AlertDialog.Builder

**示例代码**：[BuilderDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/creational/builder/BuilderDemo.java)

---

### 5. 原型模式（Prototype）

**定义**：用原型实例指定创建对象的种类，通过拷贝这些原型创建新的对象。

**核心概念**：
- 浅拷贝：复制引用
- 深拷贝：复制对象

**示例代码**：[PrototypeDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/creational/prototype/PrototypeDemo.java)

## 三、结构型模式（7种）

关注类和对象的组合，形成更大的结构。

### 1. 适配器模式（Adapter）

**定义**：将一个类的接口转换成客户希望的另一个接口。

**两种实现**：
- 类适配器：继承方式
- 对象适配器：组合方式（推荐）

**典型应用**：InputStreamReader、OutputStreamWriter

**示例代码**：[AdapterDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/structural/adapter/AdapterDemo.java)

---

### 2. 桥接模式（Bridge）

**定义**：将抽象部分与它的实现部分分离，使它们都可以独立地变化。

**核心思想**：使用组合替代继承，避免类爆炸。

**典型应用**：JDBC API（抽象）与各种数据库驱动（实现）

**示例代码**：[BridgeDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/structural/bridge/BridgeDemo.java)

---

### 3. 组合模式（Composite）

**定义**：将对象组合成树形结构以表示「部分-整体」的层次结构。

**核心组件**：
- Component：组件接口
- Leaf：叶子节点
- Composite：组合节点

**典型应用**：文件系统、组织架构

**示例代码**：[CompositeDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/structural/composite/CompositeDemo.java)

---

### 4. 装饰器模式（Decorator）

**定义**：动态地给一个对象添加一些额外的职责。

**核心特点**：
- 装饰器与组件类型相同
- 可无限嵌套
- 比继承更灵活

**典型应用**：Java I/O（InputStream、Reader）

```java
InputStream is = new BufferedInputStream(
    new FileInputStream("file.txt")
);
```

**示例代码**：[DecoratorDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/structural/decorator/DecoratorDemo.java)

---

### 5. 外观模式（Facade）

**定义**：为子系统中的一组接口提供一个一致的界面。

**核心作用**：简化复杂系统的使用。

**典型应用**：JdbcUtils、Spring BeanFactory

**示例代码**：[FacadeDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/structural/facade/FacadeDemo.java)

---

### 6. 享元模式（Flyweight）

**定义**：运用共享技术有效地支持大量细粒度的对象。

**核心概念**：
- 内部状态：可共享
- 外部状态：不可共享

**典型应用**：Integer 缓存（-128~127）、String 常量池

**示例代码**：[FlyweightDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/structural/flyweight/FlyweightDemo.java)

---

### 7. 代理模式（Proxy）

**定义**：为其他对象提供一种代理以控制对这个对象的访问。

**代理类型**：
- 远程代理：RMI、Dubbo
- 虚拟代理：图片懒加载
- 保护代理：权限校验
- 智能引用：日志记录、性能监控

**典型应用**：Spring AOP、MyBatis Mapper

**示例代码**：[ProxyDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/structural/proxy/ProxyDemo.java)

## 四、行为型模式（11种）

关注对象之间的通信和职责分配。

### 1. 策略模式（Strategy）

**定义**：定义一系列算法，把它们一个个封装起来，并且使它们可相互替换。

**核心作用**：消除大量的 if-else 或 switch 语句。

**示例代码**：[StrategyDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/behavioral/strategy/StrategyDemo.java)

---

### 2. 模板方法模式（Template Method）

**定义**：定义一个操作中算法的骨架，而将一些步骤延迟到子类中。

**核心组件**：
- 模板方法：定义算法骨架
- 基本方法：抽象方法、具体方法、钩子方法

**典型应用**：HttpServlet.service()、JdbcTemplate

**示例代码**：[TemplateMethodDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/behavioral/templatemethod/TemplateMethodDemo.java)

---

### 3. 观察者模式（Observer）

**定义**：定义对象间的一种一对多的依赖关系。

**核心组件**：
- Subject：主题
- Observer：观察者

**典型应用**：Spring 事件机制、GUI 事件处理

**示例代码**：[ObserverDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/behavioral/observer/ObserverDemo.java)

---

### 4. 迭代器模式（Iterator）

**定义**：提供一种方法顺序访问一个聚合对象中各个元素。

**核心作用**：无需暴露集合内部结构即可遍历。

**典型应用**：Java 集合框架

**示例代码**：[IteratorDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/behavioral/iterator/IteratorDemo.java)

---

### 5. 责任链模式（Chain of Responsibility）

**定义**：使多个对象都有机会处理请求，将这些对象连成一条链。

**典型应用**：Servlet Filter、Spring Interceptor

**示例代码**：[ChainOfResponsibilityDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/behavioral/chainofresponsibility/ChainOfResponsibilityDemo.java)

---

### 6. 命令模式（Command）

**定义**：将一个请求封装为一个对象。

**核心优点**：
- 支持撤销操作
- 支持宏命令
- 支持请求排队

**示例代码**：[CommandDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/behavioral/command/CommandDemo.java)

---

### 7. 备忘录模式（Memento）

**定义**：在不破坏封装性的前提下，捕获一个对象的内部状态。

**核心组件**：
- Memento：备忘录
- Originator：原发器
- Caretaker：负责人

**典型应用**：文本编辑器撤销、游戏存档

**示例代码**：[MementoDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/behavioral/memento/MementoDemo.java)

---

### 8. 状态模式（State）

**定义**：允许一个对象在其内部状态改变时改变它的行为。

**与策略模式的区别**：
- 策略模式：客户端决定使用哪个策略
- 状态模式：状态对象内部决定状态转换

**示例代码**：[StateDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/behavioral/state/StateDemo.java)

### 9. 访问者模式（Visitor）

**定义**：表示一个作用于某对象结构中的各元素的操作。

**核心机制**：双分派

**示例代码**：[VisitorDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/behavioral/visitor/VisitorDemo.java)

### 10. 中介者模式（Mediator）

**定义**：用一个中介对象来封装一系列的对象交互。

**核心作用**：将多对多的交互转化为一对多。

**典型应用**：MVC 框架、聊天室

**示例代码**：[MediatorDemo.java](../../interview-microservices-parent/interview-service/src/main/java/cn/itzixiao/interview/designpattern/behavioral/mediator/MediatorDemo.java)

### 11. 解释器模式（Interpreter）

**定义**：给定一个语言，定义它的文法的一种表示。

**典型应用**：正则表达式、SQL 解析器

## 五、设计原则（SOLID）

| 原则 | 名称 | 含义 |
|------|------|------|
| SRP | 单一职责原则 | 一个类只负责一项职责 |
| OCP | 开闭原则 | 对扩展开放，对修改关闭 |
| LSP | 里氏替换原则 | 子类可以替换父类 |
| ISP | 接口隔离原则 | 接口要小而专一 |
| DIP | 依赖倒置原则 | 依赖抽象不依赖具体 |
