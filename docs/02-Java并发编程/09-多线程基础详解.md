## 一、线程创建的三种方式

### 1.1 继承 Thread 类

**实现方式：**

```java
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread 类线程运行：" + Thread.currentThread().getName());
    }
}

// 使用
MyThread thread = new MyThread();
thread.start();
```

**特点分析：**

- ✅ **优点**：代码简单直观，可以直接操作线程对象
- ❌ **缺点**：受 Java 单继承限制，无法继承其他类
- 📌 **适用场景**：简单的线程任务，不需要继承其他类的场景

---

### 1.2 实现 Runnable 接口（推荐）

**实现方式：**

```java
class MyRunnable implements Runnable {
    @Override
    public void run() {
        System.out.println("Runnable 接口线程运行：" + Thread.currentThread().getName());
    }
}

// 使用
Runnable runnable = new MyRunnable();
Thread thread = new Thread(runnable, "RunnableThread");
thread.start();
```

**特点分析：**

- ✅ **优点**：
    - 避免单继承限制，可同时实现多个接口
    - 适合多个线程处理同一资源的场景
    - 符合面向对象的组合设计原则
- ❌ **缺点**：无法直接返回执行结果
- 📌 **适用场景**：生产环境推荐使用，特别是需要资源共享的场景

---

### 1.3 实现 Callable 接口

**实现方式：**

```java
class MyCallable implements Callable<String> {
    @Override
    public String call() throws Exception {
        Thread.sleep(500);
        return "Callable 执行结果 from " + Thread.currentThread().getName();
    }
}

// 使用
Callable<String> callable = new MyCallable();
FutureTask<String> futureTask = new FutureTask<>(callable);
Thread thread = new Thread(futureTask, "CallableThread");
thread.start();

// 获取返回值
String result = futureTask.get(); // 阻塞等待结果
System.out.println("Callable 返回结果：" + result);
```

**特点分析：**

- ✅ **优点**：
    - 支持返回值，可获取异步执行结果
    - 可以抛出受检异常
    - 配合 FutureTask 使用更灵活
- ❌ **缺点**：需要配合 FutureTask 使用，代码稍复杂
- 📌 **适用场景**：需要获取线程执行结果的场景

---

### 1.4 三种方式对比

| 对比项      | Thread 类 | Runnable 接口 | Callable 接口 |
|----------|----------|-------------|-------------|
| **继承限制** | 单继承      | 可实现多接口      | 可实现多接口      |
| **返回值**  | 无        | 无           | ✅ 有         |
| **异常处理** | 不能抛受检异常  | 不能抛受检异常     | ✅ 可以抛受检异常   |
| **灵活性**  | 低        | 高           | 高           |
| **推荐度**  | ⭐⭐       | ⭐⭐⭐⭐⭐       | ⭐⭐⭐⭐        |

**最佳实践建议：**

- 优先选择 **Runnable 接口**（无返回值场景）
- 需要返回值时选择 **Callable 接口**
- 尽量避免继承 Thread 类

## 二、线程的 6 种状态

### 2.1 状态转换图

```
┌─────────┐   start()   ┌─────────┐   获取锁   ┌─────────┐
│  NEW    │ ─────────→  │ RUNNABLE│ ───────→  │ RUNNING │
│ (新建)   │             │ (就绪)   │          │ (运行)   │
└─────────┘             └─────────┘           └────┬────┘
     ↑                                             │
     │    ┌───────────┐   wait()    ┌─────────┐    │
     └────│ TERMINATED│ ←────────  ─│ WAITING │ ←──┘ 释放锁
          │ (终止)     │   notify()  │ (等待)   │
          └───────────┘             └────┬────┘
               ↑                         │ sleep()
               │    ┌─────────┐          │ (不释放锁)
               └────│ BLOCKED │ ←────────┘
                    │ (阻塞)  │
                    └─────────┘
```

### 2.2 状态详细说明

#### 1. NEW（新建状态）

- **定义**：线程对象已创建，但未调用 `start()` 方法
- **示例**：`Thread t = new Thread();` // 此时处于 NEW 状态

#### 2. RUNNABLE（就绪/运行状态）

- **定义**：已调用 `start()`，等待 JVM 调度执行
- **包含**：操作系统中的 Ready 和 Running 两种状态
- **示例**：`t.start();` // 调用后进入 RUNNABLE

#### 3. BLOCKED（阻塞状态）

- **定义**：等待获取监视器锁（synchronized 锁）
- **触发条件**：尝试进入 synchronized 块/方法，但锁被其他线程占用
- **示例**：
  ```java
  synchronized (lock) { // 如果 lock 被占用，则进入 BLOCKED
      // 临界区代码
  }
  ```

#### 4. WAITING（无限期等待状态）

- **定义**：无限期等待其他线程唤醒
- **触发方法**：
    - `Object.wait()`（无超时参数）
    - `Thread.join()`（无超时参数）
    - `LockSupport.park()`
- **示例**：
  ```java
  obj.wait(); // 进入 WAITING，需要其他线程调用 notify()/notifyAll()
  ```

#### 5. TIMED_WAITING（超时等待状态）

- **定义**：在指定时间内自动唤醒，无需其他线程通知
- **触发方法**：
    - `Thread.sleep(long millis)`
    - `Object.wait(long timeout)`
    - `Thread.join(long millis)`
- **示例**：
  ```java
  Thread.sleep(1000); // 休眠 1 秒后自动唤醒
  ```

#### 6. TERMINATED（终止状态）

- **定义**：线程执行完毕（run() 方法结束）
- **触发条件**：正常执行完成或异常退出

---

### 2.3 状态演示代码

```java
public class ThreadStateDemo {
    public static void main(String[] args) throws InterruptedException {
        Object lock = new Object();
        
        Thread waitingThread = new Thread(() -> {
            synchronized (lock) {
                try {
                    System.out.println("等待线程：获取锁，准备 wait()");
                    lock.wait(1000); // 进入 TIMED_WAITING
                    System.out.println("等待线程：被唤醒，继续执行");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }, "WaitingThread");
        
        // 查看不同状态
        System.out.println("新建状态：" + waitingThread.getState()); // NEW
        
        waitingThread.start();
        Thread.sleep(100); // 确保线程启动
        
        System.out.println("启动后状态：" + waitingThread.getState()); // RUNNABLE 或 TIMED_WAITING
        
        Thread.sleep(1500);
        System.out.println("执行完毕后状态：" + waitingThread.getState()); // TERMINATED
    }
}
```

## 三、线程基本方法对比

### 3.1 核心方法对比表

| 方法                | 说明          | 是否释放锁 | 状态变化            | 使用场景      |
|-------------------|-------------|-------|-----------------|-----------|
| **sleep(millis)** | 暂停执行指定时间    | ❌ 不释放 | → TIMED_WAITING | 定时任务、轮询间隔 |
| **yield()**       | 提示调度器让出 CPU | ❌ 不释放 | → RUNNABLE      | 同优先级线程礼让  |
| **join()**        | 等待线程执行完毕    | ❌ 不释放 | → WAITING       | 等待线程完成后继续 |
| **interrupt()**   | 中断线程        | -     | -               | 停止阻塞中的线程  |

---

### 3.2 sleep() 方法详解

**基本用法：**

```java
try {
    Thread.sleep(1000); // 休眠 1 秒
} catch (InterruptedException e) {
    // 被中断时的处理
    Thread.currentThread().interrupt(); // 重新设置中断标志
}
```

**关键特性：**

- ⏰ **精确性**：实际休眠时间可能略长于指定时间（取决于系统调度）
- 🔒 **锁行为**：不释放已持有的锁
- ⚠️ **异常处理**：必须捕获 `InterruptedException`
- 🛑 **中断响应**：被 interrupt() 会立即唤醒并抛出异常

---

### 3.3 join() 方法详解

**基本用法：**

```java
Thread worker = new Thread(() -> {
    System.out.println("工作线程开始执行任务...");
    try {
        Thread.sleep(500);
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
    System.out.println("工作线程任务完成");
}, "Worker");

System.out.println("主线程：启动工作线程");
worker.start();

System.out.println("主线程：等待工作线程完成（join）");
worker.join(); // 阻塞直到 worker 执行完毕

System.out.println("主线程：继续执行");
```

**重载方法：**

- `join()` - 无限期等待
- `join(long millis)` - 最多等待指定毫秒
- `join(long millis, int nanos)` - 更精确的时间控制

---

### 3.4 interrupt() 方法详解

**正确的中断处理：**

```java
Thread interruptibleThread = new Thread(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        System.out.println("工作中...");
        try {
            Thread.sleep(500);
        } catch (InterruptedException e) {
            System.out.println("收到中断信号，清理资源后退出");
            Thread.currentThread().interrupt(); // 重新设置中断标志
            break;
        }
    }
});

interruptibleThread.start();
Thread.sleep(1200);
System.out.println("主线程：发送中断信号");
interruptibleThread.interrupt();
interruptibleThread.join();
```

**关键点：**

- ✅ 使用 `isInterrupted()` 检查中断标志
- ✅ 捕获 `InterruptedException` 后重新设置中断标志
- ✅ 及时清理资源，优雅退出

## 四、sleep() vs wait() 深度对比

### 4.1 核心区别对比表

| 对比项       | Thread.sleep()            | Object.wait()             |
|-----------|---------------------------|---------------------------|
| **所属类**   | Thread 类（静态方法）            | Object 类（实例方法）            |
| **是否释放锁** | ❌ 不释放锁                    | ✅ 释放锁                     |
| **使用范围**  | 任何地方                      | synchronized 同步块内         |
| **唤醒方式**  | 超时自动唤醒                    | notify()/notifyAll()      |
| **异常处理**  | 必须捕获 InterruptedException | 必须捕获 InterruptedException |
| **依赖监视器** | 不需要                       | 需要（必须先获取锁）                |

---

### 4.2 示例代码对比

#### 示例 1：sleep() 不释放锁

```java
Object sleepLock = new Object();

Thread sleepHolder = new Thread(() -> {
    synchronized (sleepLock) {
        System.out.println("线程 A: 获取锁，开始 sleep(2000ms)");
        try {
            Thread.sleep(2000); // 休眠 2 秒，但不释放锁
            System.out.println("线程 A: sleep 结束，继续执行");
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}, "SleepHolder");

Thread sleepWaiter = new Thread(() -> {
    synchronized (sleepLock) {
        System.out.println("线程 B: 获取锁，执行任务");
    }
}, "SleepWaiter");

sleepHolder.start();
Thread.sleep(100);
sleepWaiter.start();

sleepHolder.join();
sleepWaiter.join();

// 结论：线程 A sleep 时持有锁，线程 B 必须等待
```

#### 示例 2：wait() 释放锁

```java
final Object waitLock = new Object();

Thread waiter = new Thread(() -> {
    synchronized (waitLock) {
        System.out.println("线程 C: 获取锁，准备 wait()");
        try {
            waitLock.wait(2000); // 等待 2 秒或直到被通知
            System.out.println("线程 C: 被唤醒，继续执行");
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}, "Waiter");

Thread notifier = new Thread(() -> {
    synchronized (waitLock) {
        System.out.println("线程 D: 获取锁，执行任务");
        try {
            Thread.sleep(500);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        System.out.println("线程 D: 任务完成，调用 notify()");
        waitLock.notify(); // 唤醒等待的线程
    }
}, "Notifier");

waiter.start();
Thread.sleep(100);
notifier.start();

waiter.join();
notifier.join();

// 结论：线程 C wait() 时释放锁，线程 D 可以获取并执行
```

### 4.3 经典应用：生产者 - 消费者模式

```java
static class Buffer {
    private int data;
    private boolean empty = true;

    public synchronized void put(int value) {
        while (!empty) { // 缓冲区非空，等待
            try {
                wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        this.data = value;
        this.empty = false;
        System.out.println("[Producer] 生产数据：" + value);
        notifyAll(); // 通知消费者
    }

    public synchronized int get() {
        while (empty) { // 缓冲区为空，等待
            try {
                wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        this.empty = true;
        System.out.println("[Consumer] 消费数据：" + data);
        notifyAll(); // 通知生产者
        return data;
    }
}
```

**关键点：**

- 使用 `while` 循环检查条件（防止虚假唤醒）
- 使用 `notifyAll()` 而不是 `notify()`（确保所有等待线程都有机会）
- `wait()` 必须在 `synchronized` 块中调用

## 五、死锁详解

### 5.1 死锁定义

**死锁**是指两个或多个线程互相等待对方释放锁资源，导致所有线程都无法继续执行的僵局状态。

---

### 5.2 死锁产生的 4 个必要条件

| 条件          | 说明              | 破坏方法          |
|-------------|-----------------|---------------|
| **互斥条件**    | 资源一次只能被一个线程占用   | 无法破坏（资源固有属性）  |
| **请求与保持条件** | 线程已持有资源，又申请新资源  | 一次性申请所有资源     |
| **不剥夺条件**   | 已获得的资源不能被强制剥夺   | 允许抢占资源        |
| **循环等待条件**  | 线程间形成头尾相连的循环等待链 | 按固定顺序获取锁（最常用） |

---

### 5.3 经典死锁场景（ABBA 模式）

```java
Object lock1 = new Object();
Object lock2 = new Object();

Thread threadA = new Thread(() -> {
    synchronized (lock1) {
        System.out.println("线程 A: 获取 lock1");
        try { Thread.sleep(100); } catch (InterruptedException e) {}
        System.out.println("线程 A: 等待 lock2...");
        synchronized (lock2) {
            System.out.println("线程 A: 获取 lock2，执行任务");
        }
    }
}, "Thread-A");

Thread threadB = new Thread(() -> {
    synchronized (lock2) {
        System.out.println("线程 B: 获取 lock2");
        try { Thread.sleep(100); } catch (InterruptedException e) {}
        System.out.println("线程 B: 等待 lock1...");
        synchronized (lock1) {
            System.out.println("线程 B: 获取 lock1，执行任务");
        }
    }
}, "Thread-B");

threadA.start();
threadB.start();

// ⚠️ 检测到死锁！两个线程都在等待对方释放锁
```

**死锁分析：**

- 线程 A：持有 lock1，等待 lock2
- 线程 B：持有 lock2，等待 lock1
- 形成循环等待链：A → B → A

---

### 5.4 解决死锁的正确方式

**方案：按相同顺序获取锁（破坏循环等待条件）**

```java
final Object resource1 = new Object();
final Object resource2 = new Object();

Thread safeThreadA = new Thread(() -> {
    synchronized (resource1) {
        System.out.println("安全线程 A: 获取 resource1");
        try { Thread.sleep(100); } catch (InterruptedException e) {}
        synchronized (resource2) {
            System.out.println("安全线程 A: 获取 resource2，执行任务");
        }
    }
}, "SafeThread-A");

Thread safeThreadB = new Thread(() -> {
    synchronized (resource1) { // 也先获取 resource1，破坏循环等待
        System.out.println("安全线程 B: 获取 resource1");
        try { Thread.sleep(100); } catch (InterruptedException e) {}
        synchronized (resource2) {
            System.out.println("安全线程 B: 获取 resource2，执行任务");
        }
    }
}, "SafeThread-B");

safeThreadA.start();
safeThreadB.start();

// ✅ 两个线程都成功执行完成，无死锁
```

---

### 5.5 死锁预防策略

#### 策略 1：破坏请求与保持条件

- **方法**：一次性申请所有资源，否则不分配任何资源
- **缺点**：资源利用率低，可能导致饥饿

#### 策略 2：破坏不剥夺条件

- **方法**：允许抢占资源，如果一个线程请求不到所有资源，则释放已持有的资源
- **缺点**：可能导致饥饿，增加系统开销

#### 策略 3：破坏循环等待条件（最常用）✅

- **方法**：对所有资源编号，按编号顺序申请
- **示例**：总是先获取 lock1，再获取 lock2
- **优点**：简单有效，性能较好

---

### 5.6 生产环境最佳实践

#### 1. 使用定时锁

```java
ReentrantLock lock1 = new ReentrantLock();
ReentrantLock lock2 = new ReentrantLock();

if (lock1.tryLock(1, TimeUnit.SECONDS)) {
    try {
        if (lock2.tryLock(1, TimeUnit.SECONDS)) {
            try {
                // 执行业务逻辑
            } finally {
                lock2.unlock();
            }
        }
    } finally {
        lock1.unlock();
    }
} else {
    // 超时处理，回滚并重试
}
```

#### 2. 减少锁粒度

- 使用更细粒度的锁对象
- 缩小同步代码块范围
- 使用读写锁（ReadWriteLock）

#### 3. 使用并发工具类

- `ConcurrentHashMap`（分段锁）
- `ReentrantLock`（可中断、可超时）
- `Semaphore`（信号量控制并发数）

#### 4. 死锁检测工具

- **命令行工具**：`jps` + `jstack <pid>`
- **图形化工具**：VisualVM、JConsole
- **日志分析**：查找长时间 BLOCKED 的线程

## 六、高频面试题

**问题 1：线程创建的三种方式及区别？**

**答：**

1. **继承 Thread 类**
    - 优点：代码简单直观，可以直接操作线程对象
    - 缺点：受 Java 单继承限制，无法继承其他类
    - 适用场景：简单的线程任务，不需要继承其他类的场景

2. **实现 Runnable 接口（推荐）**
    - 优点：避免单继承限制，可同时实现多个接口；适合多个线程处理同一资源的场景
    - 缺点：无法直接返回执行结果
    - 适用场景：生产环境推荐使用，特别是需要资源共享的场景

3. **实现 Callable 接口**
    - 优点：支持返回值，可获取异步执行结果；可以抛出受检异常
    - 缺点：需要配合 FutureTask 使用，代码稍复杂
    - 适用场景：需要获取线程执行结果的场景

---

**问题 2：Java 线程的生命周期和状态转换？**

**答：**

Java 线程有 6 种状态：**NEW → RUNNABLE → BLOCKED/WAITING/TIMED_WAITING → TERMINATED**

**状态转换关键点：**

| 方法/事件     | 状态变化                    |
|-----------|-------------------------|
| `start()` | NEW → RUNNABLE          |
| 获取锁       | RUNNABLE → RUNNING      |
| `wait()`  | RUNNING → WAITING       |
| `sleep()` | RUNNING → TIMED_WAITING |
| 等待锁       | RUNNING → BLOCKED       |
| 执行完毕      | → TERMINATED            

---

**问题 3：sleep() 和 wait() 的本质区别？**

**答：**

| 对比项       | Thread.sleep() | Object.wait()        |
|-----------|----------------|----------------------|
| **所属类**   | Thread 类（静态方法） | Object 类（实例方法）       |
| **锁释放**   | ❌ 不释放锁         | ✅ 释放锁                |
| **使用范围**  | 任何地方           | synchronized 同步块内    |
| **唤醒方式**  | 超时自动唤醒         | notify()/notifyAll() |
| **依赖监视器** | 不需要            | 需要（必须先获取锁）           |

**最关键区别**：sleep() 不释放锁，wait() 释放锁

---

**问题 4：什么是死锁？如何预防和检测？**

**答：**

**1. 死锁定义**

多个线程互相等待对方释放锁导致的僵局状态。

**2. 死锁产生的 4 个必要条件**

| 条件      | 说明              |
|---------|-----------------|
| 互斥条件    | 资源一次只能被一个线程占用   |
| 请求与保持条件 | 线程已持有资源，又申请新资源  |
| 不剥夺条件   | 已获得的资源不能被强制剥夺   |
| 循环等待条件  | 线程间形成头尾相连的循环等待链 |

**3. 预防方法（破坏必要条件）**

- **破坏循环等待**：按固定顺序获取锁（最常用）✅
- **破坏请求与保持**：一次性申请所有资源
- **破坏不剥夺**：允许抢占资源

**4. 检测方法**

- **命令行工具**：`jps` + `jstack <pid>` 查看线程堆栈
- **图形化工具**：VisualVM、JConsole 监控
- **日志分析**：查找长时间 BLOCKED 的线程

---

**问题 5：守护线程的作用和应用场景？**

**答：**

**1. 守护线程定义**

后台服务线程，为其他线程提供服务。

**2. 核心特点**

- 所有非守护线程结束后，JVM 自动退出，不等待守护线程
- 必须在 `start()` 之前调用 `setDaemon(true)` 设置

**3. 典型应用场景**

- GC 垃圾回收线程
- JIT 编译线程
- 监控线程（如心跳检测）
- 定时任务调度线程

**问题 6：如何正确中断一个线程？**

**答：**

**1. 错误做法**

- ❌ 不要使用 `stop()` 方法（已废弃）
- ❌ 不要直接修改标志位（无法中断阻塞状态）

**2. 正确做法**

```java
Thread thread = new Thread(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        try {
            Thread.sleep(500);
        } catch (InterruptedException e) {
            // 关键：重新设置中断标志
            Thread.currentThread().interrupt();
            break;
        }
    }
});
thread.start();
thread.interrupt();
```

**3. 中断处理三步骤**

1. 使用 `interrupt()` 方法发送中断信号
2. 在线程中检查 `isInterrupted()` 标志
3. 捕获 `InterruptedException` 后重新设置中断标志
