# 前端实战指南

> 面向 Java 全栈工程师的前端实战经验汇总

## 技术选型决策

### 前端框架选型

**选择 Vue 的场景：**

- 团队前端经验相对较少
- 需要快速开发和迭代
- 项目以数据展示和管理后台为主
- 需要渐进式升级存量项目

**选择 React 的场景：**

- 团队有丰富的前端经验
- 需要高度定制化和灵活性
- 项目复杂度极高（如大型 SaaS 平台）
- 需要 React Native 跨平台能力

### 构建工具选型

| 特性    | Webpack | Vite        |
|-------|---------|-------------|
| 启动速度  | 慢（需要打包） | 快（原生 ESM）   |
| 热更新   | 较慢      | 极快（HMR 毫秒级） |
| 配置复杂度 | 复杂      | 简单          |
| 适用场景  | 大型复杂项目  | 中小型项目、快速开发  |

### 微前端方案选型

| 特性    | qiankun    | Module Federation |
|-------|------------|-------------------|
| 实现方式  | 运行时沙箱      | 构建时共享             |
| 技术栈限制 | 无限制        | 需统一构建工具           |
| 适用场景  | 异构技术栈、存量改造 | 同构技术栈、新架构         |

## 实战场景解决方案

### 场景一：大型表单性能优化

**问题**：表单字段超过 100 个，输入卡顿

**解决方案**：

1. 使用 `shallowRef` 替代 `ref` 减少响应式开销
2. 表单字段分组，使用 `markRaw` 标记静态配置
3. 虚拟滚动展示（如 `vue-virtual-scroller`）
4. 防抖处理输入事件（300ms）
5. 使用 `v-memo` 或 `React.memo` 缓存不相关组件

### 场景二：首屏加载优化

**问题**：首屏加载时间超过 3 秒

**优化策略**：

| 层面   | 具体措施                      |
|------|---------------------------|
| 资源优化 | 路由懒加载、图片懒加载、WebP 格式       |
| 构建优化 | 代码分割、Tree-shaking、Gzip 压缩 |
| 网络优化 | CDN 部署、HTTP/2、资源预加载       |
| 渲染优化 | SSR/SSG、骨架屏、关键 CSS 内联     |

### 场景三：跨团队组件库共享

**架构方案**：

1. **Monorepo 架构**：pnpm workspaces + Turborepo
2. **组件库设计**：原子化设计 + 主题定制能力
3. **发布流程**：Changesets 自动化版本管理

### 场景四：微前端存量系统整合

**实施步骤**：

1. **基座应用**：统一登录、公共布局、路由管理
2. **子应用改造**：生命周期导出、样式隔离、通信机制
3. **部署策略**：独立部署、灰度发布、回滚机制

## 常见陷阱与避坑

### Vue 开发陷阱

| 陷阱      | 错误做法                | 正确做法                 |
|---------|---------------------|----------------------|
| 响应式丢失   | `arr[0] = newVal`   | `Vue.set` 或 `splice` |
| 计算属性副作用 | 在 computed 中修改外部状态  | computed 只用于派生数据     |
| 生命周期误用  | 在 `created` 中操作 DOM | DOM 操作放在 `mounted`   |

### React 开发陷阱

| 陷阱         | 错误做法              | 正确做法                         |
|------------|-------------------|------------------------------|
| Hooks 调用顺序 | 在条件语句中调用 Hooks    | 始终在组件顶层调用                    |
| 闭包陷阱       | useEffect 依赖数组遗漏  | 使用 eslint-plugin-react-hooks |
| 状态更新异步     | 依赖更新后的 state 立即执行 | 使用 useEffect 监听              |

### 性能优化陷阱

| 陷阱      | 说明                  |
|---------|---------------------|
| 过度优化    | 所有组件都使用 memo，反而增加开销 |
| 错误的 key | 使用数组索引作为 key，导致渲染问题 |
| 内存泄漏    | 未清理的事件监听和定时器        |

## 面试高频考点

### 技术对比类

1. **Vue2 vs Vue3 核心区别**
    - 响应式原理：Object.defineProperty vs Proxy
    - API 风格：Options API vs Composition API
    - 性能：静态提升、Patch Flags、Tree-shaking

2. **React vs Vue 核心区别**
    - 模板语法：JSX vs 模板
    - 响应式：手动 setState vs 自动追踪
    - 生态：更灵活 vs 更规范

3. **Webpack vs Vite 核心区别**
    - 开发模式：打包 vs 原生 ESM
    - 热更新：全量更新 vs HMR
    - 生产构建：高度优化 vs Rollup

### 实战场景类

1. **如何处理前端大表单性能问题？**
    - 减少响应式数据量
    - 虚拟滚动
    - 防抖节流

2. **如何优化首屏加载速度？**
    - 资源优化：懒加载、压缩、CDN
    - 构建优化：代码分割、Tree-shaking
    - 渲染优化：SSR、骨架屏

3. **微前端方案如何选择？**
    - 异构技术栈选 qiankun
    - 同构技术栈选 Module Federation
    - 考虑样式隔离和 JS 沙箱需求
