# Vue3 核心原理详解

## 一、Vue3 新特性概览

### 1.1 Vue2 vs Vue3 核心对比

| 特性 | Vue2 | Vue3 |
|------|------|------|
| **响应式原理** | Object.defineProperty() | Proxy |
| **API 风格** | Options API | Options API + Composition API |
| **TypeScript 支持** | 较弱 | 原生支持 |
| **性能** | 良好 | 提升 1.3~2 倍 |
| **包体积** | ~23KB | ~10KB（运行时） |
| **Tree-shaking** | 不支持 | 支持 |
| **多根节点** | 不支持 | 支持（Fragments） |
| **Teleport** | 无 | 内置组件 |
| **Suspense** | 无 | 内置组件 |

### 2.2 响应式 API 详解

```javascript
// ===== ref：基本类型响应式 =====
import { ref, isRef, unref, toRef, toRefs } from 'vue';

const count = ref(0);
console.log(count.value); // 0

count.value++; // 修改需要使用 .value

// 判断是否为 ref
console.log(isRef(count)); // true

// unref：如果是 ref 返回 value，否则返回原值
console.log(unref(count)); // 1
console.log(unref(123));   // 123

// ===== reactive：对象响应式 =====
import { reactive, isReactive, isProxy, readonly } from 'vue';

const state = reactive({
    name: 'Vue3',
    version: 3.2,
    features: ['Composition API', 'Teleport', 'Suspense']
});

// 直接访问属性（无需 .value）
console.log(state.name); // 'Vue3'

// 嵌套对象自动响应式
state.features.push('Script Setup');

// 判断是否为 reactive
console.log(isReactive(state)); // true

// 只读对象
const readonlyState = readonly(state);
readonlyState.name = 'New'; // 警告，修改失败

// ===== toRef / toRefs：解构保持响应式 =====
const user = reactive({
    name: 'John',
    age: 25
});

// toRef：创建单个属性的 ref
const ageRef = toRef(user, 'age');

// toRefs：批量创建 refs
const { name, age } = toRefs(user);
// name 和 age 都是 ref，保持响应式

// 在 setup 中返回时使用
return {
    ...toRefs(user)  // 解构后仍保持响应式
};

// ===== computed：计算属性 =====
import { computed } from 'vue';

// 只读计算属性
const fullName = computed(() => `${firstName.value} ${lastName.value}`);

// 可写计算属性
const fullNameWritable = computed({
    get: () => `${firstName.value} ${lastName.value}`,
    set: (val) => {
        [firstName.value, lastName.value] = val.split(' ');
    }
});

// ===== watch / watchEffect =====
import { watch, watchEffect } from 'vue';

// watch：侦听特定数据源
watch(count, (newVal, oldVal) => {
    console.log('count 变化:', oldVal, '→', newVal);
});

// 侦听多个数据源
watch([count, name], ([newCount, newName], [oldCount, oldName]) => {
    console.log('变化:', newCount, newName);
});

// 深度侦听
watch(
    () => state.nested,
    (newVal) => console.log(newVal),
    { deep: true, immediate: true }
);

// watchEffect：自动追踪依赖
const stop = watchEffect(() => {
    // 自动追踪 count 和 name 的依赖
    console.log('effect:', count.value, name.value);
});

// 停止侦听
stop();

// ===== 工具函数 =====
import { shallowRef, shallowReactive, triggerRef, customRef } from 'vue';

// shallowRef：只对 .value 的替换响应，内部属性不响应
const shallow = shallowRef({ nested: { count: 0 } });
shallow.value.nested.count++; // 不会触发更新
shallow.value = { nested: { count: 1 } }; // 会触发更新
triggerRef(shallow); // 强制触发更新

// shallowReactive：顶层属性响应，嵌套属性不响应
const shallowState = shallowReactive({
    nested: { count: 0 }
});

// customRef：自定义 ref
function useDebouncedRef(value, delay = 300) {
    let timeout;
    return customRef((track, trigger) => ({
        get() {
            track();
            return value;
        },
        set(newValue) {
            clearTimeout(timeout);
            timeout = setTimeout(() => {
                value = newValue;
                trigger();
            }, delay);
        }
    }));
}

const searchText = useDebouncedRef('', 500);
```

            console.log('已卸载');

### 2.5 组合式函数（Composables）

```javascript
// composables/useMouse.js
import { ref, onMounted, onUnmounted } from 'vue';

export function useMouse() {
    const x = ref(0);
    const y = ref(0);
    
    function update(event) {
        x.value = event.pageX;
        y.value = event.pageY;
    }
    
    onMounted(() => window.addEventListener('mousemove', update));
    onUnmounted(() => window.removeEventListener('mousemove', update));
    
    return { x, y };
}

// composables/useFetch.js
import { ref, watchEffect, toValue } from 'vue';

export function useFetch(url) {
    const data = ref(null);
    const error = ref(null);
    const loading = ref(false);
    
    const fetchData = async () => {
        loading.value = true;
        error.value = null;
        
        try {
            const res = await fetch(toValue(url));
            data.value = await res.json();
        } catch (err) {
            error.value = err;
        } finally {
            loading.value = false;
        }
    };
    
    watchEffect(() => {
        fetchData();
    });
    
    return { data, error, loading, refresh: fetchData };
}

// composables/useLocalStorage.js
import { ref, watch } from 'vue';

export function useLocalStorage(key, defaultValue) {
    const stored = localStorage.getItem(key);
    const data = ref(stored ? JSON.parse(stored) : defaultValue);
    
    watch(data, (newVal) => {
        localStorage.setItem(key, JSON.stringify(newVal));
    }, { deep: true });
    
    return data;
}

// 在组件中使用
<script setup>
import { useMouse } from './composables/useMouse';
import { useFetch } from './composables/useFetch';
import { useLocalStorage } from './composables/useLocalStorage';

const { x, y } = useMouse();
const { data: user, loading, error } = useFetch('/api/user');
const preferences = useLocalStorage('preferences', { theme: 'light' });
</script>
```

## 四、Vue3 响应式原理（Proxy）

### 4.1 Proxy 基础

```javascript
// Vue3 使用 Proxy 替代 Object.defineProperty

// 模拟 reactive 实现
function reactive(target) {
    if (typeof target !== 'object' || target === null) {
        return target;
    }
    
    return new Proxy(target, {
        // 拦截读取操作
        get(target, key, receiver) {
            const result = Reflect.get(target, key, receiver);
            
            // 收集依赖（track）
            track(target, key);
            
            // 递归处理嵌套对象
            if (typeof result === 'object') {
                return reactive(result);
            }
            
            return result;
        },
        
        // 拦截设置操作
        set(target, key, value, receiver) {
            const oldValue = target[key];
            const result = Reflect.set(target, key, value, receiver);
            
            // 值变化时触发更新（trigger）
            if (oldValue !== value) {
                trigger(target, key);
            }
            
            return result;
        },
        
        // 拦截删除操作
        deleteProperty(target, key) {
            const hadKey = Object.prototype.hasOwnProperty.call(target, key);
            const result = Reflect.deleteProperty(target, key);
            
            if (hadKey && result) {
                trigger(target, key);
            }
            
            return result;
        },
        
        // 拦截 in 操作符
        has(target, key) {
            track(target, key);
            return Reflect.has(target, key);
        },
        
        // 拦截 for...in 循环
        ownKeys(target) {
            track(target, 'iterate');
            return Reflect.ownKeys(target);
        }
    });
}

// 依赖收集和触发
const targetMap = new WeakMap();
let activeEffect = null;

function track(target, key) {
    if (!activeEffect) return;
    
    let depsMap = targetMap.get(target);
    if (!depsMap) {
        targetMap.set(target, (depsMap = new Map()));
    }
    
    let dep = depsMap.get(key);
    if (!dep) {
        depsMap.set(key, (dep = new Set()));
    }
    
    dep.add(activeEffect);
}

function trigger(target, key) {
    const depsMap = targetMap.get(target);
    if (!depsMap) return;
    
    const dep = depsMap.get(key);
    if (dep) {
        dep.forEach(effect => effect());
    }
}

function effect(fn) {
    activeEffect = fn;
    fn();
    activeEffect = null;
}

// 使用示例
const state = reactive({ count: 0, user: { name: 'Vue' } });

effect(() => {
    console.log('count:', state.count);
});

state.count++; // 自动触发 effect
state.user.name = 'Vue3'; // 嵌套对象也是响应式的
```

### 4.2 ref 的实现原理

```javascript
// ref 的实现基于 Proxy 和类

function ref(value) {
    return createRef(value, false);
}

function shallowRef(value) {
    return createRef(value, true);
}

function createRef(rawValue, shallow) {
    if (isRef(rawValue)) {
        return rawValue;
    }
    
    return new RefImpl(rawValue, shallow);
}

class RefImpl {
    constructor(value, __v_isShallow) {
        this.__v_isShallow = __v_isShallow;
        this._value = __v_isShallow ? value : toReactive(value);
        this._rawValue = value;
    }
    
    get value() {
        // 收集依赖
        track(this, 'value');
        return this._value;
    }
    
    set value(newVal) {
        if (hasChanged(newVal, this._rawValue)) {
            this._rawValue = newVal;
            this._value = this.__v_isShallow ? newVal : toReactive(newVal);
            // 触发更新
            trigger(this, 'value');
        }
    }
}

function toReactive(value) {
    return isObject(value) ? reactive(value) : value;
}

function isRef(r) {
    return !!(r && r.__v_isRef === true);
}

function hasChanged(value, oldValue) {
    return !Object.is(value, oldValue);
}
```

**问题 1：Vue3 的响应式原理与 Vue2 有什么区别？**

**答：**

| 特性 | Vue2 | Vue3 |
|------|------|------|
| **实现方式** | Object.defineProperty() | Proxy |
| **数组监听** | 重写数组方法 | Proxy 原生支持 |
| **新增属性** | 需要 Vue.set() | 直接支持 |
| **删除属性** | 需要 Vue.delete() | 直接支持 |
| **Map/Set** | 不支持 | 原生支持 |
| **性能** | 初始化时递归遍历 | 懒递归，按需代理 |

**Vue3 Proxy 优势：**

1. **全面拦截**：可以拦截更多操作（in、delete、has 等）
2. **嵌套对象**：懒递归，访问时才代理
3. **数组**：原生支持索引和 length 修改
4. **性能**：初始化更快，内存占用更少

```javascript
// Vue2 的局限
const obj = {};
Vue.set(obj, 'newProp', 'value'); // 必须这样添加新属性

// Vue3 的便利
const obj = reactive({});
obj.newProp = 'value'; // 直接支持
```

**问题 2：Composition API 与 Options API 的区别？**

**答：**

| 特性 | Options API | Composition API |
|------|-------------|-----------------|
| **代码组织** | 按选项类型（data、methods） | 按功能/逻辑组合 |
| **逻辑复用** | Mixins（命名冲突、来源不明） | Composables（清晰、灵活） |
| **TypeScript** | 支持较差 | 原生支持 |
| **代码量** | 较少时清晰 | 复杂逻辑更清晰 |
| **tree-shaking** | 不支持 | 支持 |

**Composition API 优势：**

1. **逻辑复用**：通过函数复用逻辑，避免 mixins 的弊端
2. **代码组织**：相关逻辑放在一起，而非分散在不同选项
3. **类型推断**：更好的 TypeScript 支持

```javascript
// Options API - 逻辑分散
export default {
    data() {
        return { count: 0, x: 0, y: 0 };
    },
    mounted() {
        window.addEventListener('mousemove', this.updateMouse);
    },
    beforeDestroy() {
        window.removeEventListener('mousemove', this.updateMouse);
    },
    methods: {
        updateMouse(e) { this.x = e.x; this.y = e.y; }
    }
};

// Composition API - 逻辑聚合
export default {
    setup() {
        const { count, increment } = useCounter();
        const { x, y } = useMouse();
        return { count, increment, x, y };
    }
};
```

**问题 3：ref 和 reactive 的区别？**

**答：**

| 特性 | ref | reactive |
|------|-----|----------|
| **数据类型** | 基本类型和对象 | 仅对象（对象、数组、Map、Set） |
| **访问方式** | .value | 直接访问属性 |
| **模板中使用** | 自动解包，无需 .value | 直接访问 |
| **替换整个对象** | 可以 | 会失去响应式 |
| **解构** | 保持响应式 | 失去响应式（需用 toRefs） |

**使用建议：**

- **ref**：基本类型、需要替换整个对象的场景、从 composable 返回的数据
- **reactive**：表单数据、复杂对象、多个相关属性

```javascript
// ref 可以替换整个对象
const state = ref({ count: 0 });
state.value = { count: 1 }; // 仍然响应式

// reactive 替换会失去响应式
const state = reactive({ count: 0 });
state = { count: 1 }; // 错误！失去响应式
```

**问题 10：Vue3 如何兼容 Vue2 的写法？**

**答：**

Vue3 完全兼容 Options API，可以平滑迁移：

```javascript
// Vue2 代码在 Vue3 中可以直接使用
export default {
    data() {
        return { count: 0 };
    },
    computed: {
        double() {
            return this.count * 2;
        }
    },
    methods: {
        increment() {
            this.count++;
        }
    },
    mounted() {
        console.log('mounted');
    }
};
```

**迁移策略：**

1. **渐进式迁移**：新功能使用 Composition API
2. **兼容库**：@vue/composition-api（Vue2 使用 Composition API）
3. **迁移构建**：使用 @vue/compat 进行逐步迁移

**注意事项：**

- 移除的 API：$children、$destroy、filters、Event Bus 等
- 变更的 API：生命周期钩子名称、v-model 语法
