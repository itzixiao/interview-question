# Vue2 核心原理详解

## 一、Vue2 核心概念

### 1.1 MVVM 架构

Vue2 采用 **MVVM（Model-View-ViewModel）** 架构模式：

```
Model（数据层） ←→ ViewModel（Vue实例） ←→ View（视图层）
     ↑                    ↓                      ↑
   数据变更            数据劫持+发布订阅         DOM更新
```

**核心理解：**

- **Model**：数据模型，普通的 JavaScript 对象
- **View**：视图层，即 DOM 元素
- **ViewModel**：Vue 实例，负责数据绑定和 DOM 更新

### 1.2 响应式原理

Vue2 的响应式系统基于 **Object.defineProperty()** 实现数据劫持。

#### 数据劫持示例

```javascript
// 模拟 Vue2 响应式原理
function defineReactive(obj, key, val) {
    // 递归处理嵌套对象
    observe(val);
    
    // 创建依赖收集器
    const dep = new Dep();
    
    Object.defineProperty(obj, key, {
        enumerable: true,
        configurable: true,
        get() {
            // 收集依赖
            if (Dep.target) {
                dep.depend();
            }
            return val;
        },
        set(newVal) {
            if (newVal === val) return;
            val = newVal;
            // 触发更新
            dep.notify();
        }
    });
}

// 依赖收集器
class Dep {
    constructor() {
        this.subs = []; // 存储所有 Watcher
    }
    
    depend() {
        if (Dep.target) {
            this.subs.push(Dep.target);
        }
    }
    
    notify() {
        this.subs.forEach(watcher => watcher.update());
    }
}

// Watcher 观察者
class Watcher {
    constructor(vm, expOrFn, cb) {
        this.vm = vm;
        this.getter = expOrFn;
        this.cb = cb;
        this.value = this.get();
    }
    
    get() {
        Dep.target = this;
        const value = this.getter.call(this.vm);
        Dep.target = null;
        return value;
    }
    
    update() {
        const oldValue = this.value;
        this.value = this.get();
        this.cb.call(this.vm, this.value, oldValue);
    }
}

// 观察对象
function observe(obj) {
    if (typeof obj !== 'object' || obj === null) {
        return;
    }
    
    Object.keys(obj).forEach(key => {
        defineReactive(obj, key, obj[key]);
    });
}

// 使用示例
const data = { name: 'Vue', count: 0 };
observe(data);

// 模拟 Watcher
new Watcher({ name: 'Vue', count: 0 }, function() {
    return this.name + ': ' + this.count;
}, function(newVal, oldVal) {
    console.log('更新：', oldVal, '→', newVal);
});

data.count++; // 触发更新
```

#### 数组响应式处理

Vue2 对数组的响应式处理采用**重写数组方法**：

```javascript
// 需要重写的方法
const arrayProto = Array.prototype;
const arrayMethods = Object.create(arrayProto);

const methodsToPatch = [
    'push',
    'pop',
    'shift',
    'unshift',
    'splice',
    'sort',
    'reverse'
];

methodsToPatch.forEach(method => {
    const original = arrayProto[method];
    Object.defineProperty(arrayMethods, method, {
        value: function mutator(...args) {
            const result = original.apply(this, args);
            const ob = this.__ob__;
            
            // 处理新增元素
            let inserted;
            switch (method) {
                case 'push':
                case 'unshift':
                    inserted = args;
                    break;
                case 'splice':
                    inserted = args.slice(2);
                    break;
            }
            if (inserted) ob.observeArray(inserted);
            
            // 通知更新
            ob.dep.notify();
            return result;
        },
        enumerable: false,
        writable: true,
        configurable: true
    });
});
```

#### Diff 算法核心逻辑

```javascript
// 简化版 Diff 算法
function patch(oldVnode, vnode) {
    // 相同节点
    if (sameVnode(oldVnode, vnode)) {
        patchVnode(oldVnode, vnode);
    } else {
        // 不同节点，直接替换
        const oldElm = oldVnode.elm;
        const parentElm = oldElm.parentNode;
        
        createElm(vnode);
        
        if (parentElm) {
            parentElm.insertBefore(vnode.elm, oldElm);
            parentElm.removeChild(oldElm);
        }
    }
    return vnode;
}

// 判断是否为相同节点
function sameVnode(a, b) {
    return (
        a.key === b.key &&
        a.tag === b.tag &&
        a.isComment === b.isComment &&
        isDef(a.data) === isDef(b.data)
    );
}

// 对比两个节点
function patchVnode(oldVnode, vnode) {
    const elm = vnode.elm = oldVnode.elm;
    
    // 更新属性
    updateAttrs(oldVnode, vnode);
    
    const oldCh = oldVnode.children;
    const ch = vnode.children;
    
    if (!ch) {
        // 新节点没有子节点
        if (oldCh) {
            removeChildren(elm, oldCh);
        }
        if (oldVnode.text !== vnode.text) {
            elm.textContent = vnode.text;
        }
    } else {
        // 新节点有子节点
        if (!oldCh) {
            // 老节点没有子节点
            elm.textContent = '';
            addChildren(elm, ch);
        } else {
            // 都有子节点，进行 diff
            updateChildren(elm, oldCh, ch);
        }
    }
}

// 核心：子节点对比算法
function updateChildren(parentElm, oldCh, newCh) {
    let oldStartIdx = 0;
    let newStartIdx = 0;
    let oldEndIdx = oldCh.length - 1;
    let newEndIdx = newCh.length - 1;
    let oldStartVnode = oldCh[0];
    let oldEndVnode = oldCh[oldEndIdx];
    let newStartVnode = newCh[0];
    let newEndVnode = newCh[newEndIdx];
    
    while (oldStartIdx <= oldEndIdx && newStartIdx <= newEndIdx) {
        // 头头对比
        if (sameVnode(oldStartVnode, newStartVnode)) {
            patchVnode(oldStartVnode, newStartVnode);
            oldStartVnode = oldCh[++oldStartIdx];
            newStartVnode = newCh[++newStartIdx];
        }
        // 尾尾对比
        else if (sameVnode(oldEndVnode, newEndVnode)) {
            patchVnode(oldEndVnode, newEndVnode);
            oldEndVnode = oldCh[--oldEndIdx];
            newEndVnode = newCh[--newEndIdx];
        }
        // 头尾对比
        else if (sameVnode(oldStartVnode, newEndVnode)) {
            patchVnode(oldStartVnode, newEndVnode);
            parentElm.insertBefore(oldStartVnode.elm, oldEndVnode.elm.nextSibling);
            oldStartVnode = oldCh[++oldStartIdx];
            newEndVnode = newCh[--newEndIdx];
        }
        // 尾头对比
        else if (sameVnode(oldEndVnode, newStartVnode)) {
            patchVnode(oldEndVnode, newStartVnode);
            parentElm.insertBefore(oldEndVnode.elm, oldStartVnode.elm);
            oldEndVnode = oldCh[--oldEndIdx];
            newStartVnode = newCh[++newStartIdx];
        }
        // 以上都不匹配
        else {
            // 创建 key 索引
            const keyMap = {};
            for (let i = oldStartIdx; i <= oldEndIdx; i++) {
                const key = oldCh[i].key;
                if (key) keyMap[key] = i;
            }
            
            // 查找新节点在老列表中的位置
            const idxInOld = keyMap[newStartVnode.key];
            
            if (idxInOld === undefined) {
                // 新节点，创建插入
                createElm(newStartVnode, parentElm, oldStartVnode.elm);
            } else {
                // 移动节点
                const vnodeToMove = oldCh[idxInOld];
                patchVnode(vnodeToMove, newStartVnode);
                oldCh[idxInOld] = undefined;
                parentElm.insertBefore(vnodeToMove.elm, oldStartVnode.elm);
            }
            newStartVnode = newCh[++newStartIdx];
        }
    }
    
    // 处理剩余节点
    if (oldStartIdx > oldEndIdx) {
        // 新增节点
        addChildren(parentElm, newCh, newStartIdx, newEndIdx);
    } else if (newStartIdx > newEndIdx) {
        // 删除节点
        removeChildren(parentElm, oldCh, oldStartIdx, oldEndIdx);
    }
}
```

```javascript
// Vue2 组件生命周期完整示例
export default {
    name: 'LifecycleDemo',
    
    // ===== 创建阶段 =====
    beforeCreate() {
        // 实例初始化，数据观测和事件配置之前
        console.log('beforeCreate: 数据未初始化', this.message); // undefined
    },
    
    created() {
        // 实例创建完成，数据观测已配置，但 DOM 未生成
        console.log('created: 数据已初始化', this.message);
        // 适合：数据初始化、API 请求
    },
    
    // ===== 挂载阶段 =====
    beforeMount() {
        // 模板编译/挂载之前
        console.log('beforeMount: DOM 即将挂载');
    },
    
    mounted() {
        // DOM 挂载完成
        console.log('mounted: DOM 已挂载', this.$el);
        // 适合：DOM 操作、第三方库初始化
    },
    
    // ===== 更新阶段 =====
    beforeUpdate() {
        // 数据更新，DOM 重新渲染之前
        console.log('beforeUpdate: 数据已更新，DOM 即将重新渲染');
    },
    
    updated() {
        // DOM 重新渲染完成
        console.log('updated: DOM 已更新');
        // 注意：避免在此处修改数据，可能导致死循环
    },
    
    // ===== 销毁阶段 =====
    beforeDestroy() {
        // 实例销毁之前
        console.log('beforeDestroy: 实例即将销毁');
        // 适合：清除定时器、取消订阅、解绑事件
    },
    
    destroyed() {
        // 实例销毁完成
        console.log('destroyed: 实例已销毁');
    },
    
    // =====  keep-alive 专属生命周期 =====
    activated() {
        // 被 keep-alive 缓存的组件激活时调用
        console.log('activated: 组件被激活');
    },
    
    deactivated() {
        // 被 keep-alive 缓存的组件停用时调用
        console.log('deactivated: 组件被停用');
    },
    
    // ===== 错误处理 =====
    errorCaptured(err, vm, info) {
        // 捕获子组件错误
        console.error('errorCaptured:', err, info);
        return false; // 阻止错误继续向上传播
    },
    
    data() {
        return {
            message: 'Hello Vue2'
        };
    }
};
```

## 五、Vue Router

```javascript

        next();
    }
```

## 八、高频面试题

**问题 1：Vue2 的响应式原理是什么？**

**答：**

Vue2 的响应式原理基于 **Object.defineProperty()** 实现数据劫持，结合**发布-订阅模式**实现数据变化到视图的自动更新。

**核心流程：**

1. **初始化阶段**：Vue 遍历 data 对象的所有属性，使用 `Object.defineProperty()` 为每个属性添加 getter 和 setter
2. **依赖收集阶段**：组件渲染时触发 getter，将当前组件的 Watcher 收集到该属性的 Dep（依赖收集器）中
3. **派发更新阶段**：数据变化时触发 setter，通知 Dep 中的所有 Watcher 执行更新

**局限性：**

| 问题 | 说明 | 解决方案 |
|------|------|---------|
| 无法检测新增属性 | Object.defineProperty 只能劫持已存在的属性 | 使用 Vue.set() 或提前声明 |
| 无法检测删除属性 | delete 操作无法被拦截 | 使用 Vue.delete() |
| 数组响应式限制 | 无法直接通过索引设置值、修改 length | 使用 Vue 提供的数组方法 |

**代码示例：**

```javascript
// 响应式原理简化实现
function defineReactive(obj, key, val) {
    const dep = new Dep();
    
    Object.defineProperty(obj, key, {
        get() {
            if (Dep.target) {
                dep.depend(); // 收集依赖
            }
            return val;
        },
        set(newVal) {
            if (newVal === val) return;
            val = newVal;
            dep.notify(); // 触发更新
        }
    });
}
```

**问题 2：Vue2 的 Diff 算法原理是什么？**

**答：**

Vue2 使用**双端比较算法**进行虚拟 DOM 的 Diff，时间复杂度为 O(n)。

**核心策略：**

1. **同层比较**：只比较同一层级的节点，不跨层级比较
2. **双端比较**：使用四个指针（新旧列表的头尾）进行比较
3. **key 优化**：通过 key 快速定位相同节点

**比较流程：**

```
旧列表：[a, b, c, d]    新列表：[b, a, d, c]
         ↑     ↑              ↑     ↑
       start  end           start  end
```

1. **头头比较**：旧头 vs 新头
2. **尾尾比较**：旧尾 vs 新尾
3. **头尾交叉**：旧头 vs 新尾、旧尾 vs 新头
4. **key 查找**：以上都不匹配时，使用 key 在旧列表中查找

**优势：**

- 最大程度复用已有 DOM 节点
- 减少 DOM 操作次数
- key 的使用使列表更新更高效

**问题 3：computed 和 watch 的区别是什么？**

**答：**

| 特性 | computed | watch |
|------|----------|-------|
| **本质** | 计算属性，基于依赖缓存 | 侦听器，监听数据变化 |
| **缓存** | 有缓存，依赖不变不重新计算 | 无缓存，每次变化都执行 |
| **适用场景** | 根据已有数据计算新值 | 数据变化时执行异步或复杂操作 |
| **返回值** | 必须有返回值 | 不需要返回值 |
| **立即执行** | 默认立即执行 | 需设置 immediate: true |

**使用建议：**

- **computed**：简单的数据转换、过滤、排序
- **watch**：异步请求、复杂业务逻辑、监听路由变化

**问题 6：Vue Router 的导航守卫有哪些？执行顺序是什么？**

**答：**

**导航守卫类型：**

1. **全局前置守卫**：`router.beforeEach`
2. **全局解析守卫**：`router.beforeResolve`
3. **路由独享守卫**：`beforeEnter`
4. **组件内守卫**：`beforeRouteEnter`、`beforeRouteUpdate`、`beforeRouteLeave`

```

6. 解析异步路由组件
7. beforeRouteUpdate（组件复用时）
8. beforeResolve（全局解析）
9. 导航被确认
10. afterEach（全局后置）
11. DOM 更新
12. beforeRouteEnter 的 next 回调执行
```

**问题 7：Vuex 的 mutation 和 action 有什么区别？**

| 特性 | mutation | action |
|------|----------|--------|
| **同步/异步** | 必须是同步的 | 可以包含异步操作 |
| **直接修改 state** | 是 | 否，通过 commit mutation |

**问题 8：Vue2 的生命周期有哪些？created 和 mounted 的区别？**

| 创建 | beforeCreate、created | 数据观测配置完成 |

**问题 9：为什么 Vue2 中 data 必须是函数？**

**答：**

**原因：保证组件实例之间的数据隔离。**

```javascript
// ❌ 错误：使用对象
const MyComponent = {
    data: {
        count: 0  // 所有实例共享同一个对象
    }
};

// ✅ 正确：使用函数
const MyComponent = {
    data() {
        return {
            count: 0  // 每个实例返回独立的对象
        };
    }
};
```

**原理：**

- Vue 组件可能被复用创建多个实例
- 如果 data 是对象，所有实例会共享同一个引用
- 一个实例修改数据会影响其他实例
- 函数返回新对象，保证每个实例数据独立

**问题 10：Vue2 中的 key 有什么作用？**

**答：**

**key 的主要作用：**

1. **Diff 算法优化**：帮助 Vue 识别节点，提高虚拟 DOM 的对比效率
2. **强制重新渲染**：key 变化时，组件会被销毁重建
3. **保持组件状态**：配合 keep-alive 使用

**使用场景：**

```vue
<!-- 列表渲染必须加 key -->
<div v-for="item in list" :key="item.id">{{ item.name }}</div>

<!-- 强制重新渲染 -->
<component :key="refreshKey" />
<button @click="refreshKey++">刷新组件</button>
```

**注意事项：**

- 避免使用 index 作为 key（除非列表不会变化）
- key 应该是唯一且稳定的标识
- 错误的 key 会导致性能问题或渲染错误
