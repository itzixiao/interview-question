# 前端工程化详解

## 一、构建工具深度对比

### 1.1 Webpack vs Vite 核心差异

| 特性           | Webpack    | Vite           |
|--------------|------------|----------------|
| **启动速度**     | 慢（需打包整个应用） | 快（原生 ESM，按需编译） |
| **构建速度**     | 中等         | 快（Rollup 生产构建） |
| **热更新(HMR)** | 较慢         | 极快（ESM 原生支持）   |
| **配置复杂度**    | 高          | 低（约定优于配置）      |
| **生态成熟度**    | 非常成熟       | 快速发展中          |
| **浏览器兼容性**   | 支持旧浏览器     | 现代浏览器优先        |

```javascript

const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const { CleanWebpackPlugin } = require('clean-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');

const { BundleAnalyzerPlugin } = require('webpack-bundle-analyzer');

module.exports = {
    // 入口配置
    entry: {
        main: './src/index.js',
        vendor: ['vue', 'vue-router', 'vuex'] // 代码分割
    },
    
    // 输出配置

        path: path.resolve(__dirname, 'dist'),
        filename: '[name].[contenthash:8].js', // 内容哈希缓存
        chunkFilename: '[name].[contenthash:8].chunk.js',
        publicPath: '/'
    },
    
    // 模块解析

        extensions: ['.js', '.vue', '.json', '.ts'],
        alias: {
            '@': path.resolve(__dirname, 'src'),
            'components': path.resolve(__dirname, 'src/components')
        },
        modules: ['node_modules', path.resolve(__dirname, 'src')]
    },
    
    // 模块处理
    module: {
        rules: [
            // JavaScript/TypeScript
            {
                test: /\.(js|ts)x?$/,
                exclude: /node_modules/,
                use: [
                    {
                        loader: 'babel-loader',
                        options: {
                            presets: [
                                ['@babel/preset-env', { 
                                    useBuiltIns: 'usage', // 按需 polyfill
                                    corejs: 3 
                                }],
                                '@babel/preset-typescript'
                            ],
                            plugins: [
                                '@babel/plugin-syntax-dynamic-import',
                                ['@babel/plugin-transform-runtime', {
                                    corejs: 3
                                }]
                            ],
                            cacheDirectory: true // 缓存加速
                        }
                    }
                ]
            },
            
            // CSS 处理
            {
                test: /\.css$/,
                use: [
                    process.env.NODE_ENV === 'production' 
                        ? MiniCssExtractPlugin.loader 
                        : 'style-loader',
                    'css-loader',
                    'postcss-loader' // Autoprefixer
                ]
            },
            
            // 图片资源
            {
                test: /\.(png|jpe?g|gif|svg|webp)$/i,
                type: 'asset',
                parser: {
                    dataUrlCondition: {
                        maxSize: 8 * 1024 // 8KB 以下转 base64
                    }
                },
                generator: {
                    filename: 'images/[name].[hash:8][ext]'
                }
            },
            
            // 字体文件
            {
                test: /\.(woff2?|eot|ttf|otf)$/,
                type: 'asset/resource',
                generator: {
                    filename: 'fonts/[name].[hash:8][ext]'
                }
            }
        ]
    },
    
    // 插件系统
    plugins: [
        new CleanWebpackPlugin(), // 清理输出目录
        
        new HtmlWebpackPlugin({
            template: './public/index.html',
            favicon: './public/favicon.ico',
            minify: {
                removeComments: true,
                collapseWhitespace: true,
                removeRedundantAttributes: true
            }
        }),
        
        new MiniCssExtractPlugin({
            filename: 'css/[name].[contenthash:8].css',
            chunkFilename: 'css/[name].[contenthash:8].chunk.css'
        }),
        
        // 分析包大小
        process.env.ANALYZE && new BundleAnalyzerPlugin()
    ].filter(Boolean),
    
    // 优化配置
    optimization: {
        // 代码压缩
        minimizer: [
            new TerserPlugin({
                parallel: true, // 多线程
                terserOptions: {
                    compress: {
                        drop_console: true, // 移除 console
                        drop_debugger: true
                    }
                }
            })
        ],
        
        // 代码分割
        splitChunks: {
            chunks: 'all',
            cacheGroups: {
                // 提取第三方库
                vendor: {
                    name: 'vendors',
                    test: /[\\/]node_modules[\\/]/,
                    priority: 10,
                    chunks: 'all'
                },
                // 提取公共代码
                common: {
                    name: 'common',
                    minChunks: 2,
                    priority: 5,
                    reuseExistingChunk: true
                }
            }
        },
        
        // 运行时代码单独提取
        runtimeChunk: {
            name: 'runtime'
        }
    },
    
    // 开发服务器
    devServer: {
        port: 8080,
        hot: true, // 热更新
        open: true,
        proxy: {
            '/api': {
                target: 'http://localhost:3000',
                changeOrigin: true,
                pathRewrite: { '^/api': '' }
            }
        },
        historyApiFallback: true // 单页应用路由支持
    },
    
    // Source Map
    devtool: process.env.NODE_ENV === 'production' 
        ? 'source-map' 
        : 'eval-cheap-module-source-map'
};
```

### 1.3 Vite 核心原理与配置

```javascript
// vite.config.js
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';
import { resolve } from 'path';
import { visualizer } from 'rollup-plugin-visualizer';
import viteCompression from 'vite-plugin-compression';

export default defineConfig(({ mode }) => ({
    // 路径别名
    resolve: {
        alias: {
            '@': resolve(__dirname, 'src'),
            'components': resolve(__dirname, 'src/components')
        }
    },
    
    // 插件
    plugins: [
        vue(),
        
        // Gzip 压缩
        viteCompression({
            algorithm: 'gzip',
            ext: '.gz'
        }),
        
        // Brotli 压缩
        viteCompression({
            algorithm: 'brotliCompress',
            ext: '.br'
        }),
        
        // 包分析
        mode === 'analyze' && visualizer({
            open: true,
            gzipSize: true
        })
    ].filter(Boolean),
    
    // CSS 配置
    css: {
        devSourcemap: true,
        preprocessorOptions: {
            scss: {
                additionalData: `@use "@/styles/vars.scss" as *;`
            }
        }
    },
    
    // 构建配置
    build: {
        target: 'es2015',
        outDir: 'dist',
        assetsDir: 'assets',
        
        // Rollup 配置
        rollupOptions: {
            output: {
                // 手动代码分割
                manualChunks: {
                    'vue-vendor': ['vue', 'vue-router', 'pinia'],
                    'ui-vendor': ['element-plus'],
                    'utils': ['lodash-es', 'dayjs', 'axios']
                },
                
                // 资源文件名
                chunkFileNames: 'js/[name]-[hash].js',
                entryFileNames: 'js/[name]-[hash].js',
                assetFileNames: (assetInfo) => {
                    const info = assetInfo.name.split('.');
                    const ext = info[info.length - 1];
                    
                    if (/\.(png|jpe?g|gif|svg|webp)$/i.test(assetInfo.name)) {
                        return 'images/[name]-[hash][extname]';
                    }
                    if (/\.css$/i.test(assetInfo.name)) {
                        return 'css/[name]-[hash][extname]';
                    }
                    return 'assets/[name]-[hash][extname]';
                }
            }
        },
        
        // 压缩配置
        minify: 'terser',
        terserOptions: {
            compress: {
                drop_console: true,
                drop_debugger: true
            }
        },
        
        // 图片转 base64 阈值
        assetsInlineLimit: 4096
    },
    
    // 开发服务器
    server: {
        port: 3000,
        open: true,
        cors: true,
        proxy: {
            '/api': {
                target: 'http://localhost:8080',
                changeOrigin: true,
                rewrite: (path) => path.replace(/^\/api/, '')
            }
        }
    },
    
    // 预览配置
    preview: {
        port: 4173
    },
    
    // 依赖优化预构建
    optimizeDeps: {
        include: ['vue', 'vue-router', 'pinia', 'axios'],
        exclude: ['your-local-package']
    }
}));
```

### 1.4 Loader 与 Plugin 原理

```javascript
// ===== 自定义 Loader =====
// 实现一个简单的 Markdown Loader
const marked = require('marked');

module.exports = function markdownLoader(source) {
    // this 是 loader 上下文
    const options = this.getOptions();
    
    // 异步处理
    const callback = this.async();
    
    // 转换 Markdown 为 HTML
    const html = marked.parse(source, {
        gfm: true,
        breaks: true
    });
    
    // 返回 Vue 单文件组件格式
    const code = `
        <template>
            <div class="markdown-body">${html}</div>
        </template>
        
        <style scoped>
        .markdown-body {
            line-height: 1.6;
        }
        </style>
    `;
    
    callback(null, code);
};

// ===== 自定义 Plugin =====
// 实现一个生成版本文件的插件
class VersionPlugin {
    constructor(options = {}) {
        this.options = options;
    }
    
    apply(compiler) {
        // 注册钩子
        compiler.hooks.emit.tapAsync('VersionPlugin', (compilation, callback) => {
            const version = this.options.version || require('./package.json').version;
            const buildTime = new Date().toISOString();
            
            const versionInfo = JSON.stringify({
                version,
                buildTime,
                gitCommit: process.env.GIT_COMMIT || 'unknown'
            }, null, 2);
            
            // 添加文件到输出
            compilation.assets['version.json'] = {
                source: () => versionInfo,
                size: () => versionInfo.length
            };
            
            callback();
        });
    }
}

module.exports = VersionPlugin;
```

## 二、Monorepo 实践

### 2.1 Monorepo 方案对比

| 方案                      | 特点            | 适用场景      |
|-------------------------|---------------|-----------|
| **npm/yarn workspaces** | 原生支持，简单易用     | 小型项目，快速上手 |
| **pnpm workspaces**     | 高效磁盘利用，严格依赖管理 | 中大型项目，推荐  |
| **Lerna**               | 版本管理，发布流程     | 多包发布管理    |
| **Nx**                  | 构建缓存，任务编排     | 大型企业级项目   |
| **Turborepo**           | 极速构建，远程缓存     | 高性能要求     |
| **Rush**                | 微软出品，企业级      | 超大型项目     |

## 五、高频面试题

**问题 1：Vite 为什么比 Webpack 快？**

**答：**

| 阶段       | Webpack      | Vite              |
|----------|--------------|-------------------|
| **冷启动**  | 需要打包整个应用     | 利用浏览器原生 ESM，无需打包  |
| **编译方式** | 先打包后服务       | 按需编译，请求时才转译       |
| **HMR**  | 更新整个模块链      | 精确更新，边界清晰         |
| **底层实现** | 基于 Node.js 流 | 基于 esbuild（Go 编写） |

**核心原因：**

1. **开发阶段**：Vite 使用原生 ESM，浏览器直接请求源码，按需编译
2. **预构建**：使用 esbuild 预构建依赖，比 Webpack 快 10-100 倍
3. **HMR**：基于 ESM 的模块图，精确替换变更模块

---

**问题 2：Webpack 的 Tree-shaking 是如何工作的？**

**答：**

Tree-shaking 依赖 ES Module 的**静态结构**特性：

1. **标记阶段**：Webpack 分析模块依赖图，标记未使用的导出
2. **清除阶段**：Terser 插件删除死代码
3. **副作用处理**：通过 `sideEffects` 配置标记有副作用的文件

```json
// package.json
{
    "sideEffects": [
        "*.css",
        "*.scss",
        "./src/polyfill.js"
    ]
}
```

**必要条件：**

- 使用 ES Module（`import/export`）
- 开启 `optimization.usedExports: true`
- 生产模式（`mode: 'production'`）

**问题 3：Monorepo 的优势和挑战是什么？**

**答：**

**优势：**

| 优势   | 说明               |
|------|------------------|
| 代码共享 | 公共组件、工具函数易于复用    |
| 原子提交 | 跨包修改可一次提交，保持版本一致 |
| 统一规范 | 构建、测试、发布流程统一     |
| 可见性  | 所有代码在一个仓库，便于重构   |

**挑战：**

| 挑战    | 解决方案               |
|-------|--------------------|
| 仓库体积大 | 稀疏检出、浅克隆           |
| 构建时间长 | 构建缓存（Turborepo、Nx） |
| 权限管理  | Code Owners、分支保护   |
| 版本管理  | Changesets、Lerna   |

**答：**

| 特性               | npm/yarn   | pnpm               |
|------------------|------------|--------------------|
| **磁盘占用**         | 每个项目复制依赖   | 全局存储，硬链接共享         |
| **node_modules** | 扁平结构，依赖提升  | 严格结构，无幽灵依赖         |
| **安装速度**         | 较慢         | 快（硬链接无需复制）         |
| **monorepo**     | workspaces | workspaces + 更优的链接 |

**核心原理：**

- 所有包存储在全局 `~/.pnpm-store`
- 项目 `node_modules` 通过硬链接指向全局存储
- 严格依赖树，只能访问显式声明的依赖

**问题 6：如何设计一个组件库的构建流程？**

**答：**

```
组件库构建流程：

1. 源码 (TypeScript + Vue/React)
   ↓
2. 类型检查 (tsc --noEmit)
   ↓
3. 构建产物
   ├── ESM (es/) - 用于 Tree-shaking
   ├── CJS (lib/) - 用于 Node.js
   ├── UMD (dist/) - 用于 CDN
   └── 类型声明 (*.d.ts)
   ↓
4. 样式处理
   ├── 编译 SCSS → CSS
   ├── 按需加载样式
   └── 主题变量提取
   ↓
5. 发布
   ├── 版本管理 (Changesets)
   ├── 生成 changelog
   └── npm publish
```

**关键配置：**

- `package.json` 中正确配置 `main`、`module`、`types`、`exports`
- 使用 `vite-plugin-dts` 生成类型声明
- 提供按需引入的 babel 插件
