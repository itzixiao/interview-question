## 一、排序算法

### 1.1 快速排序 ⭐⭐⭐⭐⭐

#### 核心思想
分治法 + 分区操作

#### 时间复杂度
- 最好：O(n log n)
- 平均：O(n log n)
- 最坏：O(n²)

#### 空间复杂度
O(log n) - 递归栈

#### 代码实现
```java
public static void quickSort(int[] arr, int low, int high) {
    if (low < high) {
        int pivotIndex = partition(arr, low, high);
        quickSort(arr, low, pivotIndex - 1);
        quickSort(arr, pivotIndex + 1, high);
    }
}

private static int partition(int[] arr, int low, int high) {
    int pivot = arr[low];
    while (low < high) {
        while (low < high && arr[high] >= pivot) high--;
        arr[low] = arr[high];
        while (low < high && arr[low] <= pivot) low++;
        arr[high] = arr[low];
    }
    arr[low] = pivot;
    return low;
}
```

### 2.1 二分查找 ⭐⭐⭐⭐⭐

#### 前提条件
1. 数组必须有序
2. 支持随机访问

#### 代码实现
```java
public static int binarySearch(int[] arr, int target) {
    int left = 0, right = arr.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}
```

## 三、链表操作

### 3.1 反转链表 ⭐⭐⭐⭐⭐

#### 迭代法
```java
public static ListNode reverseList(ListNode head) {
    ListNode prev = null;
    ListNode curr = head;
    while (curr != null) {
        ListNode nextTemp = curr.next;
        curr.next = prev;
        prev = curr;
        curr = nextTemp;
    }
    return prev;
}
```

#### 递归法
```java
public static ListNode reverseListRecursive(ListNode head) {
    if (head == null || head.next == null) return head;
    ListNode newHead = reverseListRecursive(head.next);
    head.next.next = head;
    head.next = null;
    return newHead;
}
```

```java
public static ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;
    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) {
            curr.next = l1;
            l1 = l1.next;
        } else {
            curr.next = l2;
            l2 = l2.next;
        }
        curr = curr.next;
    }
    curr.next = (l1 != null) ? l1 : l2;
    return dummy.next;
}
```

```java
Stack<TreeNode> stack;
TreeNode curr = root;
while (curr != null || !stack.isEmpty()) {
    while (curr != null) {
        stack.push(curr);
        curr = curr.left;
    }
    curr = stack.pop();
    result.add(curr.val);
    curr = curr.right;
}
```

```java
// 递归实现
```

```java
Queue<TreeNode> queue = new LinkedList<>();
queue.offer(root);
while (!queue.isEmpty()) {
    int size = queue.size();
    List<Integer> level = new ArrayList<>();
    for (int i = 0; i < size; i++) {
        TreeNode node = queue.poll();
        level.add(node.val);
        if (node.left != null) queue.offer(node.left);
        if (node.right != null) queue.offer(node.right);
```

```java
public static int lengthOfLIS(int[] nums) {
    if (nums.length == 0) return 0;
    int[] dp = new int[nums.length];
    Arrays.fill(dp, 1);
    
    int maxLen = 1;
    for (int i = 1; i < nums.length; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[i] > nums[j]) {
                dp[i] = Math.max(dp[i], dp[j] + 1);
            }
        }
        maxLen = Math.max(maxLen, dp[i]);
    }
    return maxLen;
}
```

**问题 4:0-1 背包问题的动态规划解法？**

```java
public static int knapsack(int[] weights, int[] values, int capacity) {
    int n = weights.length;
    int[][] dp = new int[n + 1][capacity + 1];
    
    for (int i = 1; i <= n; i++) {
        for (int w = 0; w <= capacity; w++) {
            dp[i][w] = dp[i - 1][w];  // 不选
            if (w >= weights[i - 1]) {
                dp[i][w] = Math.max(dp[i][w], 
                                   dp[i - 1][w - weights[i - 1]] + values[i - 1]);
            }
        }
    }
    return dp[n][capacity];
}
```

# 一、动态规划问题 - 详解

## 1.1 斐波那契数列

### 问题描述

**题目：** 计算斐波那契数列的第 n 项

**规则：**
- F(0) = 0
- F(1) = 1
- F(n) = F(n-1) + F(n-2), n ≥ 2

**示例：**
```
输入：n = 5
输出：5
解释：0, 1, 1, 2, 3, 5
```

### 解决思路

**方法 1：递归法（不推荐）**
- 时间复杂度：O(2^n)
- 空间复杂度：O(n)
- 存在大量重复计算

**方法 2：动态规划（推荐）**
- 时间复杂度：O(n)
- 空间复杂度：O(1)
- 使用滚动数组优化

**状态转移方程：**
```
dp[i] = dp[i-1] + dp[i-2]
```

### 代码实现

```java
/**
 * 斐波那契数列 - 动态规划解法
 * 
 * @param n 第 n 项
 * @return F(n) 的值
 */
public static int fibonacci(int n) {
    // 边界条件处理
    if (n <= 1) return n;
    
    // 定义三个变量，分别表示 F(n-2), F(n-1), F(n)
    int prev2 = 0;  // F(n-2)，初始为 F(0)
    int prev1 = 1;  // F(n-1)，初始为 F(1)
    int curr = 0;   // F(n)，当前要计算的值
    
    // 从 F(2) 开始迭代计算到 F(n)
    for (int i = 2; i <= n; i++) {
        // 当前项 = 前一项 + 前两项
        curr = prev1 + prev2;
        
        // 更新 prev2 和 prev1，为下一次迭代做准备
        prev2 = prev1;  // F(n-2) 更新为原来的 F(n-1)
        prev1 = curr;   // F(n-1) 更新为原来的 F(n)
    }
    
    // 返回第 n 项的值
    return curr;
}
```

### 执行过程演示

```
计算 F(5):

i=2: curr = 0+1 = 1,  prev2=1, prev1=1
i=3: curr = 1+1 = 2,  prev2=1, prev1=2
i=4: curr = 1+2 = 3,  prev2=2, prev1=3
i=5: curr = 2+3 = 5,  prev2=3, prev1=5

返回：5
```

## 1.2 爬楼梯问题

### 问题描述

**题目：** 假设你正在爬楼梯。需要 n 阶你才能到达楼顶。每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶？

**示例：**
```
输入：n = 3
输出：3
解释：
方法 1: 1 阶 + 1 阶 + 1 阶
方法 2: 1 阶 + 2 阶
方法 3: 2 阶 + 1 阶
```

### 解决思路

**关键观察：**
- 要到第 n 阶，可以从第 (n-1) 阶爬 1 步，或从第 (n-2) 阶爬 2 步
- 因此：f(n) = f(n-1) + f(n-2)
- 这实际上就是斐波那契数列的变形

**状态定义：**
- dp[i]：爬到第 i 阶的方法数

**状态转移方程：**
```
dp[i] = dp[i-1] + dp[i-2]
```

**初始值：**
```
dp[1] = 1  (1 种方法：爬 1 阶)
dp[2] = 2  (2 种方法：1+1 或 2)
```

### 代码实现

```java
/**
 * 爬楼梯问题
 * 
 * @param n 需要爬的台阶数
 * @return 不同的方法数
 */
public static int climbStairs(int n) {
    // 边界条件：1 阶有 1 种方法，2 阶有 2 种方法
    if (n <= 2) return n;
    
    // 定义滚动数组
    int prev2 = 1;  // dp[1] = 1，表示到第 1 阶的方法数
    int prev1 = 2;  // dp[2] = 2，表示到第 2 阶的方法数
    int curr = 0;   // dp[i]，当前要计算的到第 i 阶的方法数
    
    // 从第 3 阶开始迭代计算
    for (int i = 3; i <= n; i++) {
        // 到第 i 阶的方法数 = 到第 (i-1) 阶的方法数 + 到第 (i-2) 阶的方法数
        curr = prev1 + prev2;
        
        // 更新滚动数组
        prev2 = prev1;  // 更新为 dp[i-1]
        prev1 = curr;   // 更新为 dp[i]
    }
    
    // 返回到第 n 阶的方法数
    return curr;
}
```

### 执行过程演示

```
计算 n = 5:

i=3: curr = 2+1 = 3,  prev2=2, prev1=3  (到第 3 阶：1+1+1, 1+2, 2+1)
i=4: curr = 3+2 = 5,  prev2=3, prev1=5  (到第 4 阶：...)
i=5: curr = 5+3 = 8,  prev2=5, prev1=8  (到第 5 阶：...)

返回：8 种方法
```

## 1.3 最长上升子序列（LIS）

### 问题描述

**题目：** 给定一个无序的整数数组，找到其中最长上升子序列的长度。

**示例：**
```
输入：[10, 9, 2, 5, 3, 7, 101, 18]
输出：4
解释：最长的上升子序列是 [2, 3, 7, 101] 或 [2, 3, 7, 18]，长度为 4
```

### 解决思路

**方法 1：动态规划 O(n²)**

**状态定义：**
- dp[i]：以 nums[i] 结尾的最长上升子序列的长度

**状态转移方程：**
```
对于每个 i，遍历所有 j < i：
如果 nums[i] > nums[j]，则 dp[i] = max(dp[i], dp[j] + 1)
```

**初始化：**
```
dp[i] = 1  (每个元素本身构成长度为 1 的上升子序列)
```

### 代码实现

```java
/**
 * 最长上升子序列 - 动态规划解法
 * 
 * @param nums 整数数组
 * @return 最长上升子序列的长度
 */
public static int lengthOfLIS(int[] nums) {
    // 边界条件：空数组返回 0
    if (nums.length == 0) return 0;
    
    // dp[i] 表示以 nums[i] 结尾的最长上升子序列的长度
    int[] dp = new int[nums.length];
    
    // 初始化：每个元素至少可以构成长度为 1 的子序列
    Arrays.fill(dp, 1);
    
    // 记录全局最大长度
    int maxLen = 1;
    
    // 外层循环：依次计算以每个元素结尾的 LIS 长度
    for (int i = 1; i < nums.length; i++) {
        // 内层循环：检查 i 之前的所有元素
        for (int j = 0; j < i; j++) {
            // 如果 nums[i] > nums[j]，说明可以将 nums[i] 接在 nums[j] 后面
            if (nums[i] > nums[j]) {
                // 更新 dp[i]：选择能构成更长子序列的方案
                dp[i] = Math.max(dp[i], dp[j] + 1);
            }
        }
        // 更新全局最大长度
        maxLen = Math.max(maxLen, dp[i]);
    }
    
    // 返回最长上升子序列的长度
    return maxLen;
}
```

### 执行过程演示

```
输入：[10, 9, 2, 5, 3, 7, 101, 18]

初始化：dp = [1, 1, 1, 1, 1, 1, 1, 1]

i=0 (10): dp[0] = 1
i=1 (9):  9 < 10, 不能接，dp[1] = 1
i=2 (2):  2 < 10, 2 < 9, 不能接，dp[2] = 1
i=3 (5):  5 > 2, 可以接在 2 后面，dp[3] = dp[2]+1 = 2  [2,5]
i=4 (3):  3 > 2, 可以接在 2 后面，dp[4] = dp[2]+1 = 2  [2,3]
i=5 (7):  
  - 7 > 5, dp[5] = max(1, dp[3]+1) = 3  [2,5,7]
  - 7 > 3, dp[5] = max(3, dp[4]+1) = 3  [2,3,7]
i=6 (101):
  - 101 > 所有前面的数，选择最大的 dp[5]+1 = 4  [2,5,7,101]
i=7 (18):
  - 18 > 7, dp[7] = dp[5]+1 = 4  [2,5,7,18]

最终：maxLen = 4
```

## 1.4 0-1 背包问题

### 问题描述

**题目：** 有 N 个物品，每个物品有两个属性：重量 w[i] 和价值 v[i]。有一个容量为 W 的背包。每个物品只能选择拿或不拿（不能分割）。求能装入背包的最大价值。

**示例：**
```
物品重量：[2, 3, 4, 5]
物品价值：[3, 4, 5, 6]
背包容量：8

输出：10
解释：选择物品 1(重量 3，价值 4) 和物品 3(重量 5，价值 6)，总重量 8，总价值 10
```

### 解决思路

**状态定义：**
- dp[i][w]：从前 i 个物品中选择，背包容量为 w 时能获得的最大价值

**状态转移方程：**
```
对于第 i 个物品，有两种选择：

1. 不选第 i 个物品：
   dp[i][w] = dp[i-1][w]

2. 选第 i 个物品（前提是装得下）：
   dp[i][w] = dp[i-1][w-w[i]] + v[i]

取两者最大值：
dp[i][w] = max(dp[i-1][w], dp[i-1][w-w[i]] + v[i])
```

### 代码实现

```java
/**
 * 0-1 背包问题 - 动态规划解法
 * 
 * @param weights 物品重量数组
 * @param values 物品价值数组
 * @param capacity 背包容量
 * @return 能装入的最大价值
 */
public static int knapsack(int[] weights, int[] values, int capacity) {
    // 物品数量
    int n = weights.length;
    
    // 创建二维 DP 数组
    // dp[i][w] 表示从前 i 个物品中选择，背包容量为 w 时的最大价值
    int[][] dp = new int[n + 1][capacity + 1];
    
    // 外层循环：依次考虑每个物品（从第 1 个到第 n 个）
    for (int i = 1; i <= n; i++) {
        // 内层循环：依次考虑每种容量（从 0 到 capacity）
        for (int w = 0; w <= capacity; w++) {
            
            // 选择 1：不选第 i 个物品
            // 最大价值等于从前 i-1 个物品中选择，容量为 w 的最大价值
            dp[i][w] = dp[i - 1][w];
            
            // 选择 2：选第 i 个物品（前提是当前容量装得下）
            // 注意：weights[i-1] 是因为物品索引从 0 开始
            if (w >= weights[i - 1]) {
                // 最大价值 = 前 i-1 个物品在剩余容量下的最大价值 + 当前物品的价值
                dp[i][w] = Math.max(
                    dp[i][w],  // 不选
                    dp[i - 1][w - weights[i - 1]] + values[i - 1]  // 选
                );
            }
        }
    }
    
    // 返回结果：从前 n 个物品中选择，容量为 capacity 时的最大价值
    return dp[n][capacity];
}
```

### 执行过程演示

```
物品信息：
物品 0: 重量 2, 价值 3
物品 1: 重量 3, 价值 4
物品 2: 重量 4, 价值 5
物品 3: 重量 5, 价值 6
背包容量：8

DP 表（部分）：
       容量 0  1  2  3  4  5  6  7  8
物品 0   0  0  3  3  3  3  3  3  3
物品 1   0  0  3  4  4  7  7  7  7
物品 2   0  0  3  4  5  7  8  9  9
物品 3   0  0  3  4  5  7  8  9 10  ← 最终结果

决策过程（物品 3，容量 8）：
- 不选物品 3: dp[2][8] = 9
- 选物品 3: dp[2][8-5] + 6 = dp[2][3] + 6 = 4 + 6 = 10
- 选择较大的：max(9, 10) = 10
```

## 二、链表操作 - 详解

### 2.1 反转链表

### 问题描述

**题目：** 给你单链表的头节点 head，请你反转链表，并返回反转后的链表。

**示例：**
```
输入：1 -> 2 -> 3 -> 4 -> 5 -> null
输出：5 -> 4 -> 3 -> 2 -> 1 -> null
```

### 解决思路

**方法 1：迭代法（推荐）**

**核心思想：** 逐个改变每个节点的 next 指针方向

**三个指针：**
- prev：指向前一个节点（初始为 null）
- curr：指向当前节点（初始为 head）
- nextTemp：临时保存下一个节点

**步骤：**
1. 保存下一个节点
2. 当前节点指向前一个节点
3. 移动 prev 和 curr

### 代码实现

```java
/**
 * 链表节点定义
 */
static class ListNode {
    int val;          // 节点值
    ListNode next;    // 指向下一个节点的指针
    ListNode(int x) { val = x; }  // 构造函数
}

/**
 * 反转链表 - 迭代法
 * 
 * @param head 链表头节点
 * @return 反转后的头节点
 */
public static ListNode reverseList(ListNode head) {
    // prev 指向前一个节点，初始为 null（因为反转后头节点的 next 为 null）
    ListNode prev = null;
    
    // curr 指向当前正在处理的节点，初始为原链表的头节点
    ListNode curr = head;
    
    // 遍历链表，直到 curr 为 null（到达原链表末尾）
    while (curr != null) {
        // 步骤 1：先保存下一个节点（因为修改 curr.next 后会丢失）
        ListNode nextTemp = curr.next;
        
        // 步骤 2：将当前节点的 next 指向前一个节点（实现反转）
        curr.next = prev;
        
        // 步骤 3：移动 prev 到当前节点（为下一次迭代做准备）
        prev = curr;
        
        // 步骤 4：移动 curr 到下一个节点（继续处理下一个节点）
        curr = nextTemp;
    }
    
    // 循环结束时，prev 指向原链表的最后一个节点，即反转后的头节点
    return prev;
}
```

### 执行过程演示

```
初始：1 -> 2 -> 3 -> 4 -> 5 -> null

迭代 1 (curr=1):
prev=null, curr=1, nextTemp=2
curr.next = null  (1 -> null)
prev = 1
curr = 2
结果：null <- 1   2 -> 3 -> 4 -> 5

迭代 2 (curr=2):
prev=1, curr=2, nextTemp=3
curr.next = 1  (2 -> 1)
prev = 2
curr = 3
结果：null <- 1 <- 2   3 -> 4 -> 5

... 继续迭代 ...

最终：null <- 1 <- 2 <- 3 <- 4 <- 5
返回：5 (新的头节点)
```

## 三、高频面试题目

### 动态规划专题

**问题 1：斐波那契数列有哪些解法？各自的时间复杂度？**

**答：**

1. **递归法**
   - 时间复杂度：O(2^n)
   - 空间复杂度：O(n)
   - 缺点：大量重复计算

2. **记忆化搜索**
   - 时间复杂度：O(n)
   - 空间复杂度：O(n)
   - 优点：避免重复计算

3. **动态规划（滚动数组）**
   - 时间复杂度：O(n)
   - 空间复杂度：O(1)
   - 最优解法

**问题 2:0-1 背包和完全背包有什么区别？**

**答：**

| 对比项 | 0-1 背包 | 完全背包 |
|--------|---------|---------|
| 物品数量 | 每个物品只能用 1 次 | 每个物品可以用无限次 |
| 状态转移 | 倒序遍历容量 | 正序遍历容量 |
| 应用场景 | 选择问题 | 找零钱问题 |

**问题 3：如何判断一个问题是否适合用动态规划？**

**答：**

满足以下条件：
1. **最优子结构** - 问题的最优解包含子问题的最优解
2. **重叠子问题** - 子问题会被重复计算
3. **无后效性** - 某阶段的状态一旦确定，就不受之后决策的影响

### 链表专题

**问题 1：反转链表的迭代和递归解法各有什么优缺点？**

**答：**

**迭代法：**
- ✅ 优点：空间复杂度 O(1)，效率高
- ❌ 缺点：代码相对复杂

**递归法：**
- ✅ 优点：代码简洁，易于理解
- ❌ 缺点：空间复杂度 O(n)，可能栈溢出

4. **判断两个链表是否相交**

**答：**

**作用：**
1. **简化边界处理** - 避免特殊处理头节点

3. **减少空指针异常** - 提供安全的初始引用
