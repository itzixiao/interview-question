## 🎯 学习建议

### 第一阶段：Redis 基础（2-3 天）

1. **五大基本数据结构**
    - String、List、Set、Hash、ZSet
    - 底层实现原理（SDS、链表、跳表、字典）
    - 其他高级类型（Bitmap、HyperLogLog、Stream）

2. **Redis 高性能原理**
    - 纯内存操作
    - 单线程模型（命令执行）
    - IO 多路复用
    - 高效数据结构

3. **事务与 Lua 脚本**
    - MULTI/EXEC/WATCH 命令
    - Lua 脚本保证原子性

### 第二阶段：缓存应用（3-4 天）

1. **缓存更新策略**
    - Cache Aside（旁路缓存）
    - Read/Write Through
    - Write Behind

2. **缓存三大问题**
    - 缓存穿透：布隆过滤器、空值缓存
    - 缓存击穿：互斥锁、逻辑过期
    - 缓存雪崩：随机 TTL、多级缓存

3. **缓存一致性**
    - 延迟双删
    - Canal 监听 Binlog

### 第三阶段：分布式锁与持久化（3-4 天）

1. **分布式锁演进**
    - SETNX → SET NX EX → Redisson
    - 看门狗自动续期
    - RedLock 算法

2. **持久化机制**
    - RDB 快照的 Copy-On-Write
    - AOF 日志与重写
    - 混合持久化（推荐）

3. **持久化策略选择**
    - 纯缓存场景
    - 数据重要场景
    - 生产环境配置

### 第四阶段：高可用架构（3-4 天）

1. **主从复制**
    - 全量复制与部分复制
    - 复制偏移量与缓冲区
    - 读写分离实现

2. **哨兵模式**
    - 监控与心跳检测
    - 主观/客观下线
    - 自动故障转移

3. **Cluster集群**
    - 16384 个槽位设计
    - MOVED/ASK 重定向
    - 在线扩缩容

### 第五阶段：缓存中间件扩展（2-3 天）

1. **Memcached 对比学习**
    - Slab 内存管理机制
    - LRU 淘汰策略
    - 一致性哈希算法
    - 与 Redis 的差异对比

2. **缓存中间件选型**
    - Redis vs Memcached vs MongoDB
    - 多级缓存架构设计
    - 选型决策树
