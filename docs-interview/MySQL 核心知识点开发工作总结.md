## 四、核心知识点总结

### 4.1 字段类型选择原则

1. **越小越好**：在满足需求的前提下，选择最小的类型
2. **简单为好**：避免复杂类型
3. **避免 NULL**：设置 `NOT NULL DEFAULT ''`

**典型场景：**
- 金额：`DECIMAL(10,2)`
- 手机号：`CHAR(11)`
- 状态标识：`TINYINT(1)`
- 时间存储：`DATETIME`

### 4.2 存储引擎选择

| 场景 | 推荐引擎 |
|------|----------|
| 高并发、事务安全 | InnoDB（默认） |
| 读多写少、不需要事务 | MyISAM |
| 临时表、缓存表 | Memory |
| 日志归档 | Archive |

### 4.3 索引优化黄金法则

1. **为 WHERE、ORDER BY、GROUP BY 创建索引**
2. **使用覆盖索引**（避免回表）
3. **联合索引遵循最左匹配原则**
4. **避免索引失效**（函数、类型转换、LIKE'%xxx'）
5. **定期清理无用索引**

### 4.4 事务隔离级别选择

| 隔离级别 | 适用场景 |
|----------|----------|
| READ COMMITTED | Oracle 默认，并发要求高 |
| REPEATABLE READ | MySQL 默认，推荐 |
| SERIALIZABLE | 对一致性要求极高，性能要求低 |

### 4.5 SQL 优化技巧

**推荐做法：**
- ✅ SELECT 指定字段
- ✅ 小表驱动大表（EXISTS）
- ✅ UNION ALL 代替 UNION
- ✅ 批量操作
- ✅ LIMIT 分页优化（延迟关联）

**避免做法：**
- ❌ SELECT *
- ❌ LIKE '%xxx'
- ❌ OR 连接（有一边没索引）
- ❌ 函数操作
- ❌ 深度分页

## 五、高频面试题精选

### 5.1 字段类型与存储引擎（5 道）

**Q1: VARCHAR(50) 的 50 表示什么？**
- 最多存储 50 个字符
- UTF8 编码下，一个汉字占 3 个字节

**Q2: CHAR 和 VARCHAR 的区别？**
- CHAR：定长，查询效率高
- VARCHAR：变长，节省空间

**Q3: InnoDB 和 MyISAM 的区别？**
- 事务支持、锁粒度、外键支持、MVCC、索引结构

**Q4: 为什么金额要用 DECIMAL？**
- 避免精度丢失

**Q5: DATETIME 和 TIMESTAMP 的区别？**
- 范围、时区影响、占用空间

### 5.2 索引原理（6 道）

**Q1: 为什么 InnoDB 使用 B+Tree？**
- 非叶子节点只存索引
- 查询稳定 O(log N)
- 适合范围查询

**Q2: 聚簇索引和二级索引的区别？**
- 存储内容、数量限制、查询方式

**Q3: 什么是最左匹配原则？**
- 联合索引必须从最左开始匹配

**Q4: 什么是覆盖索引？**
- 二级索引包含所有查询字段，无需回表

**Q5: 索引失效的场景？**
- 7 大常见场景

**Q6: 如何优化慢查询？**
- 6 步分析法

### 5.3 事务与锁（8 道）

**Q1: MySQL 如何保证 ACID？**
- Undo Log、MVCC、Redo Log

**Q2: 脏读、不可重复读、幻读的区别？**
- 定义 + 示例

**Q3: MySQL 默认隔离级别？为什么？**
- REPEATABLE READ，通过 MVCC+Next-Key Lock 基本避免幻读

**Q4: 什么是 MVCC？如何实现？**
- 隐藏列、Undo Log、Read View

**Q5: InnoDB 的行锁算法？**
- Record Lock、Gap Lock、Next-Key Lock

**Q6: 共享锁和排他锁的区别？**
- 兼容性、获取方式

**Q7: 什么是死锁？如何解决？**
- 超时、主动回滚、预防

**Q8: RC 和 RR 级别的区别？**
- Read View 生成时机不同

### 5.4 日志与优化（8 道）

**Q1: Redo Log 和 Binlog 的区别？**
- 所属层级、写入方式、记录内容、作用

**Q2: 什么是两阶段提交？**
- Prepare → Commit，保证一致性
